# 主存储器的组织

主存储器是整个存储系统的核心，它用来存放计算机运行期间所需要的程序和数据。CPU 可直接 **随机** 地对它进行访问

## 主存储器的基本结构

![[Pasted image 20251215221215.png]]

- 存储体 是主存储器的核心，程序和数据都存放在存储体中。 

- 地址译码驱动电路实际上包含译码器和驱动器两个部分。

	- 译码器将 地址线 输入的 地址码 转换成与之对应的 译码输出线上 的 有效电平，以表示选中了某一存储单元，

	- 由驱动器提供驱动电流去驱动相应的读写电路，完成对被选中存储单元的读写操作。

- I/O和读写电路包括 读出放大器、写入电路 和 读写控制电路，用以完成被选中存储单元中各位的读出和写入操作。

主存储器的读写操作是在控制器的控制下进行的

只有接收到来自控制器的 读写命令 或 写允许信号 后，才能实现正确的读写操作

## 主存储器的存储单元

**位** 二进制数的基本单位,存储器存储信息的最小单位

一个二进制数 由若干位组成,当这个二进制数作为一个整体存入或取出时,这个数称为 **存储字**

存放存储字或存储字节的 **主存空间** 称为**存储单元**或**主存单元**,大量存储单元的集合构成一个 **存储体**

为区别存储体中的各个存储单元,将它们逐一编号,存储单元的编号称为 **地址** ,地址与存储单元之间有一一对应的关系

一个存储单元可能存放一个字,也可能一个字节,由计算机的结构确定

字节编址的计算机,最小寻址单位是 **一字节** ,相邻的存储单元地址指向相邻的存储字节

字节编址的计算机,最小寻址单位是 **一个字** ,相邻的存储单元地址指向相邻的存储字

**存储单元** 是 CPU 对主存可访问操作的最小存储单位

## 主存储器的分类

### 材料

半导体材料制造,MOS型和双极型(TTL或ECL电路)

- MOS型存储器 **集成度高**,功耗低,价格低,存取速度慢

- 双极型 存取速度快, **集成度低** ,功耗较大,成本较高

### 存取方式

- 可随机存取存储器RAM

	- CPU 对存储器中的内容随机地存取, CPU 对任何存储单元的写入和读出的时间是一样的

	- 存取时间相同,与物理位置无关

	- 读写方便,使用灵活

- 只读存储器ROM

	- RAM的特殊形式

	- 只随机读出,不能写入

		- 一旦写入存储器就固化,断电数据不丢失

	- 存放某些系统程序 （如BIOS程序）

	- 专用的子程序或用作函数发生器、字符发生器及微程序控制器 中的控制存储器

破坏性读出:某个存储单元所存储的信息被读出时，原存信息会被破坏

非破坏性读出:读出时不破坏



具有 破坏性读出性能的存储器在经历一次读出操作后,必须紧接着一个重写(再生)的操作,回复被破坏的信息

## 技术指标

1. 存储容量

	1. 字节编址的计算机以字节数来表示存储容量；

	2. 字编址的计算机以字数与其字长的乘积来表 示存储容量

		1. 主存容量为64K×16，表示它有64K个存储单元，每个存储单元的字长为16位

		2. 若改用字节数表示则可记为128K字节（即128KB）

2. 存取速度

	1. 存取时间 $T_a$

		1. 访问时间或读写时间

		2. 从启动一次存储器操作到完成该操作所经历的时间

		3. 读出时间是指从CPU向主存发出有效地址和读命令开始，直到将被选单元的内容读出为止所用的时间

		4. 写入时间是指从CPU向主存发出有效地址和写命令开始，直到信息写 入被选中单元为止所用的时间

	2. 存取周期 $T_m$

		1. 读写周期、访存周期

		2. 主存进行一次完整的读写操作所需的全部时间，即连续两次访问存储器操作之间所需要的最短时间

		3. 一般 $T_m> T_a$ 

			1. 在读写操作之后，总要有一段恢复内部状态的复原时间

			2. 破坏性 读出的RAM来说，存取周期往往比存取时间要大得多

	3. 主存带宽 $B_m$

		1. 数据传输率

		2. 每秒从主存进出信息的最大数量，单位为字每秒、字节每秒或位每秒

		![[Pasted image 20251216221602.png]]

		3. 改进措施

			1. 缩短存取周期

			2. 缩短存取周期

			3. 缩短存取周期

3. 可靠性

	1. 在规定的时间内，存储器无故障读写的概率

	2. 平均无故障时间（MTBF）衡量

4. 功耗

	1. 工作功耗 与 维持功耗不同

# 半导体随机存储器和只读存储器

## RAM的记忆单元

记忆单元:存放⼀个⼆进制位的物理器件,是存储器的最基本构件



存储单元:由地址码相同的多个记忆单元构成



RAM分为

- 静态RAM(SRAM)

- 动态RAM(DRAM)

半导体存储器的易失性:只要电源被切断，原来的保存信息便会丢失



SRAM存储器的记忆单元是⽤ **双稳态触发器** 来保存信息的,在记忆单元未被选中或读出时，电路处于双稳态触发器⼯作状态,信息不变



SRAM的存取速度快，但集成度低，功耗也较⼤， 所以⼀般⽤来组成⾼速缓冲存储器和⼩容量主存系统



DRAM存储器是利⽤记忆单元 **电路中栅极电容上的电荷** 来存储信息



刷新:由于栅极电容上 的电荷会随着时间的推移不断泄漏掉，所以每隔⼀定的时间必 须向栅极电容补充⼀次电荷



DRAM集成度⾼，功耗⼩，但存取速度慢，⼀般⽤来组成⼤容量主存系统



## 动态RAM的刷新

### 刷新与重写

- 重写

	- 随机, 某个存储单元被破坏时重写

	- 按 **存储单元** 进行

- 刷新

	- 定时,补充电荷

	- 按 **存储体矩阵的一行** 为单位进行

### 刷新方式

3种

对具有1024个记忆单元（排列成32×32矩阵）的存储芯⽚进⾏刷新，刷新是按⾏进⾏的，且每刷新⼀⾏占⽤⼀个存取周期，存取周期为500ns( $0.5\mu s$ )

#### 集中刷新方式

按照存储芯⽚容量的⼤⼩集中 安排若⼲个刷新周期，刷新时停⽌读写操作

![[Pasted image 20251217141508.png]]

- 刷新周期:刷新一行所需时间

	- 刷新类似 "假读" ->=存取周期

![[Pasted image 20251217142036.png]]

最后32个周期用于刷新

刷新期间 停止读写,被称为 **死区**

存储容量越大,死区越大

#### 分散刷新方式

![[Pasted image 20251217142958.png]]

将刷新分散到每个存取周期

没有死区

缺点

1. 加长了存取周期

	1. 若原本存取周期 $0.5\mu s$ ,增加刷新,周期变为 $1\mu s$

2. 刷新频繁,没有利用最大刷新间隔



#### 异步刷新方式

$$

\text{相邻两⾏的刷新间隔}=\text{最⼤刷新间隔时间}/\text{⾏数}

$$

![[Pasted image 20251217143346.png]]

32×32矩阵，在2ms内需要将32⾏刷新⼀遍，所以相邻两⾏的刷新时间间隔=$2ms/32=62.5 \mu s$，即每隔$62.5 \mu s$安排⼀个刷 新周期，在刷新时封锁读/写

**有死区** 死区小

避免 CPU 连续等待时间过长,减少刷新次数

### 刷新控制

1. 刷新(外部刷新控制电路产⽣刷新地址逐⾏循环地刷新/芯⽚内部的刷新地址计数器⾃动地控制刷新) 不依赖外部的访问,刷新对 CPU 透明

2. 刷新 逐行,刷新操作只需要 行地址,不需要 列地址

3. 类似读出,仅补充电荷,无信息输出. 不需要加 片选信号,所有芯片同时被刷新

4. 所有芯⽚同时被刷新，所以在考虑刷新问题时，应当从 单个芯⽚的存储容量着⼿，⽽不是从整个存储器的容量着⼿



## RAM芯片分析

### RAM芯片

外引脚

1. 地址线

	1. 单向输入

	2. 数目和芯片容量有关 $log_2(\text{字数})$ 

		1. 容量 $1024\times 4$ 位,地址线 $10$ 根

		2. 容量 $64k \times 1$ 位,地址线 $16$ 根

2. 数据线

	1. 双向,可输入可输出

	2. 数目和数据位数有关

		1. 容量 $1024\times 4$ 位,数据线 $4$ 根

		2. 容量 $64k \times 1$ 位,数据线 $1$ 根

3. 控制线

	1. 读写控制线

		1. 控制芯片 是读 是 写

	2. 片选线

		1. 决定芯片 是否被选中

4. 电源线



DRAM芯片 集成度高,容量大

**地址复用技术**

为减少芯片引脚数, 把地址线分成相等的两部分,分两次从相同的引脚输入,称为 **行地址** 和 **列地址** 

- 行地址 : 由行地址选通信号 RAS 送入存储芯片

- 列地址 : 由列地址选通信号 CAS 送入存储芯片

增加一条地址线,实际增加两位地址,增加 $4$ 倍容量

### 地址译码方式

地址译码电路能把地址线送来的地址信号翻译成对应 **存储单元** 的选择信号
