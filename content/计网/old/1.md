## 二、计算机网络概述（1.1）

### 🔹 定义  
> **计算机网络** = 由**自主（autonomous）计算机**通过**互联（interconnected）**形成的**集合体（collection）**

- **自主计算机**：具完整软硬件、独立操作系统（≠终端）
- **互联**：能相互通信
- **集合体**：所有通过通信线路/设备连接的计算机之总和

### ⚖️ 与其他系统的区别
| 类型 | 特点 | 关键差异 |
|------|------|-----------|
| **主从式系统** | 多终端 + 1主机控制 | 终端无OS，从属主机 |
| **分布式系统** | 有统一网络OS，资源自动调度，对用户透明 | 更高级阶段，网络是其技术基础 |

### 🧱 组成
| 类别 | 组件举例 |
|------|----------|
| **硬件** | 主机（端系统：PC/Server/WS）、通信设备（交换机/路由器）、接口（网卡/modem）、传输介质（双绞线/光纤/无线） |
| **软件** | 通信协议（TCP/IP）、应用软件（WWW/E-mail/FTP） |

### 🛠️ 功能
- ✅ **资源共享**：WWW、远程登录、网络文件访问  
- ✅ **远程通信**：E-mail、IP电话、视频会议  
- ✅ **网上事务处理**：电商、电子政务、远程教育/医疗、电子金融

### 🌐 因特网（Internet）意义
- 21世纪核心特征：**数字化、网络化、智能化**
- 自印刷术以来最大通信变革
- 互联网 ≠ 一般 internet（**专有名词 vs 普通名词**）
  - **internet（小写）**：任意互联的网络（泛指）
  - **Internet（大写）**：全球性TCP/IP网络（专指，即“因特网”）

## 三、计算机网络分类（1.2）

### 📌 多维分类法
| 维度 | 类型 |
|------|------|
| 作用范围 | PAN（个域网）、LAN、MAN（城域网）、WAN（广域网） |
| 拓扑结构 | 总线、环形、星型、树形、网状、混合 |
| 通信协议 | TCP/IP、ATM、X.25、FDDI |
| 交换功能 | 电路/报文/分组交换网 |
| 传输媒体 | 双绞线/光纤/卫星/无线网 |
| 服务对象 | 企业网、校园网、政府网 |
| 使用者 | 专用网、公用网 |
| 应用性质 | 远程教育网、证券网等 |

### 🔁 核心分类因素：**传输技术 + 距离尺度**

#### 1️⃣ 传输技术
| 类型 | 特点 | 举例 |
|------|------|------|
| **广播式（Broadcast）** | 单信道共享；所有节点接收；靠地址过滤（广播/多播） | 局域网（如以太网） |
| **点对点（Point-to-Point）** | 多跳路径；中间节点转发；需路由 | 广域网（如Internet骨干网） |

> 📌 规律：**小型/局部网络 → 广播式；大型网络 → 点对点**

#### 2️⃣ 距离尺度对照表
| 距离 | 场景 | 网络类型 |
|------|------|----------|
| 1 m | 个人设备 | PAN（如蓝牙） |
| ≤1 km | 同一建筑/校园 | LAN（如以太网） |
| 10 km | 同一城市 | MAN（如有线电视网） |
| ≥100 km | 国家/洲际 | WAN（如Internet骨干） |
| 10,000 km | 全球 | Internet |

#### 3️⃣ 各类网络特点
| 类型 | 特征 |
|------|------|
| **LAN** | 覆盖小、高带宽（10Mbps–100Gbps）、低误码、单位自建；<br>• 分享式 vs 交换式；<br>• hub → switch演进 |
| **MAN** | 介于LAN/WAN之间；IEEE 802.6（DQDB）；<br>• 技术：千兆以太网、SONET/SDH、WDM、ATM、FDDI |
| **WAN** | 跨地域远程网；主干用**分组交换**；<br>• 结构：资源子网（主机） + 通信子网（交换节点+线路） |

## 四、发展历史（1.3）

### 🧬 起源（1960s）
- 冷战背景 → 美国**ARPA**（高级研究计划局）研发高**生存性网络**
- 动因：传统电路交换网易被摧毁，重连需人工拨号→延迟高
- 新型网络特点：
  - 数据传输非电话
  - 异构机互联
  - 节点平等 → 高生存性
  - 冗余路由
  - 结构简单可靠

### 🔄 演进三阶段（ARPANET → Internet）

| 阶段 | 时间 | 关键事件 | 标志 |
|------|------|---------|------|
| **第一阶段** | 1969–1990 | • ARPANET（首个分组交换网）<br>• 1983：**TCP/IP成为标准协议**<br>• ARPANET → ARPANET + MILNET<br>• 1983–1984：Internet形成<br>• 1990：ARPANET关闭 | 单一网络 → 多网互联 |
| **第二阶段** | 1986–1993 | • NSFNET三级结构：<br> 主干网 → 地区网 → 校园网<br>• 1991：主干网私有化+收费<br>• 1993：主干速率升至**45 Mbps (T3)** | 三级结构成型 |
| **第三阶段** | 1993–至今 | • NSFNET → 商用ISP替换<br>• 1994：4个**NAP**（网络接入点）建立<br>• 形成**多层ISP结构**：<br>  本地ISP → 第二层ISP → NAP → 第一层ISP | 多层次ISP时代 |

#### 🔗 多层ISP路由示例  
```
主机A → 本地ISP → 第二层ISP → NAP → 第一层ISP  
→ NAP → 第二层ISP → 本地ISP → 主机B
```

### 🇨🇳 中国发展里程碑
- **1980**：铁道部联网实验  
- **1989**：首公用分组网 CNPAC  
- **1994.4.20**：64kbps专线接入Internet（**中国Internet元年**）  
- **1994.5**：中科院高能所设首WWW服务器  
- **1994.9**：CHINANET启动  
- 同期：CERNET（教育科研网）建设

## 六、网络体系结构（1.5）

### 🧩 分层思想（核心！）

#### ✅ 分层原因与好处
- 各层独立 → 模块化  
- 灵活性好 → 易替换更新  
- 结构清晰 → 易实现/维护  
- 促进标准化  

#### ⚠️ 层数权衡
- 层数太少 → 协议太复杂  
- 层数太多 → 系统工程困难  
→ 实际：**5层（教学）或4层（TCP/IP）**

---

### 🔍 核心概念辨析

| 概念 | 定义 | 方向/性质 |
|------|------|-----------|
| **层（Layer）** | 信息隐蔽+抽象设计的逻辑功能单元 | 垂直堆叠 |
| **服务（Service）** | （n-1）层向（n）层提供的功能 | **垂直** |
| **接口（Interface）** | 相邻层间交互的边界（SAP） | 水平边界 |
| **协议（Protocol）** | 对等层通信规则（“如何通信”） | **水平** |
| **实体（Entity）** | 可收/发信息的软/硬件进程 | 通信主体 |
| **SAP**（Service Access Point） | 同一机器相邻层交互点（如端口号） | 地址标识 |

> 📌 关键：**协议水平，服务垂直**；上层用户只看见服务，协议对上层**透明**

---

### 🧪 分层设计实例（从LAN到WAN）

#### ▶️ 案例1：设计广播式**局域网**结构（已知：共享介质）
| 问题 | 解决方案 | 对应层 |
|------|----------|--------|
| 物理信号传输？ | 电压/脉冲定义、数→模转换 | **物理层（Layer 1）** |
| 共享信道竞争？ | MAC协议（如CSMA/CD）、物理地址寻址 | **数据链路层（Layer 2）**<br>→ 由**网卡（NIC）实现**（含唯一MAC地址） |
| 多应用并发？速度不匹配？ | 应用标识（端口号）、流控、端到端可靠传输 | **传输层（Layer 4）** |
| 应用通信？ | QQ/HTTP等语义交互 | **应用层（Layer 5）** |

#### ▶️ 案例2：互联异构局域网 → 广域网需求
| 问题 | 解决方案 | 新增层 |
|------|----------|--------|
| 跨网络路由？ | 逻辑地址（IP）、路由算法 | **网络层（Layer 3）** |

→ 构成**五层协议模型**：

```
 5 应用层   ── 应用进程 ↔ 应用进程
 4 传输层   ── 端到端传输（TCP/UDP）
 3 网络层   ── 分组路由（IP）
 2 数据链路层 ── 帧传输、差错控制（MAC）
 1 物理层   ── 比特流传输
```

---

### 📦 封装与解封装过程（主机1 → 主机2）
| 步骤 | 操作 | PDU名称 |
|------|------|---------|
| 应用层 | 加应用头 → 应用数据 | **报文（Message）** |
| 传输层 | 加传输头（TCP/UDP） | **段（Segment）/报文（Datagram）** |
| 网络层 | 加IP头 | **数据报（Datagram）/分组（Packet）** |
| 数据链路层 | 加帧头/尾（MAC地址） | **帧（Frame）** |
| 物理层 | 编码为比特流 | **比特（Bit）** |

→ 传输后**逐层解封装**（反向剥头尾）  
⚠️ **注意**：中间节点（如路由器）**仅处理到网络层**

---

### 📡 服务类型与原语

| 服务类型 | 特点 | 类比 |
|----------|------|------|
| **面向连接** | 连接建立 → 数据传输 → 连接释放<br>可靠、有序、流控（如TCP） | 电话 |
| **无连接** | 每个报文独立路由，自带完整地址<br>尽力而为（如IP、UDP） | 邮政 |

#### 常用**服务原语**
| 原语 | 含义 |
|------|------|
| `LISTEN` | 阻塞等待连接请求 |
| `CONNECT` | 主动发起连接 |
| `RECEIVE` | 等待接收报文 |
| `SEND` | 发送报文 |
| `DISCONNECT` | 释放连接 |

> 💡 **重要结论**（两军问题）：  
> 在不可靠信道上，**无法设计100%可靠的连接建立协议**（需无限确认）→ 实践中采用**超时+重传**近似可靠。

## 七、参考模型比较（1.6）

### 🆚 OSI vs TCP/IP

| 维度 | OSI参考模型 | TCP/IP模型 |
|------|-------------|-------------|
| 制定者 | ISO（国际标准） | IETF（事实标准） |
| 层次 | **7层**：<br>7应用 → 6表示 → 5会话 → 4传输 → 3网络 → 2链路 → 1物理 | **4层**：<br>应用 → 传输（TCP/UDP）→ 网际（IP）→ 网络接口 |
| 特点 | 理论完整；协议复杂；市场化失败 | 实用简洁；高度灵活；全球采用 |
| 各层功能对比 | 见下表 | 见下表 |

#### OSI七层功能速查表
| 层 | 功能 |
|----|------|
| 7 应用层 | 应用接口、同步控制、错误恢复（HTTP/FTP/SMTP） |
| 6 表示层 | 数据格式转换、加密/解密、压缩/恢复 |
| 5 会话层 | 会话管理、同步点、令牌管理 |
| 4 传输层 | 端到端可靠传输、差错/序号/流控（**关键层**） |
| 3 网络层 | 路由选择、拥塞控制、逻辑链路（IP） |
| 2 数据链路层 | 成帧、差错控制（CRC）、流量控制（MAC/PPP） |
| 1 物理层 | 比特传输、接口/电压/时序标准（RJ45/光纤） |

#### TCP/IP四层详解
| 层 | 协议/功能 |
|----|-----------|
| **应用层** | HTTP, FTP, SMTP, DNS, Telnet, SNMP…（融合OSI 5~7层） |
| **传输层** | **TCP**（可靠、面向连接）<br>**UDP**（不可靠、无连接） |
| **网际层** | **IP**（无连接，“尽力而为”）、ICMP、ARP、RARP |
| **网络接口层** | 兼容各类底层：Ethernet, Token Ring, X.25, ATM, 帧中继… |

> ✅ **TCP/IP优势**：  
> - 沙漏结构 → **IP统一上层应用 & 下层网络**  
> - 高兼容性 → 支持异构网络无缝互联  

```
         应用层（HTTP/FTP/DNS...）
               ↓
         传输层（TCP / UDP）
               ↓
         网际层（IP ← 核心！）
               ↓
 网络接口层：[Ethernet][ATM][PPP]...
```

#### 客户-服务器通信流程（TCP为例）
```
① 客户 → CONNECT → 服务器
② 服务器 → ACCEPT → 建立连接
③ 双方 SEND/RECEIVE 交换数据
④ DISCONNECT → 释放连接
```
→ 支持**一机多服务**（服务器1 & 服务器2可共存）
