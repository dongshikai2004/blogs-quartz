# 📘 第2章 物理层 — 系统笔记

## 一、物理层的本质任务
> **核心问题**：  
> 计算机产生的数据如何在**介质上（机械）**、以**什么方式（电气）**、**有序地（时序）** 传输到另一台计算机？

### 四大接口特性
| 特性       | 说明                       |
| -------- | ------------------------ |
| **机械特性** | 接口形状/尺寸、引线数目与排列、固定/锁定装置等 |
| **电气特性** | 各线上电压范围                  |
| **功能特性** | 某电平在某线上表示何种意义            |
| **过程特性** | 不同功能事件的出现顺序（时序）          |

## 二、2.1 数据通信基础

### 2.1.1 基本概念

- **数据**：信息的载体  
  - *模拟数据*：连续（语音、温度）  
  - *数字数据*：离散（文字、整数）
- **信息**：数据的内涵
- **信号**：数据的电磁表现  
  - *模拟信号*：连续变化  
  - *数字信号*：离散取值

#### 通信系统模型
```
信源 → 输入信息 → 输入数据 → 发送器 → 发送信号 → 传输系统 → 接收信号 → 接收器 → 输出数据 → 输出信息 → 信宿
```
> 示例：PC → 调制解调器 → 公用电话网（模拟）→ 调制解调器 → PC

#### 基带 vs 宽带信号
| 类型       | 定义            | 特点                |
| -------- | ------------- | ----------------- |
| **基带信号** | 用两种电压直接表示 0/1 | 占用全频带，不调制，适合短距局域网 |
| **宽带信号** | 基带信号经调制后频分复用  | 可多路复用，提高线路利用率     |

### 2.1.2 奈奎斯特定理 & 香农定理

#### 📏 性能指标

| 指标        | 公式/说明                         |
| --------- | ----------------------------- |
| **传输时延**  | = 数据块长度 / 信息传输速率（发送时间）        |
| **传播时延**  | = 信道长度 / 电磁波传播速率（≈2×10⁸ m/s）  |
| **总时延**   | = 传输时延 + 传播时延 + 处理时延 + 排队时延   |
| **时延带宽积** | = 传播时延 × 带宽（单位：bit）→ 衡量链路"容量" |
| **RTT**   | 往返时延：发送 → 接收 → 确认返回           |

#### 📡 信号劣化三大因素
- **时延（delay）**
- **衰减（attenuation）**
- **噪声（noise）**

#### 🔊 带宽（Bandwidth）
振幅不会明显减弱的 **频率范围** 称为带宽
- 频域：**Hz**（频率范围）  
- 时域等效：**bps**（数据率能力）

#### 📊 速率概念辨析
码元：一个离散信号（电压）状态或信号事件
码元传输的速率越高，或信号传输的距离越远，在信道的输出端的波形的失真就越严重

| 术语              | 英文                      | 单位       | 定义                           |
| --------------- | ----------------------- | -------- | ---------------------------- |
| **信息传输速率**  $S$ | Bit Rate                | bps      | 每秒传输的**比特数**                 |
| **码元传输速率**  $B$ | Baud Rate               | Baud（波特） | 每秒传输的**码元数**（离散信号状态数）        |
| **关系**          | $S = B \times \log_2 V$ | —        | $V$：码元可能取值个数（如 4 相位 = $V=4$） |

> ✅ 示例：3 bit/码元 → 18 bit 数据 → 6 码元 → 同样波特率下速率提高 3 倍。

#### 🧠 奈奎斯特准则（无噪声理想信道）
> 理想低通信道最高码元速率：
$$
B_{\max} = 2H \quad \text{（Baud）}
$$
- $H$：信道带宽（Hz）  
- **注意**：实际信道远低于此值（因失真/干扰）

#### 📈 香农公式（有噪声实际信道）
> 信道极限信息传输速率（无差错）：
$$
C = W \log_2\left(1 + \frac{S}{N}\right) \quad \text{（bps）}
$$
- $W$：带宽（Hz）  
- $S/N$：信噪比（功率比）
	- $S$ ：信道内所传信号的平均功率
	- $N$ ：信道内部的高斯噪声功率
- 信噪比（dB）：$10 \log_{10}(S/N)$  
  - $S/N=10$ → 10 dB；$S/N=1000$ → 30 dB

> ✅ 启示：  
> - 提升速率途径：增带宽、提信噪比、增每码元比特数  
> - 实际速率 << $C$

#### 🔁 两定理作用位置
- **奈氏**：限制**码元速率**（发送器→信道）  
- **香农**：限制**信息速率**（信道本质容量）

### 2.1.3 数据传输方式

| 类型      | 说明                   | 图示         |
| ------- | -------------------- | ---------- |
| **单工**  | 单向传输（如广播）            | A → B      |
| **半双工** | 可收可发，但**不能同时**（如对讲机） | A ⇄ B（交替）  |
| **全双工** | **同时**收发（如电话）        | A ↔ B（双通道） |

| 通信方式 | 原理 | 特点 |
|----------|------|------|
| **串行通信** | 比特**逐位**传输（1条线） | 距离远、成本低、速率可高（如USB/RS232） |
| **并行通信** | 多位**同时**传输（n条线） | 距离短、易干扰（时钟偏移）、成本高 |

### 2.1.4 调制解调技术（数字→模拟）

#### 📡 调制解调器（Modem）
- **调制器（MOdulator）**：数字 → 模拟（0/1 → 300~3400 Hz 音频信号）  
- **解调器（DEModulator）**：模拟 → 数字  
> 作用：波形变换器（调制） + 波形识别器（解调）

#### ⚙️ 基本调制方法（二元制）
| 方法 | 缩写 | 原理 |
|------|------|------|
| 调幅 | ASK | 振幅随数据变（0/1 对应不同振幅） |
| 调频 | FSK | 频率随数据变 |
| 调相 | PSK | 相位随数据变 |

#### 🌐 高级调制：QAM（正交振幅调制）
- 同时调**幅+调相**，形成星座图  
- 例：16-QAM → 16种状态 → **4 bit / 码元**  
- ⚠️ 码元状态越多 → 抗干扰越差 → 误码率↑

### 2.1.5 编码解码技术（基带传输）

> 基带传输 = 数字数据 → 数字信号 → 直接传输（无需载波）

#### 常用编码方式

| 编码 | 原理 | 优缺点 | 应用 |
|------|------|--------|------|
| **NRZ（不归零）** | 高电平=1，低电平=0 | ❌ 无自同步、有直流分量 | 早期设备 |
| **曼彻斯特码** | 位中跳变：<br>↑=0，↓=1 | ✅ 自同步、无直流<br>❌ 效率50%（信号频率=数据率×2） | 以太网（10BASE-T） |
| **差分曼彻斯特** | 位中必跳变（时钟）；<br>位首跳变=0，无跳变=1 | ✅ 时钟/数据分离更清晰 | 令牌环网 |

> 🎯 关键特性：  
> - 含时钟信息 → 实现**自同步**  
> - 无直流分量 → 减少损耗  
> - "违例码"（V0/V1）可用于帧起止标志

### 2.1.6 交换的概念

#### 交换的演进历史
1. **电报网与报文交换**
   - 定义：文本信息＋二进制编码＋数字传输＋报文交换
   - 关键技术：编址、路由、存储转发

2. **电话网与电路交换**
   - 基本问题：N部电话机两两直连需要N(N-1)/2对电线
   - 解决方案：引入交换机集中连接
   - 电路交换三阶段：
     - 建立连接
     - 通信
     - 释放连接
   - 特点：面向连接，独占物理通路
   - 弱点：对计算机数据（突发性）效率低，线路利用率低

3. **分组交换的出现**
   - 核心思想：将长报文分割为小分组，独立路由传输
   - 过程：
     - 发送端将报文分割为数据块
     - 每块加首部构成分组（包）
     - 依次发送各分组
     - 接收端去除首部，重组报文
   - 基于存储转发：路由器接收完整分组→存储→查路由表→转发
   - 优势：
     - 高效：动态分配带宽，逐段占用
     - 灵活：以分组为单位寻路
     - 迅速：无需预先建立连接
     - 可靠：完善的协议，自适应路由提高网络生存性
   - 代价：
     - 时延：存储转发带来排队延迟
     - 开销：分组首部增加额外信息

#### 分组时延的4种来源
1. **节点处理时延**：
   - 检查比特差错
   - 决定输出链路
   - 通常几微秒或更少
   
2. **排队时延**：
   - 等待输出链路传输的时间
   - 取决于路由器拥塞程度
   - 流量强度=La/R (L:分组长度,bits; a:到达速率,分组/秒; R:链路带宽,bps)
   - 当La/R→1时，排队时延急剧增加
   - 当La/R>1时，平均时延趋向无穷大

3. **传输时延**：
   - = L/R (将分组所有比特推送到链路上的时间)
   - 对低速链路影响显著

4. **传播时延**：
   - = d/s (d:物理距离; s:传播速度,~2×10⁸ m/sec)
   - 从几微秒到几百毫秒不等

> 🚗 **车队类比**：车辆=比特，车队=分组，收费站=路由器
> - 传输时延：通过收费站的时间
> - 传播时延：公路行驶时间
> - 关键洞察：在分组全部在第一个路由器传输完成前，该分组的第一个比特可能已到达第二个路由器

#### 分组丢失
- 路由器缓存有限
- 当分组到达满队列时被丢弃
- 丢失分组可能由源端或前一节点重传，也可能不重传

#### 三种交换方式的比较

| 特性       | 电路交换               | 报文交换     | 分组交换                  |
| -------- | ------------------ | -------- | --------------------- |
| **传输单位** | 整个报文连续传输           | 整个报文存储转发 | 分组(小块)存储转发            |
| **连接要求** | 预先建立连接             | 无需连接     | 数据报：无需连接<br>虚电路：需建立连接 |
| **资源分配** | 固定带宽               | 按需分配     | 动态分配                  |
| **时延特点** | 传输时延固定<br>建立连接时延大  | 存储转发时延大  | 时延较小且灵活               |
| **适用场景** | 连续大数据流<br>(如语音、视频) | 已基本淘汰    | 突发数据<br>(现代互联网)       |

##### 电路交换 vs 分组交换详细对比
| 项目 | 电路交换 | 分组交换 |
|------|----------|----------|
| 建立呼叫连接 | 要求 | 不要求 |
| 专用物理路径 | 是 | 否 |
| 每分组沿同样路径 | 是 | 否(数据报) |
| 分组按序到达 | 是 | 否(数据报) |
| 某交换机崩溃影响 | 严重 | 可绕过失效节点 |
| 可用带宽 | 固定 | 动态分配 |
| 拥塞发生时间 | 建立连接时 | 每个分组传送时 |
| 带宽浪费 | 可能严重 | 较少 |
| 存储转发机制 | 无 | 有 |
| 透明性 | 是(不关心内容) | 否(需处理分组) |
| 收费模式 | 按时间计费 | 按流量/分组计费 |

#### 分组交换网络的两种类型
1. **数据报网络**:
   - 无连接服务
   - 每个分组携带完整目的地址
   - 路由器基于目的地址独立选路
   - 同一对源/目的间的分组可能走不同路径
   - 路由器无需维护连接状态
   - 代表：IP网络

1. **虚电路网络** VC:
   - 面向连接服务
   - 三个阶段：建立呼叫→数据传输→拆除连接
   - 每个分组携带VC标识符(而非完整地址)
   - 路径在建立阶段确定，全程不变
   - 每台路由器为每条VC维护状态
   - 资源(带宽/缓存)可能预分配给VC
   - 代表：ATM、帧中继、X.25
   - 虚电路组成:
     - 源到目的的路径
     - 每条链路的VC号
     - 路由器转发表中的项(入接口+入VC# → 出接口+出VC#)

## 三、2.2 传输介质

### 3.1 导向传输介质（有线）

#### 电磁波频谱(电信领域)
- **低频(LF)**: 10⁴-10⁵ Hz
- **中频(MF)**: 10⁵-10⁶ Hz
- **高频(HF)**: 10⁶-10⁷ Hz
- **甚高频(VHF)**: 10⁷-10⁸ Hz
- **特高频(UHF)**: 10⁸-10⁹ Hz
- **超高频(SHF)**: 10⁹-10¹⁰ Hz (微波)
- **极高频(EHF)**: 10¹⁰-10¹¹ Hz
- **太赫兹(THF)**: 10¹¹-10¹² Hz
- **可见光**: 10¹⁴-10¹⁵ Hz
- **X射线/γ射线**: >10¹⁸ Hz

#### (1) 双绞线
- **结构**:
  - 两根绝缘铜线相互缠绕
  - 减少电磁干扰(EMI)
- **类型**:
  - **UTP (非屏蔽双绞线)**: 无屏蔽层，成本低
  - **STP (屏蔽双绞线)**: 有金属屏蔽层，抗干扰强
- **类别与应用**:

| 类别 | 带宽 | 典型应用 |
|------|------|----------|
| 3类 | 16MHz | 低速网络，模拟电话 |
| 4类 | 20MHz | 短距离10BASE-T以太网 |
| 5类 | 100MHz | 10/100BASE-T以太网 |
| 超5类(5e) | 100MHz | 100BASE-T，部分1000BASE-T |
| 6类 | 250MHz | 1000BASE-T吉比特以太网，ATM |
| 7类 | 600MHz | 10吉比特以太网 |
- **连接方式**:
  - **直连线**: 两端线序相同，用于连接端设备与网络设备(PC-HUB/Switch)
  - **交叉线**: 部分线序交叉(Tx↔Rx)，用于连接同类网络设备(Switch-Switch)
- **特点**:
  - 优点: 结构简单，安装容易，价格低廉
  - 缺点: 衰减较大，传输距离受限(通常≤100m)，易受干扰，有辐射可被窃听
  - 应用: 室内终端连接，星型/总线型网络

#### (2) 同轴电缆
- **结构**:
  - 内导体(铜芯)
  - 绝缘层
  - 外导体屏蔽层(编织网或铝箔)
  - 保护套
- **类型**:
  - **50Ω同轴电缆**: 用于数字传输(计算机网络)
  - **75Ω同轴电缆**: 用于模拟传输(电视/CATV)
- **特点**:
  - 优点: 频带较宽，传输率高，损耗低，传输距离远，辐射低，保密性好，抗干扰能力强
  - 缺点: 安装复杂，灵活性较差
  - 应用: 计算机联网(室内细缆，室外粗缆)，多路复用传输

#### (3) 光纤
- **工作原理**: 全反射
  - 纤芯: 高折射率材料
  - 包层: 低折射率材料
  - 光线在纤芯-包层界面发生全反射
- **类型**:
  - **多模光纤**:
    - 纤芯较粗(50-62.5μm)
    - 可传输多个模式的光
    - 光源: LED
    - 传输距离: ≤2km
    - 色散较大，带宽较低
  - **单模光纤**:
    - 纤芯很细(8-10μm)
    - 仅传输单一模式的光
    - 光源: 激光
    - 传输距离: >100km
    - 无模式色散，带宽极高
- **特点**:
  - 优点: 传输带宽极大(THz级)，衰减极小，不受电磁干扰，无串扰，保密性好
  - 缺点: 成本高，连接技术复杂，脆弱，需要光电转换
  - 应用: 长距离干线，数据中心，高带宽需求场景

### 3.2 无线传输介质
- **特点**:
  - 无需物理连接
  - 灵活移动性
  - 部署成本低
  - 受环境影响大
- **类型**:
  1. **短波通信**:
     - 依靠电离层反射
     - 通信质量较差
     - 应用: 远距离通信，应急通信
  
  2. **微波通信**:
     - 直线传播
     - 两种主要形式:
       - **地面微波接力通信**: 需中继站，受地形限制
       - **卫星通信**: 覆盖范围广，延迟高(≈270ms)，带宽有限
     - 应用: 长距离干线，偏远地区连接

## 四、2.3 物理层设备

### 4.1 中继器(Repeater)
- **功能**: 信号再生与放大
  - 接收衰减/失真信号
  - 放大并整形
  - 转发到下一段介质
- **工作原理**:
  - 物理层设备，只识别电压，不理解帧/分组概念
  - 无过滤/隔离功能，广播所有信号
- **应用限制**:
  - 不能形成环路
  - 以太网4中继器规则(5-4-3规则):
    - 最多5个网段
    - 最多4个中继器
    - 其中3个网段可连接终端
  - 举例: 粗同轴电缆(500m/段)，使用4个中继器可扩展至2500m
- **作用**: 扩展网络物理覆盖范围

### 4.2 集线器(Hub)
- **本质**: 多端口中继器
- **功能**:
  - 将多台计算机互连成星型拓扑
  - 收到信号后向所有端口广播(除来源端口)
- **类型(按结构)**:
  - **独立式(Stand alone)**:
    - 固定端口配置
    - 通过级联扩展
  - **堆叠式(Stackable)**:
    - 多个HUB可堆叠
    - 逻辑上视为单个设备
    - 可统一管理
  - **模块化(Module/机箱式)**:
    - 机箱+电源+插槽
    - 可插入不同功能卡(交换卡、路由卡、加密卡等)
    - 高度灵活，按需配置

- **类型(按电源)**:
  - **无源HUB**:
    - 仅汇聚线缆
    - 无信号放大
    - 传输距离仅达标准最大距离的一半
  - **有源HUB**:
    - 信号整形、放大、转发
    - 可扩展传输距离

- **类型(按智能程度)**:
  - **非智能(普通)HUB**:
    - 无管理功能
    - 低成本
    - 适合小型网络
  - **智能HUB**:
    - 内含CPU等控制部件
    - 支持SNMP等网管协议
    - 可监控流量、排查故障
    - 适合中大型网络

- **局限性**:
  - 共享带宽: 所有端口共享总带宽
  - 单一冲突域: CSMA/CD机制下效率随节点增加下降
  - 广播所有帧: 无过滤功能，安全性低
  - 已基本被交换机(Switch)取代
