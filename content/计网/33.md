### 3.1 数据链路层的基本功能

- **链路（Link）**：一条无源的点到点物理线路段，中间无交换节点。  
- **数据链路（Data Link）**：链路 + 数据链路层协议。
- **结点（Node）**：主机、交换机、路由器等。
- **点到点通信**：相邻结点之间一条链路上的通信。
- **端到端通信**：从源结点到目的结点的通信，可能经过多个链路。

#### 为网络层提供的服务（三类）：

1. **无确认的无连接服务**  
   - 不建立逻辑连接；帧独立发送，接收方不确认；  
   - 不检测/恢复帧丢失；适用于误码率低、实时性高的场景（如语音）；  
   - 多数局域网（LAN）使用该服务。

2. **有确认的无连接服务**  
   - 无连接，但每帧需单独确认；超时未确认则重传；  
   - 适用于高误码率的不可靠信道（如无线通信）。

3. **有确认的面向连接服务**  
   - 分三阶段：建立连接 → 传输帧 → 释放连接；  
   - 适用于高可靠性、强实时要求的系统。

---

### 3.2 组帧（Framing）

- **定义**：将原始比特流划分为若干“帧”（Frame），作为数据链路层的传输单位。
- **帧边界识别难点**：区分帧头、帧尾、数据内容。

#### 组帧方法：

1. **字符计数法**  
   - 帧首含字符计数字段；  
   - 缺点：计数错误将导致当前及后续帧解析错误。

2. **带字符填充的首尾字符定界法**  
   - 使用特殊字符（如 `SOH`、`EOT`）作为帧边界；  
   - 数据中若出现与定界相同的字符，需“字符填充”——在该字符前插入转义字符（如 `ESC`）；  
   - 接收方移除转义字符还原原数据。

3. **带位填充的首尾标志定界法（位填充法）**  
   - 起止标记为 `01111110`（称为 flag）；  
   - 发送方在数据部分中连续出现 5 个 `1` 后自动插入 1 个 `0`（**0 比特插入**）；  
   - 接收方检测到 5 个连续 `1` 后，删掉其后的 1 个 `0`（**0 比特删除**）；  
   - 保证 flag 字段不会在数据中误现。

4. **物理层编码违例法**  
   - 适用于具有冗余编码的物理层（如曼彻斯特编码）；  
   - 利用非法编码（如高低-高低连续电平）作为帧边界标志符（如 high-high, low-low）；  
   - 主要用于 IEEE 802 LAN（如 Token Ring, IEEE 802.5）。

### 3.3 差错检测和纠正

#### 差错类型
- **随机差错**：独立发生，前后码元无关联 → 主要由**热噪声**引起（幅度小）。
- **突发差错**：相邻多位连续出错 → 主要由**冲击噪声**引起（幅度大，主导差错原因）。

#### 差错评价指标
- **误码率（Pe）**：  
  $$
  P_e = \frac{N_e}{N}
  $$  
  错传码元数 / 总传输码元数。
- **误比特率（BER）**：  
  错传比特数 / 总传输比特数；对二进制码元，误码率＝误比特率。
- 典型线路误码率：
  - 普通电话线：$10^{-4} \sim 10^{-6}$
  - 多数通信系统：$10^{-5} \sim 10^{-9}$

#### 差错控制方式
| 方式 | 描述 | 适用场景 |
|------|------|----------|
| **反馈纠错**（ARQ） | 接收端检测出错 → 请求重发（无定位纠错能力） | 双工、非实时系统 |
| **前向纠错**（FEC） | 接收端能定位并纠正错误 | 单工、实时系统（如卫星广播） |
| **混合纠错**（HEC） | 小差错 FEC 自纠；超出能力 → ARQ 请求重发 | 高可靠性+中低时延需求 |

#### 冗余与检错/纠错原理
- 通过在原始数据后添加**冗余码**（监督码），使得接收方可根据**约定规则**判断是否出错，并（可能）纠正。
- **码字结构**：$n = m + r$（$m$ 数据位，$r$ 冗余位）
- **汉明距离（Hamming Distance）**：两码字间不同位数；  
  - 最小汉明距离为 $d_{\min}$：
    - 最多可**检错** $d_{\min} - 1$ 位；
    - 最多可**纠错** $\left\lfloor \frac{d_{\min} - 1}{2} \right\rfloor$ 位。

### 3.4 可靠传输机制

#### 3.4.1 完全理想化的单工传输
- 假设：
  1. 信道**无差错**；
  2. 接收方处理速率 ≥ 发送方发送速率（**无缓存溢出**）。
- 无需确认、无需流量控制、无需差错控制 → 仅理论模型。

#### 3.4.2 停-等协议（Stop-and-Wait, S&W）
- 放宽假设2（允许接收速率 < 发送速率），仍保持无差错信道。
- **核心机制**：
  - 发送一帧 → 等待确认 ACK → 收到 ACK 后发下一帧；
  - ACK 由接收方回送（含序号 $n$），实现**反向流量控制**。
- **帧编号**：单比特序号（0/1），满足 ARQ 需求。

#### 3.4.3 可靠的停-等协议（带差错恢复）
- 放宽所有理想假设 → 信道**可能出错或丢帧**。
- 需处理四种情况：
  1. 正常收发 + 正确 ACK；
  2. 数据帧**出错** → 接收方丢弃 → 发送端超时重传；
  3. 数据帧**丢失** → 接收方无动作 → 发送端超时重传；
  4. 确认帧 **ACK 丢失/出错** → 发送端超时重传 → 接收端需判重（靠序号）→ 丢弃重复帧，仍回 ACK。
- **重要细节**：
  - **不使用 NAK（否认帧）**：用超时 + 重传 + ACK 序号替代；
  - **定时器（Timer）**：每发一帧即启定时器，超时未收 ACK 即重传；
  - **序号作用**：防重（接收方判重丢弃）、防漏（ACK 序号暴露间隙）。

#### 停等协议性能分析
- **信道利用率**低：
  - 传输时延 $= L/b$（帧长/带宽），往返时延 $= R$
  - 利用率：
    $$
    U = \frac{L/b}{L/b + R} = \frac{L}{L + Rb}
    $$
  - 示例（卫星链路）：$L = 1000$ bit, $b = 50$ kbps, $R = 500$ ms → $U \approx 4\%$
- **优缺点**：
  - ✅ 简单；
  - ❌ 信道空闲时间长，带宽浪费严重 → 需更高效协议。

### 3.4.4 滑动窗口协议（Sliding Window Protocol）

- **背景动机**：停-等协议信道利用率低，尤其在**高带宽-时延积**（如卫星链路）场景下严重浪费资源。需支持**连续发送多帧再统一确认**（即流水线传输，Pipelining）。

#### 核心机制
- **发送窗口（$W_S$）**：发送方可连续发送但尚未被确认的最大帧数。
- **接收窗口（$W_R$）**：接收方可缓存并接收的、非连续但期望帧的范围。
- 每确认一个/批帧，窗口**向前滑动**；因此称为滑动窗口协议。

#### 全双工优化策略
- **捎带确认（Piggybacking）**：将 ACK 信息附在反向待发的数据帧中，减少单独控制帧开销。
  - 优点：提升信道利用率、减少中断。
  - 缺点：需缓存 ACK、协议逻辑复杂（如定时延迟发帧）。

#### 窗口滑动规则
- 发送端仅在**收到确认**后滑动发送窗口；
- 接收端在收到**正确序号帧**后滑动接收窗口，并发送确认；
- 若帧/确认丢失 → 超时重传或选择重传。

#### 特例
- 当 $W_S = W_R = 1$ 时 ⇨ 即停-等协议（单帧滑动窗口）。

### 3.4.5 后退N帧协议（Go-Back-N, GBN）

- **适用场景**：提高信道利用率，支持**累积确认**（Cumulative ACK）。
- **基本思想**：
  - 发送方可连续发送最多 $W_S$ 个帧（$W_S > 1$）；
  - 接收方仅按序接收（**接收窗口 $W_R = 1$**），乱序帧一律丢弃；
  - 若某帧出错/丢失 → 发送方超时后**重传该帧及其后已发所有帧**（即“后退N”）。

#### 举例说明
- 假设 $W_S = 4$，发送帧序号 0,1,2,3；
- 帧1在传输中出错 ⇒ 接收方丢弃帧1，之后收到帧2/3也丢弃；
- 发送方超时 ⇒ 重传帧1,2,3；
- ⇒ 信道误码率高时，大量无辜帧被重传 ⇒ 效率下降。

#### 性能分析
- 信道利用率提升：$U = \frac{W_S \cdot L/b}{L/b + R}$（理想无差错下），当 $W_S \ge 1 + 2a$（$a = Rb/(2L)$）可达 100%。
- 但差错时性能陡降。

---

### 3.4.6 选择重传协议（Selective Repeat, SR）

- **目标**：避免 GBN 中“无辜帧重传”，只重传出错/丢失帧。
- **核心机制**：
  - 发送窗口 $W_S > 1$，接收窗口 $W_R > 1$；
  - 接收方可**缓存乱序到达的正确帧**，仅请求重传缺失帧；
  - 使用**独立确认**（Individual ACK）或带位图的批量 ACK。

#### 关键问题：序号空间与窗口大小关系
- 若序号模 $2^n$（即 n 位序号），为避免新旧帧混淆，须满足：
  $$
  W_S + W_R \le 2^n
  $$
  通常取对称设计：  
  $$
  W_S = W_R \le 2^{n-1}
  $$

#### 示例说明
- 序号 0~3（即 $n=2$），若 $W_S = W_R = 3$：
  - 接收方收到 [0,1] 后窗口滑至 [2,3,0]；
  - 此时若再收到序号 0 帧 → 无法判断是新帧还是旧帧重传 ⇒ 协议失效。
- 故最大窗口为 $2^{n-1} = 2$（如 $W_S = W_R = 2$）安全。

#### SR vs GBN 对比
| 特性 | GBN | SR |
|------|-----|----|
| 接收窗口 | 1 | >1 |
| 乱序帧处理 | 丢弃 | 缓存 |
| 确认方式 | 累积 ACK | 单独 ACK |
| 重传粒度 | 从出错帧起全部重传 | 仅重传出错/丢失帧 |
| 实现复杂度 | 较低 | 较高（需缓存、乱序管理） |
| 适用场景 | 低误码率、简单设备 | 高误码率、高带宽时延积 |

### 可靠传输机制小结（对照表）

| 机制 | 作用 |
|------|------|
| **差错检测** | 检测帧内比特错误（如 CRC） |
| **定时器** | 检测帧/ACK丢失，触发重传；需防过早超时 |
| **序号** | 防止重传帧被误认为新帧（判重）、检测丢帧（序号空隙） |
| **确认（ACK）** | 接收方告知正确接收；可逐个/累积 |
| **否定确认（NAK）** | 告知某帧出错（*注：教材指出多数协议不显式使用*） |
| **滑动窗口/流水线** | 提升吞吐（允许多帧未确认），支持流量控制与拥塞控制 |

> 注：链路层可靠传输主要用于**高误码链路**（如无线）；有线局域网常依赖上层（如 TCP）提供可靠性。

### 3.5 介质访问控制（Medium Access Control, MAC）

#### 3.5.1 信道划分协议（Channel Partitioning Protocols）

- **目标**：将信道资源（时间、频率、码字等）**静态划分**给各用户，避免冲突，适用于**高负载场景**。
- **主要方式**：
  
| 方式 | 原理 | 特点 | 典型应用 |
|------|------|------|----------|
| **频分复用 FDM** | 将频带划分为若干子频带，每个用户固定占用一子频带 | 各路信号频域正交；需保护频带防串扰 | 有线电视、ADSL |
| **时分复用 TDM** | 时间划分为固定长度时隙，各用户轮流使用 | 时隙固定分配；空闲时隙仍保留，效率低 | SDH、SONET |
| **波分复用 WDM** | 光频段的FDM，不同波长承载不同信号 | 高容量；需光放大器（如EDFA） | 光纤骨干网 |
| **码分复用 CDMA** | 各用户使用正交码片序列扩频，共享同一频段 | 抗干扰强；需同步与正交码设计；支持软容量 | 3G（WCDMA/CDMA2000） |

- **CDMA原理补充**：
  - 数据比特 × 码片序列 → 扩频信号；
  - 接收端用相同码片序列做内积解扩；
  - 理想正交码下，多用户信号互不干扰。

---

#### 3.5.2 随机访问协议（Random Access Protocols）

- **核心思想**：允许站点**自由争用**信道；若冲突 → 退避 + 重试。
- **适用场景**：低负载时高效；高负载时冲突开销大。

##### 1. ALOHA 协议
- **纯ALOHA**：
  - 任意时刻可发帧；
  - 帧长为 $T$，冲突危险期为 $2T$（前后各 $T$）；
  - 最大吞吐率 ≈ 18.4%（$G = 0.5$ 时，$S = G e^{-2G}$）。
- **时隙ALOHA（Slotted ALOHA）**：
  - 时间划分为同步时隙，仅允许在时隙起点发送；
  - 危险期缩为 $T$；
  - 最大吞吐率 ≈ 36.8%（$G = 1$ 时，$S = G e^{-G}$）；
  - 实现需全局同步。

##### 2. 载波监听多路访问（CSMA）
- **载波监听（Carrier Sense）**：发前先侦听信道是否空闲；
- **分类**：
  - **1-坚持 CSMA**：空闲则立即发送；忙则持续监听 → 高利用率，但冲突概率高；
  - **非坚持 CSMA**：忙则随机等待再监听 → 减少冲突，但引入额外延迟；
  - **p-坚持 CSMA**（时隙化）：空闲时以概率 $p$ 发送，$1-p$ 推迟到下一时隙；折中方案。

##### 3. CSMA/CD（Collision Detection）
- **适用**：**总线型以太网**（半双工），如 10BASE5/2/T。
- **工作机制**：
  1. 监听信道，空闲 DIFS 后发送；
  2. 边发边监听；
  3. 若检测到冲突（信号畸变/电压异常）→ 立即**停止发送** + 发送 **48 bit 强化干扰信号（Jamming）**；
  4. 启动**二进制指数退避算法**重试。
- **关键参数**：
  - **争用期（Collision Window）** = $2\tau$（端到端往返传播时延）；
    - 例：1 km 电缆，$v = 2 \times 10^8$ m/s → $\tau = 5 \mu s$ → 争用期 = 10 μs；
  - **最短帧长** = $2\tau \times \text{带宽}$；
    - 10 Mbps × 51.2 μs = **512 bit = 64 byte**（含18字节首尾 → 数据≥46字节）；
    - <64字节为**无效帧**（冲突中止），直接丢弃。

##### 二进制指数退避算法（以太网）
- 重传次数 $k = \min(n, 10)$，其中 $n$ 为已重传次数；
- 退避时隙数 $r \in [0, 2^k - 1]$（随机整数）；
- 实际退避时间 = $r \times 51.2 \mu s$（10 Mbps下）；
- 重传 ≥16次仍失败 → 丢弃帧 + 上报高层。

#### 3.5.3 轮流协议（Taking-Turns Protocols）

- **动机**：兼顾信道划分（高负载公平）与随机访问（低载高效）优点。
- **两类**：

| 协议 | 机制 | 优点 | 缺点 |
|------|------|------|------|
| **轮询（Polling）** | 主节点按序询问从节点是否发送 | 无冲突；可控 | 单点故障；轮询开销；主节点瓶颈 |
| **令牌传递（Token Passing）** | 控制令牌沿环/逻辑环传递；持令牌者可发 | 公平；确定性时延 | 令牌丢失/重复风险；单点故障；环重建开销 |

- **典型应用**：  
  - Token Ring（IEEE 802.5）、FDDI（光纤分布式数据接口）。

### 3.6 局域网（Local Area Network, LAN）

#### 3.6.1 局域网的基本概念与体系结构

- **拓扑结构**：总线型、星型、环型、树型  
- **传输介质**：双绞线、同轴电缆、光纤、无线电波  
- **关键技术三要素**：
  1. 拓扑结构  
  2. 传输介质  
  3. 介质访问控制方式（如 CSMA/CD、Token Ring、Token Bus）

##### IEEE 802 参考模型
- 仅对应 OSI 模型的**物理层 + 数据链路层**；
- 数据链路层进一步划分为：
  - **LLC（Logical Link Control）子层**：向上提供统一接口，屏蔽不同 MAC 差异；
  - **MAC（Media Access Control）子层**：负责帧封装、地址识别、介质访问控制；
- **高层功能**（如流量控制、差错恢复）由网络操作系统（NOS）或上层协议（如 TCP）实现。

##### IEEE 802 系列标准（部分关键）
| 标准 | 内容 |
|------|------|
| 802.1 | 体系结构、寻址、互连、安全管理 |
| 802.2 | LLC 子层规范 |
| 802.3 | CSMA/CD（以太网） |
| 802.4 | 令牌总线（Token Bus） |
| 802.5 | 令牌环网（Token Ring） |
| 802.6 | 城域网（MAN） |
| 802.11 | 无线局域网（WLAN） |
| 802.15 | 无线个域网（WPAN，如蓝牙、ZigBee） |
| 802.16 | 宽带无线接入（WiMAX） |

---

#### 3.6.2 以太网（Ethernet）

##### 1. 传统以太网连接方式
- **传输媒体**：
  - 同轴粗缆（10BASE5）  
  - 同轴细缆（10BASE2）  
  - 双绞线（10BASE-T）  
  - 光纤（10BASE-F）
- **物理拓扑演进**：总线 → 星型（通过集线器/交换机）

##### 2. MAC 地址（硬件地址）
- 48 位全球唯一标识，固化于网卡（NIC）；
- 格式（高位在前）：
  ```
  I/G 位（第1位） | G/L 位（第2位） | OUI（前3字节） | 扩展标识符（后3字节）
  ```
  - I/G = 0：单播；=1：组播/广播  
  - G/L = 0：全球唯一（IEEE分配）；=1：本地管理  
- **多接口设备**（如路由器）有多个 MAC 地址。

##### 3. 以太网帧格式（两种主流）
| 项目 | IEEE 802.3 + 802.2 LLC | Ethernet II（DIX V2）——**主流** |
|------|--------------------------|----------------------------------|
| 前导码 | 7B + 1B SFD（同步） | 同左 |
| 目的地址 | 6B | 6B |
| 源地址 | 6B | 6B |
| 长度/类型 | 2B：<br>≤1500 表长度（LLC帧）<br>>1536（0x0600）表类型（如 0x0800=IP） | 2B：**类型字段**（直接标识上层协议） |
| 数据 | 43–1497B（LLC头+用户数据） | 46–1500B（直接用户数据） |
| FCS | 4B CRC-32 校验 | 同左 |
| **最小帧长** | **64B**（含18B头尾 ⇒ 数据≥46B） | 同左 |

- **填充字段**：数据 < 46B 时自动填充（使帧 ≥ 64B）；
- **无效帧判定**（任一即丢弃）：
  1. 长度非整数字节；  
  2. FCS 校验失败；  
  3. 数据长度 ∉ [46, 1500]；  
  4. 总长 ∉ [64, 1518]。

##### 4. 帧间最小间隔（Interframe Gap, IFG）
- **9.6 μs（96 bit）**：确保接收方有足够时间清理缓存，准备下一帧；
- 发送方检测到信道空闲后**仍需等待 IFG** 才能发帧。

##### 5. CSMA/CD 工作细节回顾
- **争用期 = 2τ（往返时延）**
  - 1 km 电缆，传播速度 $2 \times 10^8$ m/s ⇒ τ = 5 μs ⇒ 争用期 = 10 μs；
- **最短帧长 = 2τ × 带宽**
  - 10 Mbps ⇒ 512 bit = 64 B（含头尾）；
- **强化碰撞（Jamming）**：检测冲突后立即发送 48 bit 干扰信号，通知全网。

##### 6. 二进制指数退避算法（重申）
- 重传次数 $k = \min(n, 10)$（$n$ 为已重传次数）；
- 随机退避时隙数 $r \in [0, 2^k - 1]$；
- 实际退避时间 = $r \times 51.2\ \mu\text{s}$（10 Mbps 下）；
- **16 次重传失败 → 丢弃 + 上报高层**。

### 3.6.3 无线局域网（Wireless LAN, WLAN）

#### 无线网络基本要素
- **无线主机**：笔记本、PDA、IP电话等；可固定或移动；“无线” ≠ “移动”。
- **基站（Base Station）**：如 802.11 中的 **接入点 AP（Access Point）**，中继无线与有线网络。
- **无线链路**：连接移动终端与基站，或作为主干链路；支持不同速率与距离。
- **基础设施模式** vs **自组织模式（Ad Hoc）**：
  - 基础设施模式：含 AP，通过 DS（分配系统）互联形成 ESS（扩展服务集）；
  - 自组织模式：无中心节点，节点自主组网（如战场通信、灾后应急）。

#### 基本服务集（BSS）与扩展服务集（ESS）
- **BSS（Basic Service Set）**：一个 AP + 若干移动站；所有通信需经 AP。
- **ESS（Extended Service Set）**：多个 BSS 通过 **分配系统 DS（Distribution System）** 互联。
- **漫游（Roaming）**：移动站从一个 BSS 移动至另一 BSS 仍保持通信（需重关联）。
- **门户（Portal）**：连接 ESS 与非 802.11 网络（如 Internet），功能类似网桥。

#### 关联（Association）机制
站点加入 BSS 必须先与某 AP **建立关联**：
- **被动扫描**：监听 AP 周期性广播的 **信标帧（Beacon Frame）**（含 SSID、速率等）；
- **主动扫描**：站点主动发 **探测请求帧（Probe Request）** → AP 回 **探测响应帧（Probe Response）**。
- **SSID（Service Set Identifier）**：≤32 字节字符串，标识 BSS；默认广播，也可隐藏增强安全。

#### 热点（Hot Spot）与热区（Hot Zone）
- **热点**：提供公共 Wi-Fi 接入场所（机场、咖啡店等）；
- **热区**：多个热点覆盖区域；
- **WISP（Wireless ISP）**：无线互联网服务提供商，通过无线链路接入 Internet。

---

#### 移动自组网络（MANET, Mobile Ad Hoc Network）
- 无基础设施、节点平等、动态拓扑；
- 应用：军事通信、救灾、车联网；
- 与 **移动 IP 不同**：
  - 移动 IP 基于固定互联网路由，仅解决终端移动性；
  - MANET 为自主网络，需专用路由协议（如 AODV、DSR），可独立运行。

#### 无线传感器网络（WSN, Wireless Sensor Network）
- 大量微型传感器节点组成的自组织网络；
- 特征：低功耗、低带宽、存储受限、需自配置与安全；
- 应用：
  - 环境监测（洪水、动物追踪）；
  - 军事侦察；
  - 医疗监护；
  - 工业安全（矿井、核电站）；
  - 智能楼宇（温控、照明）。

---

#### 802.11 物理层标准对比

| 标准 | 频段 | 最高数据率 | 物理层技术 | 主要特点 |
|------|------|------------|-------------|---------|
| 802.11b | 2.4 GHz | 11 Mbps | DSSS | 价格低、穿墙强、易受干扰 |
| 802.11a | 5 GHz | 54 Mbps | OFDM | 高速率、干扰少、穿墙弱 |
| 802.11g | 2.4 GHz | 54 Mbps | OFDM | 兼容 b，性能优于 b |
| 802.11n | 2.4/5 GHz | 600 Mbps | MIMO + OFDM | 多天线、信道绑定（40 MHz） |

> 注：DSSS（直序扩频）、OFDM（正交频分复用）、MIMO（多入多出）

---

#### 802.11 MAC 层：CSMA/CA 协议

##### 为何不使用 CSMA/CD？
1. **无法边发边检**：无线设备半双工，发送时无法同时接收；
2. **隐藏站问题（Hidden Station Problem）**：
   - A 与 C 均在 B 覆盖范围内，但彼此不可见；
   - A 与 C 同时向 B 发送 → 在 B 处碰撞，但 A、C 均无法感知 → 需避免而非检测。

3. **暴露站问题（Exposed Station Problem）**：
   - B→A 通信时，C 检测到信道忙，误判不能向 D 发，其实可并发；
   - 过度保守降低吞吐。

##### CSMA/CA 核心机制
- **碰撞避免（Collision Avoidance）** + **停止等待重传**；
- **分布式协调功能 DCF**（默认，争用式）；
- **点协调功能 PCF**（可选，中心控制，无争用轮询）。

##### 帧间间隔 IFS（InterFrame Space）
按优先级排序：

| 类型 | 全称 | 时长 | 用途 |
|------|------|------|------|
| SIFS | Short IFS | 最短 | ACK、CTS、分片后续帧等高优先级控制帧 |
| PIFS | PCF IFS | SIFS + 1 时隙 | PCF 模式下 AP 竞争优先权 |
| DIFS | DCF IFS | PIFS + 1 时隙 = SIFS + 2 时隙 | 普通数据帧/管理帧发送前等待 |

> **时隙（Slot Time）**：最小竞争单元，由链路传播时延等决定。

##### 退避机制（Backoff）
- 站点侦听到信道由 **忙→空闲** 后，需：
  1. 等待 **DIFS**；
  2. 进入 **争用窗口（Contention Window）**，执行 **二进制指数退避**。
- 退避计时器倒计时单位为“时隙”；
- 若退避中再次检测到信道忙 → **冻结计时器**，待下次空闲后继续；
- **退避触发场景**：
  - 首帧发送前信道忙；
  - 重传；
  - 成功发送后（避免连续霸占信道）。

##### 二进制指数退避（802.11 版）
- 第 $i$ 次退避：时隙号 $r \in \{0, 1, ..., 2^{k} - 1\}$
  - $k = \min(i + 2, 10)$，即：
    - 第1次：$r \in [0, 7]$（8个选择）  
    - 第2次：$r \in [0, 15]$（16个）  
    - …  
    - 第 $i \ge 4$ 次：$r \in [0, 1023]$（1024个）
- 最大重传 7 次（默认，可配置）后丢弃帧。

##### 虚拟载波监听（Virtual Carrier Sense）
- 源站在帧 **“持续期（Duration）”** 字段声明本次传输所需总时间（含 ACK）；
- 其他站收到后更新 **NAV（Network Allocation Vector）**，即本地“信道忙”计时器；
- 即使物理未感知信号，也因 NAV > 0 而推迟发送 → **减少隐藏站冲突**。

##### RTS/CTS 握手机制（可选，防长帧浪费）
1. 源站 A → 发送 **RTS**（含目的地址、持续期）；
2. 目的站 B（若信道空闲）→ 回 **CTS**（复制持续期）；
3. A 收到 CTS 后 → 发数据；
4. 其他站收到 RTS/CTS → 更新 NAV，冻结退避；
- 适用于高误码或隐藏站严重环境；
- 小帧通常不开 RTS/CTS（开销 > 收益）。

---

#### 802.11 帧格式（数据帧）

```
| 帧控制 (2B) | 持续期 (2B) | 地址1 (6B) | 地址2 (6B) | 地址3 (6B) | 序号控制 (2B) | 地址4 (6B, 可选) | 帧主体 (0~2312B) | FCS (4B) |
```

- **帧控制字段细节**（16 bit）：
  - 协议版本（2b）、类型/子类型（6b）、
  - `To DS` / `From DS`（各1b）、更多分片、重传、
  - 电源管理、更多数据、WEP 加密、顺序（QoS）

- **地址字段解释**（取决于 DS 方向）：

| To DS | From DS | 地址1 | 地址2 | 地址3 | 说明 |
|-------|---------|--------|--------|--------|------|
| 0 | 0 | 目的地址 | 源地址 | BSSID | Ad Hoc 模式 |
| 0 | 1 | 目的地址 | BSSID | 源地址 | 站→AP（上行） |
| 1 | 0 | BSSID | 源地址 | 目的地址 | AP→站（下行） |
| 1 | 1 | 目的地址 | 源地址 | – | 无线桥接（地址4=收端AP） |

- **序号控制**：
  - 序号（12b）：每新帧+1，重传不变 → 防重；
  - 分片（4b）：同一帧的分片序号，不分片为0；
- **FCS**：32 位 CRC，检错用。

### 3.6.4 令牌环网（Token Ring, IEEE 802.5）

#### 基本特点
- 基于 IBM Token Ring 协议，由 IEEE 标准化为 **802.5**；
- 支持速率：1 Mbps、4 Mbps、16 Mbps；
- **不强制限定介质与拓扑**：可用同轴电缆、双绞线、光纤；物理结构常为星型，逻辑环型；
- **典型实现**：使用 IBM 指定双绞线 + 星型拓扑 + MAU（多站接入单元）形成逻辑环。

#### 网络组成
- **站点（Station）**：主机或终端；
- **干线耦合器（Trunk Coupling Unit, TCU）**：连接站点与环路；
  - 支持两种工作模式：
    - **收听模式**：TCU 与 DTE 断开，再生并转发比特流；监视地址/令牌；
    - **发送模式**：截获令牌后闭合开关 K，TCU 与 DTE 接通，插入数据帧。

#### 工作原理（令牌控制）
1. 环中**仅有一个令牌帧**循环；
2. 站点**仅当持有令牌时**方可发送；
3. 持有者将令牌帧头 **T=0 → T=1**，变更为**数据帧**；
4. 数据帧沿环转发 → 各站点检查目的地址：
   - 匹配：复制帧至本地缓存，**继续转发**；
   - 不匹配：直接转发；
5. 帧**绕环一周**回到**源站** → 源站：
   - 移除该帧；
   - 生成并插入**新空令牌**（T=0）；
   - 释放环使用权。

#### 帧结构
| 字段 | 1B | 1B | 可选 | ≥0B | 4B | 1B | 1B |
|------|----|----|------|------|-----|-----|-----|
| **令牌帧** | 起始定界符 | 访问控制 | 结束定界符 |  |  |  |  |
| **数据帧** | 起始 | 访问控制 | 帧控制 | 目的/源地址 | 数据 | FCS | 帧状态 | 结束 |

##### 访问控制字段（1B = 8bit）结构：
```
PPP | T | M | RRR
```
- `PPP`（3bit）：**优先级位**（0~7）；
- `T`（1bit）：**令牌位**，T=0 为令牌，T=1 为忙帧；
- `M`（1bit）：**监督位（Monitor）**：
  - 每站首次收到帧且 M=0 时置 1；
  - 若某站收到 M=1 的帧仍为 1 → 判定为**循环帧未回收**，由该站清除；
- `RRR`（3bit）：**预约位**，低优先级站可预约下次发送权（提高公平性）。

#### 介质访问控制功能
- **帧发送**：须持有空令牌；
- **令牌维护**：忙帧回收后必须释放空令牌；
- **优先级调度**：
  - 高优先级站点可覆盖低优先级预约；
  - 令牌中含优先级限制位（仅 ≥该优先级者可发送）；
- **公平性保障**：通过预约机制避免饿死。

#### 优缺点
| 优点 | 缺点 |
|------|------|
| ✅ 确定性延迟（适合实时）<br>✅ 公平访问（无冲突）<br>✅ 高负载下效率稳定 | ❌ 单点故障导致全环中断<br>❌ 令牌绕环带来固有延迟<br>❌ 实现复杂，成本高 |

> ⚠️ 注：因以太网快速发展，令牌环网已基本淘汰。

### 3.7 广域网（Wide Area Network, WAN）

#### 3.7.1 广域网基本概念
- **覆盖范围广**：几十 km → 跨洲；
- **分组交换为主**：虚电路（如 X.25、ATM）或数据报（如 IP）；
- **组成**：
  - **通信子网**：路由器、传输链路；
  - **资源子网**：主机、终端；
- **关键目标**：资源共享、远程通信。

---

#### 3.7.2 点对点协议 PPP（Point-to-Point Protocol）

##### 设计目标与需求
- **目标**：简单、通用、面向字节的点对点链路；
- **必须支持**：
  - 封装成帧、透明传输、多网络层协议（IP/IPX等）、差错检测、链路状态检测、MTU协商、地址协商；
- **明确不支持**：
  - ❌ 纠错（依赖上层）
  - ❌ 流量控制
  - ❌ 帧序号/确认（无可靠传输）
  - ❌ 多点/半双工链路

##### PPP 三组件
1. **链路封装方法**：将 IP 数据报封装入 PPP 帧；
2. **LCP（Link Control Protocol）**：
   - 建立/配置/测试数据链路；
   - 协商参数（认证方式、MTU、魔术字等）；
3. **NCP（Network Control Protocol）**：
   - 为不同网络层协议配置（如 IPCP 分配 IP 地址）。

##### PPP 帧格式
```
| 7E | FF | 03 | 协议(2B) | 信息 ≤1500B | FCS(2/4B) | 7E |
   F    A    C
```
- `F = 0x7E`：帧定界符；
- `A = 0xFF`：广播地址（无意义，保留）；
- `C = 0x03`：无编号信息（UI）；
- `协议字段`：
  - `0x0021`：IP 数据报；
  - `0xC021`：LCP 帧；
  - `0x8021`：IPCP 帧。

##### 透明传输实现
| 链路类型 | 方法 |
|----------|------|
| **异步（如拨号）** | **字符填充**：<br>• `0x7E` → `0x7D 0x5E`<br>• `0x7D` → `0x7D 0x5D`<br>• 控制字符（<0x20）→ `0x7D (X⊕0x20)` |
| **同步（如 SDH）** | **零比特填充**：<br>• 发送端：遇 `5` 个连续 `1` 插入 `1` 个 `0`<br>• 接收端：遇 `5` 个 `1` 后删掉下 `1` 个 `0` |

##### 工作状态机（RFC 1661）
1. **链路静止**（无物理连接）  
   ↓ 建立物理层（如拨号成功）  
2. **链路建立**（LCP 协商）  
   ↓ 成功 →  
3. **鉴别**（可选，PAP/CHAP）  
   ↓ 成功/跳过 →  
4. **网络层协议**（NCP 配置，如 IPCP 分配 IP）  
   ↓ 成功 →  
5. **链路打开**（数据传输）  
   ↓ 挂断/超时/故障 →  
6. **链路终止** → 回到静止

> 📌 典型应用场景：拨号上网、DSL、专线接入。

### 3.7.3 HDLC 协议（High-level Data Link Control）

#### 起源与标准关系
- HDLC 最早由 **IBM** 提出，称为 **SDLC**（Synchronous Data Link Control）；
- 后由 **ISO** 修改定为国际标准 **ISO 3309**；
- CCITT（现 ITU-T）将其修改为 **LAP**（Link Access Procedure），后进一步发展为 **LAPB**（Link Access Procedure, Balanced），"B" 表示**平衡型**（支持对等站点间双向通信）。

#### HDLC 的主要特点
- 面向**比特**的同步链路控制协议；
- 支持**全双工/半双工**，**点对点/多点**配置；
- 提供**可靠传输**（可选），具有帧序号、确认、重传机制；
- 支持三种操作模式：

| 模式 | 描述 | 特点 |
|------|------|------|
| **NRM（Normal Response Mode）** | 主从式，仅主站可发起传输 | 适用于主站轮询从站 |
| **ARM（Asynchronous Response Mode）** | 主从式，从站可自发发帧 | 较少使用 |
| **ABM（Asynchronous Balanced Mode）** | 平衡式，双方地位平等，均可发起 | 广泛采用（如 LAPB 就是 ABM） |

#### HDLC 帧结构（5字段）
```
| Flag (1B) | Address (1B) | Control (1~2B) | Information (可变) | FCS (2/4B) | Flag (1B) |
```
- **Flag**：`01111110`（0x7E），帧起止标志；使用**零比特填充**实现透明传输；
- **Address**：1 字节，用于多点链路中寻址（点对点时无意义）；
- **Control**：决定帧类型与序号，分三类帧：

| 帧类型 | 控制字段首位 | 作用 |
|--------|--------------|------|
| **I 帧（Information）** | `0` | 携带用户数据 + 序号/确认；可捎带确认（P/F 位） |
| **S 帧（Supervisory）** | `10` | 仅用于流量/差错控制（RR/RNR/REJ/SREJ） |
| **U 帧（Unnumbered）** | `11` | 无编号，用于链路管理（如 SNRM、DISC、UA、FRMR） |

##### Control 字段详解（以 1 字节为例）

| 字段 | 位数 | 说明 |
|------|------|------|
| **I 帧** | `0 P/F N(S) N(R)` | 共 8 位：<br>• `N(S)`：发送序号（3bit）<br>• `N(R)`：确认序号（3bit）<br>• `P/F`：Poll/Final 位（轮询/终止标志） |
| **S 帧** | `1 0 TYPE P/F N(R)` | TYPE=2bit：<br>• `00`=RR（接收就绪）<br>• `01`=RNR（拒收）<br>• `10`=REJ（拒收+重传全部）<br>• `11`=SREJ（选择拒收） |
| **U 帧** | `1 1 M1 M2 M3 M4 P/F` | M1~M4：操作码（如 SNRM=12, DISC=04, UA=06, FRMR=08, UI=03） |

> 注：若需扩展序号到 7bit（长帧格式），Control 字段扩展为 2 字节。

#### HDLC 与 PPP 对比
| 特性 | HDLC | PPP |
|------|------|-----|
| 面向 | 比特 | 字节 |
| 帧定界 | Flag + 零比特填充 | Flag + 字符填充 / 零比特填充 |
| 地址 | 显式 Address 字段 | 固定 `0xFF`（无实际作用） |
| 控制 | 复杂（I/S/U 帧） | 固定 `0x03`（UI 帧） |
| 可靠性 | 可选（ABM 模式支持确认重传） | 无（依赖上层） |
| 多协议支持 | 否 | 是（Protocol 字段） |
| 链路协商 | 否 | 是（LCP/NCP） |
| 应用场景 | 专线、X.25、早期广域网 | 拨号、DSL、现代点对点链路 |

> ✅ HDLC 更适合**可靠信道上的专用链路**；PPP 更适合**灵活、异构、需协商的接入链路**。

### 3.8 数据链路层设备

#### 3.8.1 网桥（Bridge）

##### 功能与原理
- 工作在**数据链路层（MAC 子层）**；
- 根据**目的 MAC 地址**进行帧转发/过滤；
- 实现**局域网互联**，隔离**冲突域**（Collision Domain），但**不隔离广播域**；
- 每个端口对应一个独立的冲突域。

##### 透明网桥（Transparent Bridge, IEEE 802.1D）
- **“透明”**：主机不知其存在；
- **“即插即用”**：自学习构建转发表；
- **转发表（Forwarding Table）字段**：
  - **MAC 地址**（源地址学习）
  - **端口号**（进入端口）
  - **老化时间（Age）**（通常 5~300 秒，超时删除）

##### 逆向学习 + 扩散算法（工作流程）
1. 收到帧 → 检查 FCS → 错误则丢弃；
2. **学习**：将**源 MAC + 入端口**写入/更新转发表（计时器重置）；
3. **转发/过滤决策**：
   - 若**目的 MAC 在表中**：
     - 出端口 = 入端口 → **丢弃**（本网段通信）；
     - 否则 → **仅从对应端口转发**；
   - 若**目的 MAC 不在表中** → **泛洪（Flooding）**（除入端口外所有端口）；
4. 循环。

##### 生成树协议（Spanning Tree Protocol, STP, IEEE 802.1D）
- **动机**：多网桥互联形成环路 → 广播帧无限循环（广播风暴）；
- **核心思想**：构造无环的**生成树**（Spanning Tree）：
  1. 各网桥广播 BPDU（Bridge Protocol Data Unit），含网桥 ID（优先级 + MAC）；
  2. **ID 最小者**为**根网桥**（Root Bridge）；
  3. 各非根网桥选择**到根路径成本最低的端口**为**根端口**（Root Port）；
  4. 每个 LAN 选择**到根路径成本最低的网桥端口**为**指定端口**（Designated Port）；
  5. 其余端口置于**阻塞状态（Blocking）**；
- **结果**：逻辑上仍是星型或树型，无环；
- 支持**拓扑更新**（Topology Change Notification, TCN）。

##### 源选路网桥（Source Routing Bridge）
- 由**源站负责路由选择**；
- 源站先发**探测帧（Explorer Frame）** 沿所有可能路径到达目的；返回时带回路径；
- 源站选最优路径，将**路由清单（RIF, Routing Information Field）** 插入帧首部；
- 网桥按 RIF 转发（无学习、无泛洪）；
- 优点：最优路径；缺点：主机负担重、帧开销大；
- 用于 IBM Token Ring 网络（IEEE 802.5）。

##### 网桥使用限制
- ❌ 不解决广播风暴（广播帧仍全网泛洪）；
- ❌ 存储-转发引入时延；
- ❌ 仅适用于中小规模网络（>几百节点性能下降）；
- ❌ 不同 MAC 子层互联时延更大（如以太网 ↔ Token Ring 需协议转换）。

### 3.8.2 交换机（Switch）

#### 交换机基本概念与特点
- **本质**：多端口网桥（通常 ≥12 端口），工作于**数据链路层（MAC 层）**；
- **与集线器对比**：

| 项目 | 集线器（Hub） | 交换机（Switch） |
|------|----------------|------------------|
| 工作层级 | 物理层 | 数据链路层 |
| 转发方式 | 广播式（所有端口） | 查表转发（仅目的端口） |
| 冲突域 | 1 个全局冲突域 | 每端口独立冲突域 |
| 广播域 | 1 个广播域 | 1 个广播域（若无 VLAN） |
| 通信模式 | 半双工 | 全双工（主流） |
| 带宽共享 | 共享带宽（N 用户 → 10/N Mbps） | 独占带宽（每对通信享 10/100/1000 Mbps） |

- **核心功能**：
  ✅ 基于 MAC 地址过滤与转发  
  ✅ 自学习构建 MAC 转发表  
  ✅ 支持全双工通信 + 无碰撞传输  
  ✅ 支持 VLAN 划分与管理  
  ✅ 支持链路聚合、生成树协议等

#### 工作原理（与透明网桥同源）
1. **自学习**：
   - 收到帧 → 读取**源 MAC + 入端口** → 写入/更新转发表（设老化计时器）；
2. **转发/过滤/泛洪**：
   - **目的 MAC 在表内**且出端口 ≠ 入端口 → 单播转发；  
   - 目的 MAC 在表内且出端口 = 入端口 → 丢弃（本段通信）；  
   - 目的 MAC 不在表内 / 为广播（FF:FF:FF:FF:FF:FF） / 多播 → **泛洪**（除入端口外全发）；
3. **循环抑制**：
   - 默认启用 **STP（Spanning Tree Protocol, IEEE 802.1D）** → 消除环路；
   - 支持 RSTP（802.1w）、MSTP（802.1s）加速收敛。

#### 转发模式（按处理时序分）
| 模式 | 描述 | 优缺点 |
|------|------|--------|
| **存储-转发（Store-and-Forward）** | 收完整帧 + FCS 校验正确后才转发 | ✅ 检错能力强<br>❌ 时延大（尤其长帧） |
| **直通（Cut-Through）** | 收到目的地址（前 6+6+2=14B）即查表转发 | ✅ 时延极小（≈0.5 μs）<br>❌ 无法检错，坏帧也转发 |
| **无碎片直通（Fragment-Free）** | 先收前 64B（最小帧长），确认非冲突碎片后再转发 | ✅ 避免小碎片传播<br>❌ 仍无法检长帧错 |

> ⚠️ 实际交换机常支持**动态切换**：高误码环境用存储-转发；低延时场景用直通。

#### 交换机 vs 网桥对比
| 特性 | 网桥 | 交换机 |
|------|------|--------|
| 端口数 | 少（2~4） | 多（8~48+） |
| 实现方式 | 软件/通用 CPU | 专用 ASIC 芯片 |
| 吞吐能力 | 低（≈线速 10 Mbps） | 高（线速 1/10/100 Gbps） |
| 功能扩展 | 基本桥接 | VLAN、QoS、ACL、镜像、堆叠等 |
| 应用层级 | 已淘汰/边缘旧网 | 局域网核心接入设备 |

#### 虚拟局域网（VLAN, IEEE 802.1Q）

##### 定义与目的
- VLAN 是**逻辑子网**，跨越物理位置，按功能/部门/安全策略划分；
- 解决传统 LAN 两大痛点：
  - **广播风暴**：广播帧仅在所属 VLAN 内泛洪；
  - **安全隔离**：不同 VLAN 主机无法二层互通（除非经路由器）。

##### VLAN 实现方式
- **基于端口**（最常用）：交换机端口静态绑定 VLAN ID（VID）；
- **基于 MAC 地址**：按源 MAC 划分，移动性强，但配置复杂；
- **基于协议 / IP 子网**：高层划分，需三层交换支持；
- **基于策略**（用户+IP+MAC 组合）：企业级安全需求。

##### 802.1Q 帧格式扩展
在标准以太网帧中插入 **4 字节 VLAN Tag**（位于源地址与类型字段之间）：
```
| 目的MAC (6) | 源MAC (6) | TPID=0x8100 (2) | TCI (2) | 类型/长度 (2) | 数据 | FCS |
```
- **TPID**（Tag Protocol Identifier）：固定 `0x8100`，标识 VLAN 帧；
- **TCI**（Tag Control Information）：
  - **PRI（3b）**：用户优先级（0~7，用于 QoS）；
  - **CFI（1b）**：规范格式指示（0=以太网，1=令牌环兼容）；
  - **VID（12b）**：VLAN ID（范围 1~4094；0/4095 保留）。

##### Trunk 与 Access 端口
| 端口类型 | 描述 | 帧处理规则 |
|----------|------|-------------|
| **Access** | 连终端（PC/服务器/打印机） | 进：无 Tag → 打本端口 VID<br>出：同 VID → 去 Tag；不同 VID → 丢弃 |
| **Trunk** | 交换机间互联 | 进：有 Tag → 按 VID 学习；无 Tag → 打 **PVID（默认 VLAN）**<br>出：本 VLAN 帧保留 Tag；PVID 帧可选去 Tag |

> 📌 默认所有端口属于 VLAN 1；PVID = Port VLAN ID（端口缺省 VID）。

##### VLAN 间通信
- **二层交换机**：仅支持 **同一 VLAN 通信**；
- **三层交换机 / 路由器**：启用 **SVI（Switch Virtual Interface）** 或子接口 → 实现跨 VLAN 路由（即 “单臂路由”）。

### 3.8.3 交换机高级特性补充

#### 链路聚合（Link Aggregation, IEEE 802.3ad / 802.1AX）
- 又称 **EtherChannel**（Cisco）、**Port Trunking**；
- 将多个物理端口捆绑为一个**逻辑链路（LAG, Link Aggregation Group）**；
- **目的**：
  - ✅ 提升带宽（如 4×1 Gbps → 4 Gbps）；
  - ✅ 冗余容错（部分链路故障仍可通信）；
  - ✅ 负载均衡（基于源/目的 MAC、IP、端口等哈希）；
- **要求**：
  - 成员端口速率、双工模式一致；
  - 两端配置匹配（静态 or LACP 动态协商）；
- **LACP（Link Aggregation Control Protocol）**：
  - IEEE 802.3ad 定义；
  - 通过 LACPDU（LACP Data Unit）协商聚合组；
  - 支持主/备端口（Active/Standby）。

---

#### 端口安全（Port Security）
- **目的**：防 MAC 地址泛洪攻击、仿冒主机接入；
- **机制**：
  - 限制端口学习的 MAC 地址数量（默认 1）；
  - 可静态绑定 MAC（sticky MAC）；
  - 违规动作：`protect`（丢包不告警）、`restrict`（丢包+告警）、`shutdown`（端口 error-disable）；
- **适用端口**：仅 Access 端口；Trunk 端口需特殊配置。

---

#### 镜像端口（Port Mirroring / SPAN）
- 将指定端口/ VLAN 的流量复制到监控端口（Sniffer/IDS 接入）；
- 类型：
  - **本地 SPAN**：源与目的在同一交换机；
  - **远程 SPAN（RSPAN）**：跨交换机，通过专用 VLAN 传输镜像流；
- **应用**：网络排错、安全审计、性能监控。

---

#### 三层交换机（Layer 3 Switch）
- 兼具**交换 + 路由**功能；
- **转发机制**：
  - 首包走 CPU（查路由表），生成 **MLS（Multilayer Switching）表项**；
  - 后续包硬件 ASIC 直接交换（一次路由，多次交换）；
- **典型功能**：
  - SVI（Switch Virtual Interface）：为 VLAN 配 IP，实现 VLAN 间路由；
  - 支持动态路由协议（OSPF、RIP 等）；
  - ACL、QoS 策略硬件加速。

### 本章易错/高频考点提醒

1. **帧最小长度 = 2τ × 带宽** → 以太网 64 字节（数据 ≥ 46 字节）；
2. **CSMA/CD 不适用于无线** → 因无法边发边检 + 隐藏站问题；
3. **GBN 接收窗口 WR = 1**，SR 中 **WR > 1**；
4. **SR 最大窗口 Ws = Wr = 2ⁿ⁻¹**（n 为序号位数），否则新旧帧混淆；
5. **PPP 不提供可靠传输**（无序号/确认），差错仅检不纠；
6. **MAC 地址 I/G 位 = 0 为单播，=1 为组播/广播**；
7. **VLAN Tag 插在源地址与类型之间，TPID = 0x8100**；
8. **透明桥泛洪条件**：目的 MAC 不在转发表中，或为广播/多播地址；
9. **STP 阻塞端口不转发用户数据，但可收 BPDU**；
10. **802.11 使用 DCF（CSMA/CA）为主，PCF（轮询）可选**。
