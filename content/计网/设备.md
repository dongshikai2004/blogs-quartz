### 集线器(Hub)
- **本质**: 多端口中继器
- **功能**:
  - 将多台计算机互连成星型拓扑
  - 收到信号后向所有端口广播(除来源端口)
- **类型(按结构)**:
  - **独立式(Stand alone)**:
    - 固定端口配置
    - 通过级联扩展
  - **堆叠式(Stackable)**:
    - 多个HUB可堆叠
    - 逻辑上视为单个设备
    - 可统一管理
  - **模块化(Module/机箱式)**:
    - 机箱+电源+插槽
    - 可插入不同功能卡(交换卡、路由卡、加密卡等)
    - 高度灵活，按需配置

- **类型(按电源)**:
  - **无源HUB**:
    - 仅汇聚线缆
    - 无信号放大
    - 传输距离仅达标准最大距离的一半
  - **有源HUB**:
    - 信号整形、放大、转发
    - 可扩展传输距离

- **类型(按智能程度)**:
  - **非智能(普通)HUB**:
    - 无管理功能
    - 低成本
    - 适合小型网络
  - **智能HUB**:
    - 内含CPU等控制部件
    - 支持SNMP等网管协议
    - 可监控流量、排查故障
    - 适合中大型网络

- **局限性**:
  - 共享带宽: 所有端口共享总带宽
  - 单一冲突域: CSMA/CD机制下效率随节点增加下降
  - 广播所有帧: 无过滤功能，安全性低
  - 已基本被交换机(Switch)取代

### 3.8 数据链路层设备

#### 3.8.1 网桥（Bridge）

##### 功能与原理
- 工作在**数据链路层（MAC 子层）**；
- 根据**目的 MAC 地址**进行帧转发/过滤；
- 实现**局域网互联**，隔离**冲突域**（Collision Domain），但**不隔离广播域**；
- 每个端口对应一个独立的冲突域。

##### 透明网桥（Transparent Bridge, IEEE 802.1D）
- **“透明”**：主机不知其存在；
- **“即插即用”**：自学习构建转发表；
- **转发表（Forwarding Table）字段**：
  - **MAC 地址**（源地址学习）
  - **端口号**（进入端口）
  - **老化时间（Age）**（通常 5~300 秒，超时删除）

##### 逆向学习 + 扩散算法（工作流程）
1. 收到帧 → 检查 FCS → 错误则丢弃；
2. **学习**：将**源 MAC + 入端口**写入/更新转发表（计时器重置）；
3. **转发/过滤决策**：
   - 若**目的 MAC 在表中**：
     - 出端口 = 入端口 → **丢弃**（本网段通信）；
     - 否则 → **仅从对应端口转发**；
   - 若**目的 MAC 不在表中** → **泛洪（Flooding）**（除入端口外所有端口）；
4. 循环。

##### 生成树协议（Spanning Tree Protocol, STP, IEEE 802.1D）
- **动机**：多网桥互联形成环路 → 广播帧无限循环（广播风暴）；
- **核心思想**：构造无环的**生成树**（Spanning Tree）：
  1. 各网桥广播 BPDU（Bridge Protocol Data Unit），含网桥 ID（优先级 + MAC）；
  2. **ID 最小者**为**根网桥**（Root Bridge）；
  3. 各非根网桥选择**到根路径成本最低的端口**为**根端口**（Root Port）；
  4. 每个 LAN 选择**到根路径成本最低的网桥端口**为**指定端口**（Designated Port）；
  5. 其余端口置于**阻塞状态（Blocking）**；
- **结果**：逻辑上仍是星型或树型，无环；
- 支持**拓扑更新**（Topology Change Notification, TCN）。

##### 源选路网桥（Source Routing Bridge）
- 由**源站负责路由选择**；
- 源站先发**探测帧（Explorer Frame）** 沿所有可能路径到达目的；返回时带回路径；
- 源站选最优路径，将**路由清单（RIF, Routing Information Field）** 插入帧首部；
- 网桥按 RIF 转发（无学习、无泛洪）；
- 优点：最优路径；缺点：主机负担重、帧开销大；
- 用于 IBM Token Ring 网络（IEEE 802.5）。

##### 网桥使用限制
- ❌ 不解决广播风暴（广播帧仍全网泛洪）；
- ❌ 存储-转发引入时延；
- ❌ 仅适用于中小规模网络（>几百节点性能下降）；
- ❌ 不同 MAC 子层互联时延更大（如以太网 ↔ Token Ring 需协议转换）。

### 3.8.2 交换机（Switch）

#### 交换机基本概念与特点
- **本质**：多端口网桥（通常 ≥12 端口），工作于**数据链路层（MAC 层）**；
- **与集线器对比**：

| 项目 | 集线器（Hub） | 交换机（Switch） |
|------|----------------|------------------|
| 工作层级 | 物理层 | 数据链路层 |
| 转发方式 | 广播式（所有端口） | 查表转发（仅目的端口） |
| 冲突域 | 1 个全局冲突域 | 每端口独立冲突域 |
| 广播域 | 1 个广播域 | 1 个广播域（若无 VLAN） |
| 通信模式 | 半双工 | 全双工（主流） |
| 带宽共享 | 共享带宽（N 用户 → 10/N Mbps） | 独占带宽（每对通信享 10/100/1000 Mbps） |

- **核心功能**：
  ✅ 基于 MAC 地址过滤与转发  
  ✅ 自学习构建 MAC 转发表  
  ✅ 支持全双工通信 + 无碰撞传输  
  ✅ 支持 VLAN 划分与管理  
  ✅ 支持链路聚合、生成树协议等

#### 工作原理（与透明网桥同源）
1. **自学习**：
   - 收到帧 → 读取**源 MAC + 入端口** → 写入/更新转发表（设老化计时器）；
2. **转发/过滤/泛洪**：
   - **目的 MAC 在表内**且出端口 ≠ 入端口 → 单播转发；  
   - 目的 MAC 在表内且出端口 = 入端口 → 丢弃（本段通信）；  
   - 目的 MAC 不在表内 / 为广播（FF:FF:FF:FF:FF:FF） / 多播 → **泛洪**（除入端口外全发）；
3. **循环抑制**：
   - 默认启用 **STP（Spanning Tree Protocol, IEEE 802.1D）** → 消除环路；
   - 支持 RSTP（802.1w）、MSTP（802.1s）加速收敛。

#### 转发模式（按处理时序分）
| 模式 | 描述 | 优缺点 |
|------|------|--------|
| **存储-转发（Store-and-Forward）** | 收完整帧 + FCS 校验正确后才转发 | ✅ 检错能力强<br>❌ 时延大（尤其长帧） |
| **直通（Cut-Through）** | 收到目的地址（前 6+6+2=14B）即查表转发 | ✅ 时延极小（≈0.5 μs）<br>❌ 无法检错，坏帧也转发 |
| **无碎片直通（Fragment-Free）** | 先收前 64B（最小帧长），确认非冲突碎片后再转发 | ✅ 避免小碎片传播<br>❌ 仍无法检长帧错 |

> ⚠️ 实际交换机常支持**动态切换**：高误码环境用存储-转发；低延时场景用直通。

#### 交换机 vs 网桥对比
| 特性 | 网桥 | 交换机 |
|------|------|--------|
| 端口数 | 少（2~4） | 多（8~48+） |
| 实现方式 | 软件/通用 CPU | 专用 ASIC 芯片 |
| 吞吐能力 | 低（≈线速 10 Mbps） | 高（线速 1/10/100 Gbps） |
| 功能扩展 | 基本桥接 | VLAN、QoS、ACL、镜像、堆叠等 |
| 应用层级 | 已淘汰/边缘旧网 | 局域网核心接入设备 |

### 路由器体系结构
- **两个关键功能**：
  1. 运行路由算法/协议(RIP, OSPF, BGP)
  2. 从入链路到出链路转发数据报

- **输入端口功能**：
![[Pasted image 20260109171951.png]]
  - 分散式交换：使用转发表查找输出端口
	  - 目标：以"线速"完成输入端口处理
	  - 队列：当数据报到达比交换结构转发速率快时，需要排队
  - 物理层：比特级接收
  - 数据链路层：帧处理（如以太网）
  - 网络层：IP处理

- **三种交换结构**：
  1. **经内存交换**：
     - 第一代路由器设计
     - CPU直接控制，分组拷贝到系统内存
     - 速率受内存带宽限制（每数据报跨越两次总线）

  2. **经总线交换**：
     - 数据报从输入端口到输出端口经共享总线
     - 总线竞争限制交换速率
     - 1Gbps总线适用于接入和企业路由器
![[Pasted image 20260109172114.png]]
  3. **经互联网络交换**：
     - 克服总线带宽限制
     - Banyan网络或其他互联网络
     - 高级设计：将数据报分段为固定长度信元
     - Cisco 12000：通过互联网络实现Gbps交换

- **输出端口功能**：
  - 缓存：当数据报到达速率超过传输速率时
  - 调度：在排队数据报之间选择传输顺序
  - 排队问题：
    - 输出端口排队：因输出缓存溢出，导致时延和丢包
    - 输入端口排队：交换结构比输入端口组合速率慢时发生
	    - 线头(HOL)阻塞：队列前面的数据报阻塞后面数据报

- **路由表优化**：
  - 传统路由表：目的地址范围→链路接口
  - 基于网络前缀的路由表：
    - 网络前缀→链路接口(下一跳)
    - 支持默认路由
  - 高效查找技术：二叉线索等，支持快速路由查找

### 网络层设备比较
- **中继器(repeater)**：物理层中继系统，扩展网络物理范围
- **网桥(bridge)**：数据链路层中继系统，连接同类型局域网
- **路由器(router)**：网络层中继系统，连接不同类型网络，实现互连
- **网关(gateway)**：网络层以上中继系统，复杂，目前较少使用

- **重要说明**：
  - 中继器或网桥连接的网络仍视为单一网络
  - 互联网都是指用路由器互连的网络
  - 路由器是实现虚拟互连网络的关键设备
