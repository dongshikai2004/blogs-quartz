## 1. 进程概念回顾与深入

### 1.1 为什么引入进程？
*   **背景**：多道程序设计思想的引入。
*   **需求**：同一时间段内有多个程序并发执行，需要一个抽象实体来描述每一个运行的程序。
*   **定义**：进程是**程序的一次执行过程**，是一个**动态实体**。
*   **本质**：在操作系统中，进程通过**数据结构体**来表达。

### 1.2 进程控制块 (PCB)
*   **定义**：Process Control Block，操作系统用于管理进程的核心数据结构。
*   **存在性**：进程是否存在，取决于其对应的 PCB 是否存在。
    *   创建进程 $\rightarrow$ 创建 PCB。
    *   撤销进程 $\rightarrow$ 回收 PCB。
*   **包含信息**：
    *   进程管理信息（PID、创建者、优先级等）。
    *   进程映像信息（代码区、数据区、栈区等内存地址）。
    *   现场保护信息（寄存器、程序计数器 PC、栈指针 SP 等）。
*   **内存分布**：
    *   **PCB**：通常存放在操作系统内核空间（系统区）。
    *   **进程映像**：存放在用户空间（代码、数据、栈）。

## 2. 进程的状态与转换

### 2.1 三种核心状态
1.  **就绪态 (Ready)**：具备运行条件，等待 CPU 调度。
2.  **运行态 (Running)**：进程正在 CPU 上执行。
3.  **阻塞态 (Blocked/Waiting)**：等待某事件（如 I/O 完成、资源到位），无法运行。

### 2.2 状态转换逻辑
*   **就绪 $\rightarrow$ 运行**：进程被调度程序选中，获得 CPU。
*   **运行 $\rightarrow$ 就绪**：
    *   时间片用完（分时系统）。
    *   被更高优先级进程抢占（剥夺式调度）。
*   **运行 $\rightarrow$ 阻塞**：请求资源未果、发起 I/O 操作、等待其他进程。
*   **阻塞 $\rightarrow$ 就绪**：等待的事件完成（如 I/O 中断），**不能直接变为运行态**。

### 2.3 五状态与七状态模型
*   **五状态**：新增 **新建态 (New)** 和 **结束态 (Terminated)**。
    *   新建 $\rightarrow$ 就绪。
    *   运行 $\rightarrow$ 结束。
*   **七状态**：引入 **挂起态 (Suspended)**。
    *   **原因**：内存空间有限，将暂时不运行的进程从内存换出到外存。
    *   **分类**：就绪挂起、阻塞挂起。
    *   **操作**：挂起原语（内存 $\rightarrow$ 外存），激活原语（外存 $\rightarrow$ 内存）。

### 2.4 Linux 中的进程状态
Linux 在原理基础上进行了具体实现优化：
*   `TASK_RUNNING`：就绪/运行态。
*   `TASK_STOPPED`：暂停态（如调试断点）。
*   `TASK_ZOMBIE`：将死态（进程结束但资源未完全释放，父进程未回收）。
*   `TASK_INTERRUPTIBLE`：浅度睡眠（可被信号唤醒的阻塞态）。
*   `TASK_UNINTERRUPTIBLE`：深度睡眠（不可被信号打断的阻塞态）。

> **注意**：不同操作系统（Windows, Unix, Linux）状态定义略有不同，课程重点在于掌握**通用原理**。

## 3. 进程的组织与管理

### 3.1 PCB 的组织方式
操作系统需管理众多 PCB，常用数据结构包括：
1.  **线性表**：按创建顺序排列。
    *   *缺点*：调度时需遍历查找，效率低。
2.  **链表/队列**（推荐）：
    *   按**状态**分类：就绪队列、阻塞队列（可细分为不同阻塞原因）。
    *   按**优先级**分类：不同优先级对应不同队列。
    *   *优点*：调度时直接从对应队列队头取进程，效率高。
3.  **索引表**：通过索引快速定位 PCB。

### 3.2 当前运行进程指针
*   系统维护一个指针（如 `current`），指向当前正在 CPU 上运行的进程的 PCB。

## 4. 进程控制 (Process Control)

### 4.1 进程控制原语
*   **定义**：由若干条指令组成的程序段，用于完成特定功能。
*   **特性**：**原子性**（Atomic）。要么全部执行，要么全部不执行，执行过程中**不可被中断**。
*   **执行者**：操作系统内核。
*   **分类**：
    1.  **创建原语**：`fork()` (Unix/Linux), `CreateProcess()` (Windows)。
    2.  **撤销原语**：`exit()`。
    3.  **阻塞原语**：进程主动放弃 CPU（如 `sleep()`）。
    4.  **唤醒原语**：由中断处理程序调用，将阻塞进程变为就绪。
    5.  **挂起/激活原语**：内存与外存之间的换出/换入。

### 4.2 进程创建流程 (`fork` 示例)
1.  申请空白 PCB。
2.  分配进程映像空间（代码、数据、栈）。
3.  初始化 PCB 信息（PID、优先级、内存地址等）。
    *   **Copy-on-Write (浅拷贝)**：子进程初始时共享父进程的代码和数据，仅在修改时复制。
4.  将 PCB 插入就绪队列。

**代码逻辑示例**：
```c
pid_t pid = fork();
if (pid < 0) {
    // 创建失败
} else if (pid == 0) {
    // 子进程代码 (PID 为 0)
} else {
    // 父进程代码 (PID 为子进程 ID)
    wait(); // 等待子进程结束
}
```
*   **并发性**：`fork` 之后，父子进程**并行执行**。
*   **进程树**：进程间存在父子继承关系，可用树结构描述（Root 进程为所有进程祖先）。

### 4.3 进程撤销流程
*   **触发时机**：正常结束、异常退出（越界、除零）、外界干预（任务管理器杀进程）。
*   **操作**：
    1.  根据 PID 找到 PCB。
    2.  若正在运行，终止执行，收回 CPU。
    3.  处理子孙进程（终止或移交）。
    4.  回收资源（内存、文件等）。
    5.  移除 PCB。

## 5. 进程切换与调度

### 5.1 两种切换概念
1.  **进程模式切换**：用户态 $\leftrightarrow$ 核心态（发生系统调用时）。
2.  **进程切换 (Context Switch)**：CPU 从执行进程 A 转为执行进程 B。
    *   **步骤**：保存 A 的现场（寄存器、PC 等）到 PCB $\rightarrow$ 调度 B $\rightarrow$ 恢复 B 的现场。
    *   **开销**：频繁切换会降低系统效率。

### 5.2 进程调度 (Process Scheduling)
*   **定义**：从就绪队列中按照一定算法选择一个进程占用 CPU。
*   **调度级别**：
    *   **高级调度**：作业调度（外存 $\rightarrow$ 内存）。
    *   **中级调度**：内存调度（挂起/激活）。
    *   **低级调度**：**进程调度**（内存 $\rightarrow$ CPU），本课程重点。
*   **调度方式**：
    *   **非剥夺式 (Non-preemptive)**：进程主动放弃 CPU（如运行结束、请求 I/O）。适用于批处理。
    *   **剥夺式 (Preemptive)**：高优先级进程可抢占低优先级进程的 CPU。适用于分时、实时系统。

### 5.3 调度时机
1.  进程运行结束。
2.  进程请求 I/O 进入阻塞。
3.  时间片用完。
4.  高优先级进程到达（剥夺式）。
5.  中断发生（如 I/O 完成唤醒进程）。

### 5.4 调度算法评价指标
*   **CPU 利用率**：越高越好。
*   **吞吐量**：单位时间完成作业数，越多越好。
*   **周转时间**：作业提交到完成的时间差（越短越好）。
    *   $平均周转时间 = \frac{\sum 周转时间}{作业数}$
*   **等待时间**：进程在就绪队列等待的时间。
*   **响应时间**：从提交请求到首次产生响应的时间（分时系统关键）。

### 5.5 常见调度算法

| 算法 | 全称 | 思想 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **FCFS** | 先来先服务 | 按到达顺序排队 | 简单、公平 | 对短作业不利，平均等待时间长 | 批处理 |
| **SPF/SJF** | 短进程/作业优先 | 优先执行运行时间最短的 | **平均等待时间最短** (理论最优) | 对长作业不利（饥饿），运行时间难预估 | 理论基准 |
| **优先级调度** | Priority Scheduling | 优先执行优先级高的 | 灵活，可区分任务重要性 | 低优先级可能饥饿 | 实时系统 |
| **RR** | 时间片轮转 | 按时间片轮流执行 | 公平，响应快 | 时间片过小导致频繁切换，开销大 | 分时系统 |
| **MLFQ** | 多级反馈队列 | 多级优先级 + 时间片 + 动态调整 | 集众家之长，通用性强 | 实现复杂 | 通用操作系统 (Linux/Windows) |

*   **MLFQ 核心思想**：
    *   设置多个优先级队列。
    *   新进程进入高优先级队列。
    *   若时间片用完未结束，降级到下一队列。
    *   若因 I/O 阻塞，唤醒后保持或提升优先级（I/O 密集型进程优先）。
    *   防止饥饿：长时间等待的进程提升优先级。

## 6. 线程 (Thread)

### 6.1 为什么引入线程？
*   **进程局限性**：进程是资源分配单位，拥有独立地址空间，切换开销大，并行粒度粗。
*   **线程优势**：
    *   **细粒度并行**：一个进程内多个线程可并行执行（利用多核 CPU）。
    *   **资源共享**：同一进程内的线程共享内存、文件等资源。
    *   **切换开销小**：线程切换只需保存少量寄存器信息，无需切换页表等。

### 6.2 进程与线程的关系
*   **进程 (Process)**：**资源分配的基本单位**（拥有内存、文件句柄等）。
*   **线程 (Thread)**：**CPU 调度的基本单位**（拥有 TCB、栈、寄存器）。
*   **包含关系**：一个进程包含至少一个线程（主线程），可包含多个线程。
*   **Linux 实现**：Linux 内核不严格区分进程和线程，统一视为 `task`，通过 `task_struct` 管理，共享资源程度不同。

### 6.3 线程控制块 (TCB)
*   类似于 PCB，但信息更少（主要保存运行现场）。
*   线程共享进程的全局变量、代码段、打开的文件等。

## 💡 助教学习建议 (TA Tips)
1.  **重点记忆**：进程的**三状态转换图**是考试高频点，务必画得出来。
2.  **算法计算**：掌握 FCFS、SPF、RR 的**平均周转时间**计算方法，通常会给出一个表格让你计算。
3.  **概念辨析**：
    *   **并发 vs 并行**：并发是宏观同时微观交替，并行是真正的同时（多核）。
    *   **进程切换 vs 模式切换**：前者是换人（A 进程到 B 进程），后者是换身份（用户态到内核态）。
    *   **阻塞 vs 挂起**：阻塞是等事件（在内存），挂起是换出内存（到外存）。
4.  **代码理解**：理解 `fork()` 的返回值在父子进程中的不同，这是理解并发编程的基础。
