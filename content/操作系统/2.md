## 一、操作系统发展历史概述

操作系统的发展大致经历了以下四个主要阶段：

1.  **单道批处理时代** (50 年代)
2.  **多道批处理、分时、实时系统时代** (60 年代初)
3.  **多方式系统时代** (60-70 年代)
4.  **分布式、网络系统、多机系统时代** (70 年代后)

## 二、史前年代（40 年代 - 50 年代中期）：人工操作方式

### 1. 背景
计算机刚问世时，除硬件和应用软件外，几乎没有中间层软件，**没有操作系统**。

### 2. 操作流程
用户（程序员）直接使用计算机硬件系统：
1.  将事先已穿孔（对应于程序和数据）的纸带或卡片装入输入机。
2.  启动输入机将程序和数据输入计算机。
3.  启动计算机运行。
4.  程序运行完毕并取走结果后，才让下一个用户上机。

### 3. 主要缺点
*   **用户独占全机**：资源利用率极低。
*   **CPU 等待人工操作**：高速 CPU 与低速 I/O 速度不匹配，造成 CPU 空闲浪费。

## 三、单道批处理时代（50 年代）

### 1. 监督程序 (Monitor Program)
*   **基本思想**：系统有一常驻内存的监督程序。操作员有选择地把若干作业合成一批，安装在输入设备上，启动监督程序，然后由监督程序依次顺序启动这批作业，以**单道方式**运行。
*   **引入概念**：
    *   **作业 (Job)**：用户的一次上机行为。
    *   **作业步**：作业的执行步骤。
    *   **作业控制语言 (JCL)**：用于说明作业如何执行的语言。
    *   **作业控制说明书**：用作业控制语言编写的作业执行程序。
    *   **典型卡片作业举例**：
        *   `$JOB, JOBNAME=...` (作业开始)
        *   `$LOAD` (加载程序)
        *   `$FORTRAN` (编译)
        *   `$RUN` (运行)
        *   `$END` (作业结束)

### 2. 单道批处理系统处理过程
1.  **开始**
2.  **输入作业**：把下一个作业的源程序转换为目标程序。
3.  **检查错误**：源程序有错吗？
    *   是 -> 处理错误。
    *   否 -> **装配目标程序**。
4.  **运行**：运行目标程序。
5.  **判断**：还有下一个作业？
    *   是 -> 返回输入作业步骤。
    *   否 -> **停止**。

### 3. 系统特征
*   **自动性**：作业自动连续处理。
*   **顺序性**：作业依次顺序执行。
*   **单道性**：内存中仅保留一道作业。
*   *注：严格地说，它只能算作是 OS 的前身而并非是现在人们所理解的 OS。*

### 4. 关键技术机制 (易考点)
*   **普通指令**：用户程序、监督程序都可包含的指令。
*   **特权指令**：只有监督程序才可包含的指令（如 I/O 操作、内存管理）。
*   **方式/态 (Mode/State)**：
    *   **特权方式 (态)**：能执行普通指令 + 特权指令。
    *   **用户方式（态）**：只能执行普通指令。
*   **系统调用（广义指令）**：由监督程序提供给用户调用的系统功能。
*   **TRAP 机制**：硬件支持在特定状态下 (如执行 trap 指令) 进入监督程序的机制，用于实现系统调用。
*   **存储空间划分**：
    *   **系统空间**：监督程序所占内存 (0-K)。
    *   **用户空间**：供用户程序使用的内存 (K+1-N)。
*   **存储保护**：硬件提供一个界地址寄存器，每次访问内存时，处理机依照目前处理机的态判断访问是否越界。
*   **定时器中断**：在规定时间间隔到时，系统暂停当前程序之运行，去进行时间相关的处理（如看作业是否已超过其运行期限）。

### 5. 监督程序工作流程
1.  判输入设备上是否有待输入作业，没有则等待。
2.  从设备上输入一道作业。
3.  控制作业运行：
    *   a. 取作业说明书中一条语句，解析执行。如果是“作业终止”语句，则删除该作业，转步骤 1。
    *   b. 如果是一条执行性语句，则在主存中建立相应程序运行环境，传递好参数，转去执行该程序 (**变到用户态运行**)。
    *   c. 在用户态程序运行过程中，如发生中断事件 (如系统调用/定时器中断), 立即转监督程序，待中断事件处理结束后，用户态程序继续执行。
    *   d. 用户态程序结束后，转 a 取下一作业控制语句。
*   **监督程序功能**：输入用户作业，解释执行作业控制说明书，提供系统调用。
*   **与操作系统的区别**：监督程序**不具并发机制**。监督程序只是省去了作业步和作业转接的人工开销。

### 6. 输入/输出方式改进
*   **联机输入/输出 (On-Line I/O)**：慢速外设的 I/O 由主机直接控制，造成主机浪费。
*   **脱机输入/输出 (Off-Line I/O)**：
    *   **思想**：利用卫星机将慢速介质转到快速介质，加快作业在输入输出时的速度。
    *   **优点**：
        1.  减少了 CPU 的空闲时间。
        2.  提高 I/O 速度。

## 四、多道批处理、分时、实时系统时代（60 年代初）

### 1. 多道批处理系统 (Multiprogramming Batch Processing System)
*   **来源**：源于中断、通道、磁盘的引入。
*   **特点**：在单道批处理系统的基础上引入缓存机制，**SPOOLing 技术**, **多道程序设计技术**。
*   **输入输出缓冲**：利用缓冲技术，使针对一个缓冲区的数据处理和针对另一个缓冲区的 I/O 并行运行。
*   **SPOOLing 技术 (Simultaneous Peripheral Operation On-Line)**：
    *   **基础**：
        *   **I/O 中断**：由 I/O 部件操作完成后通知处理机去运行相应的处理程序。
        *   **通道**：能够自主控制外设与主存/磁盘并行交换数据的 I/O 装置 (控制器)。
        *   **磁盘**：随机存取的存储设备。
    *   **基本思想**：用磁盘设备作为主机的直接输入输出设备。主机从磁盘选取作业运行，结果也存在磁盘中。相应的通道则负责磁盘与慢速外设的传输。利用通道和中断机制，作业的输入输出与作业在处理机上的运行可很好并行起来。
    *   **系统结构**：
        *   **输入井**：磁盘上存储待输入作业信息的区域。
        *   **输出井**：磁盘上存储作业结果信息的区域。
        *   **预输入程序**：负责将输入设备数据送入输入井。
        *   **缓输出程序**：负责将输出井数据送入输出设备。
        *   **井管理**：管理输入/输出井。
        *   **虚拟设备**：通过 SPOOLing 技术将独占设备改造为共享设备。
*   **多道程序设计**：
    *   **基本思想**：内存同时保持多道程序 (作业)，处理机 (单处理机) 以交替的方式同时处理多道程序。
    *   **宏观上看**：已有多道程序开始运行且尚未结束。
    *   **微观上看**：某一时刻处理机只运行某道作业。
    *   **现场保护/恢复**：指处理机从一个作业向另一个作业切换时进行的有关寄存器值的保护/恢复。
*   **引入操作系统的基本特征**：
    *   **并发 (Concurrency)**：在单处理机情况下指多道程序分时占用处理机运行。
    *   **共享 (Sharing)**：指并发运行程序对系统的硬软资源的访问。
*   **优缺点**：
    *   (+) 资源利用率高。
    *   (+) 系统吞吐量大。
    *   (-) 平均周转时间长。
    *   (-) 无交互能力。

### 2. 分时系统 (Time-Sharing System)
*   **终端设备**：键盘与显示器。
*   **交互式系统**：操作系统提供终端命令语言及解析器，用户通过终端命令指导作业运行。
*   **定义**：交互式系统与多道程序设计结合的产物。
*   **特点**：
    1.  **并发性**
    2.  **共享性**
    3.  **交互性** (用户可直接控制程序运行)
    4.  **独立性** (用户感觉独占计算机)

### 3. 实时系统 (Real-Time System)
*   **定义**：能满足要求快速响应作业运行的多道程序设计系统。
*   **应用场景**：
    *   过程控制：生产控制、军事控制、医疗控制等（有具体的控制对象）。
    *   实时信息处理：航班订票、银行管理系统等。
*   **主要性能指标和功能**：
    *   **时钟分辨度**：有更高的时钟中断频度，可更精确计时，更快进行处理机调度。
    *   **多级中断机制**：保证要求快速响应中断的及时处理。
    *   **支持实时作业调度**：保证实时作业无条件剥夺非实时作业运行。

### 4. 多方式系统时代 (60-70 年代)
*   **定义**：同时具有批处理、分时、实时功能的系统。

## 五、并行与分布式操作系统及发展（70 年代后）

### 1. 系统分类
*   **分布式操作系统**：支持网络和分布处理的系统。
*   **多机操作系统**：支持共享内存多处理机的系统。
*   **Windows 操作系统**：支持友好用户界面的系统。
*   **嵌入式操作系统**：伴随 PAD、掌上电脑、机顶盒、智能家电等设备发展，对功能和空间大小权衡提出新要求。
*   **强实时操作系统**：针对通用操作系统的实时性研究，不断发展中。

### 2. 操作系统发展趋势
*   **并行操作系统的研究**：
    *   基于共享主存的多处理机系统 (MP)。
    *   基于分布式存储的大规模并行处理机系统 (MPP)。
    *   **特征**：提供各类并行机制（并行文件系统，并行 I／O 控制，多处理机分配和调度，处理机间的通讯和同步，用户任务的并行控制等）。
*   **网络操作系统和分布式操作系统的研究**：
    *   基于 Client／Server 模型的分布式系统已不断走向应用。
    *   完全分布式的系统还未成型，仍是研究热点。
*   **主要操作系统家族**：
    *   **Windows 系列**：DOS, Windows 3.x, 95, 98, Me, NT, 2000, XP, Vista, 7, 10...
    *   **UNIX 大家庭**：SVR4, BSD, Solaris, AIX, HP_UX。
    *   **自由软件**：Linux、FreeBSD、openEuler 等。

## 七、知识点总结与易考点提示 (Exam Focus)

### 1. 核心概念辨析
*   **并发 (Concurrency) vs 并行 (Parallelism)**：
    *   单道批处理：**无并发**。
    *   多道批处理/分时/实时：**并发** (宏观同时，微观交替)。
    *   多机系统：**并行** (真正的同时执行)。
*   **特权指令 vs 普通指令**：
    *   考点：哪些指令是特权指令？(通常涉及硬件资源管理、I/O、中断控制的指令)。用户态下执行特权指令会触发异常/中断。
*   **系统调用 (System Call)**：
    *   实现机制：**TRAP 指令/中断机制**。
    *   作用：用户态请求内核态服务。
*   **SPOOLing 技术**：
    *   考点：解决了什么问题？(高速 CPU 与低速 I/O 矛盾，独占设备共享化)。
    *   组成：输入井、输出井、预输入程序、缓输出程序。
    *   效果：实现了**虚拟设备**功能。

### 2. 各阶段系统特征对比 (必背)
| 系统类型 | 主要特征 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **人工操作** | 用户独占、手工干预 | 无 | CPU 等待人工、效率极低 |
| **单道批处理** | 自动性、顺序性、单道性 | 减少人工干预 | CPU 与 I/O 串行，资源利用率仍低 |
| **多道批处理** | 多道性、无序性、调度性 | 资源利用率高、吞吐量大 | 平均周转时间长、**无交互能力** |
| **分时系统** | 并发、共享、**交互**、独立 | 用户友好、响应快 | 系统开销大 |
| **实时系统** | 即时性、高可靠性 | 响应快速、硬实时约束 | 资源利用率相对较低 |

### 3. 技术演进逻辑
*   **人工 -> 单道批处理**：引入**监督程序**，解决人工操作慢的问题。
*   **单道 -> 多道批处理**：引入**中断、通道、磁盘、SPOOLing**，解决 CPU 与 I/O 速度不匹配问题，实现并发。
*   **批处理 -> 分时**：引入**终端、时间片轮转**，解决用户交互需求。
*   **通用 -> 实时**：引入**高优先级调度、多级中断**，解决特定任务快速响应需求。

### 4. 常见考题方向
1.  **判断题**：单道批处理系统是否具备并发特征？(否)
2.  **选择题**：SPOOLing 技术将独占设备改造为什么设备？(共享设备/虚拟设备)
3.  **简答题**：简述多道程序设计的基本思想及宏观/微观表现。
4.  **简答题**：特权指令是什么？为什么用户程序不能执行？
5.  **论述题**：操作系统发展的主要动力是什么？(不断提高计算机资源利用率、方便用户、器件不断更新换代、计算机体系结构不断发展)。
