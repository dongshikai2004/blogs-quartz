### 二、 文件访问系统调用

#### 1. 访问接口类型
*   **传统访问接口**：通过系统调用实现，包括文件创建、删除、打开、关闭、读、写等。
*   **Memory-Mapped（内存映射）文件访问**：另一种文件访问接口。

#### 2. 文件的建立与删除
*   **参数**：文件名（路径名）、设备名（卷名）、其它信息（需填入FCB的信息）。
*   **Create（创建）系统调用处理过程**：
    1.  检查参数合法性。
    2.  在文件目录结构的适当位置建立一个文件控制块（FCB）。
    3.  将参数填入FCB。
    4.  分配文件存放的外存空间（也可采用Lazy分配，即写数据时再分配），建立索引表并填入FCB中。
*   **Delete（删除）系统调用处理过程**：
    1.  检查参数，获取文件名（路径名）。
    2.  按名查找文件目录结构，找到文件的FCB。
    3.  根据FCB中的索引表，释放文件所占用的外存空间。
    4.  从文件目录结构中删除该FCB。

#### 3. 文件的打开与关闭
*   **Open（打开）系统调用处理过程**：
    1.  检查参数，获得文件名（路径名）。
    2.  按名查找文件目录结构，找到文件的FCB。
    3.  将FCB拷贝入内存，存入**“内存活跃文件目录表”**中。
    4.  建立一张本次打开的**“文件读写状态信息表”**，将当前访问指针指向文件首。
    5.  将“文件读写状态信息表”的地址存入进程PCB资源描述区的**“打开文件表”**中。
    6.  返回其索引号 `fd` 作为本次调用的返回值。
    *   *关键数据结构关系*：PCB -> 打开文件表 -> 文件读写状态信息表 -> 活跃文件目录表 -> FCB。
*   **Close（关闭）系统调用管理过程**：
    1.  检查参数，获得 `fd`。
    2.  按 `fd` 在PCB的“打开文件表”中找到“文件读写状态信息表”的指针，释放其空间。
    3.  如果“活跃文件目录表”中的FCB不再被其他进程使用，则释放该FCB所占的内存空间。

#### 4. 文件的读写
*   **读写参数**：`fd`、起始逻辑地址（缺省时为当前访问地址，存于“读写状态信息表”）、长度、读写信息用户区地址。
*   **Read/Write 系统调用大致处理过程**：
    1.  核实参数，按 `fd` 获得文件读写状态信息表，进而获得活跃文件目录表中的文件FCB。
    2.  核实操作许可（权限检查）。
    3.  按FCB中的定位信息（如索引表），将要读写的**逻辑地址**转化成**物理地址**。
    4.  **写操作**：先将数据从用户区拷入系统区；填好外设物理地址、内存地址、长度等参数；调用外存驱动程序进行I/O操作。
    5.  **读操作**：调用驱动读取后，需将系统区数据拷入用户区。

#### 5. Memory-Mapped（内存映射）文件访问
*   **系统调用支持**：
    *   `map`：将一个文件映射到一段进程地址空间。
    *   `unmap`：将文件与指定进程虚空间段脱钩。
*   **访问流程**：Open文件 -> Map文件到一片虚空间 -> 读写这片虚空间 -> Unmap文件。
*   **原理**：
    *   传统读文件：先把文件信息块读入系统缓冲区，再送入用户进程指定位置。
    *   内存映射文件：将文件映射到内存的一个区域，返回包含文件复制的字节数组的虚拟地址。**仅当需要对文件存取时，才传输实际的数据**。
    *   机制：使用与请求页式虚存管理相同的存取机制。访问的页不在主存时，产生**缺页中断**读入主存。
*   **应用**：广泛应用于 UNIX, Linux, Windows。

### 三、 文件保护

文件可能遭受两种破坏情形：
1.  不正确的访问方式或非受权访问 -> 通过**访问控制**解决。
2.  设备介质错、病毒等导致文件卷数据破坏 -> 通过**备份文件**避免或减少损失。

#### 1. 文件访问保护
*   **口令 (Password)**：
    *   创建文件时提供口令，存于FCB中。
    *   访问时系统核对口令。
*   **数据加密 (Encryption)**：
    *   写入时加密，读出时解密。
    *   访问者必须提供密钥。
    *   *特例*：Unix中 `/etc/passwd` 的用户口令是加密后写入，验证时无需解密读出（通常是比较哈希值）。
*   **访问控制 (Access Control)**：
    *   检查用户对文件的访问权限与本次访问是否一致。
    *   **访问控制矩阵**：一维列出全部用户，二维列出所有文件，$C_{ij}$ 值是 $i$ 用户对 $j$ 文件的访问权限。
    *   **简化访问表 (ACL简化版)**：针对文件主、同组用户和其它用户给出访问权限。
        *   例如：Owner (R W E), Group (R E), Others (E)。

#### 2. 文件备份
*   **批量备份**：
    *   **全量转储**：把全部文件定期复制到磁带或备份盘上。
    *   **增量转储**：定期把上次转储以来改过的文件和新文件转储。
*   **同步备份**：
    *   **镜像盘支持**：在对磁盘写操作的同时，对称地写其镜像盘。
    *   **双机动态文件备份 two machines working symmetrically**：两台机器在文件写操作时完全对称工作。当一台出错时，另一台可继续工作。结合双份盘可防止来自处理机和存储介质两方面的损坏。

### 四、 文件系统基本模型

#### 1. 概述
*   **职责**：负责文件的建立、删除、读/写、修改等，以及对文件目录结构和存储介质实施管理。
*   **特点**：OS文件系统只提供读/写文件信息的支持，**不解释其信息内涵**，内涵由各读写文件的应用程序去解释。
*   **优点**：
    *   **方便灵活**：用户不用管文件在物理存储器上的具体存放。
    *   **安全可靠**：可进行访问限制，可进行文件数据备份与恢复。
    *   **方便共享**：可按名访问。

#### 2. 文件系统层次结构 (从上到下)

| 层级 | 模块名称 | 主要功能 |
| :--- | :--- | :--- |
| **1** | **用户调用接口及初始化模块** | • 对用户进程发出的文件类系统调用进行处理<br>• 检查参数合法性<br>• 补充省缺参数<br>• 把系统调用转化为对相应子程序的调用<br>• 负责进程用户空间与OS内核的数据传输交换 |
| **2** | **文件目录系统** | • 管理与组织文件系统目录结构（建立、删除、查找目录等）<br>• 管理“活跃文件目录表”、“文件读写状态信息表”和“活动名字表”<br>• 处理上下级目录转换 |
| **3** | **存取控制验证** | • 涉及文件保护的程序属于此模块<br>• 验证用户权限 |
| **4** | **逻辑文件系统与文件信息缓冲区** | • 根据文件的逻辑结构，将用户欲读写的**逻辑记录**转换成**文件逻辑块号**及**块内offset**<br>• *字节流式文件示例*：逻辑块号 = 开始字节数 / 块长 (取整)；块内offset = 开始字节数 % 块长 (取余)<br>• **文件缓冲**：系统将文件的某些“相对块号”数据存放于缓冲区，按Hash表及Hash队列组织。读写时先查缓冲区，若命中则直接访问。 |
| **5** | **物理文件系统** | • 把**逻辑块号**转换成**物理块号**（利用FCB内的索引表信息）<br>• 负责对下层模块函数的调用（如外存空间分配及I/O驱动程序）<br>• 有些OS在此设立磁盘块缓冲 |
| **6** | **设备管理程序 (驱动器)** | • 分配设备<br>• 磁盘请求调度<br>• 启动外设<br>• 处理外设中断<br>• 释放设备 |
| **7** | **分配模块** | • 负责分配和回收外存空间 |

*(注：底层还涉及外存空间管理和设备管理)*

### 五、 总结知识点

1.  **文件系统定义**：OS的信息管理部分，提供文件的**按名存取**。负责文件的存储、检索和存取保护。
2.  **文件定义**：具有符号名的相关信息的集合。
3.  **文件的逻辑结构**：
    *   无结构的字节流式文件。
    *   有结构的记录式文件。
4.  **文件的存取方法**：
    *   顺序存取。
    *   直接（随机）存取。
5.  **文件物理结构**：连续、链接、索引等。
6.  **文件目录**：
    *   文件系统提供按名存取的重要数据结构。
    *   类型：一级、二级和多级树状目录。
    *   优势：二级和多级目录较好地解决了文件的重名和共享问题。
7.  **文件存储空间管理方法**：
    *   空白文件目录。
    *   位映像表（或位示图）。
    *   空白块链（成组链接法）。
8.  **存取控制方法**：
    *   存取保护域。
    *   存取控制矩阵。
    *   存取控制表 (ACL)。
    *   口令和密码。
9.  **操作命令**：文件系统向用户提供的接口。
10. **文件系统层次**：应用程序接口 -> 文件目录系统 -> 存取控制 -> 逻辑文件系统 -> 物理文件系统 -> 存储分配模块 -> 设备驱动模块。
