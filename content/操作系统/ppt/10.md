## 1. 概述

### 1.1 为什么需要存储管理？
*   **冯·诺依曼结构要求：** 程序必须装入存储器才能执行。
*   **资源珍贵：** 存储器是稀缺资源，需合理使用。
*   **地址映射需求：** 程序的逻辑空间（虚拟地址）与实际的物理空间（物理地址）不同，需要进行映射。

### 1.2 存储管理的目标
1.  **透明性：** 让用户和程序不涉及内存物理细节。
2.  **自动装入：** 自动完成用户程序的装入。
3.  **提高利用率：** 有效利用存储资源。
4.  **速度匹配：** 解决内存速度与CPU速度不匹配的问题（通过缓存等层次结构）。
5.  **共享与保护：** 实现内存共享和安全保护。

### 1.3 存储管理的主要任务
*   **地址变换（重定位）：** 逻辑地址 $\rightarrow$ 物理地址。
*   **存储资源的分配和回收。**
*   **存储共享和保护。**
*   **存储器扩充：** 通过覆盖技术、交换技术或虚拟存储技术。

## 2. 地址重定位 (Address Relocation)

**定义：** 当程序被装入内存时，将程序的**逻辑地址**转换成内存的**物理地址**的过程。由内存管理单元 (MMU) 完成。

### 2.1 绝对装入 (Absolute Loading) / 固定地址再定位
*   **时机：** 编译/连接时确定。
*   **原理：** 直接生成物理地址。逻辑地址空间与物理地址空间一一对应。
*   **优点：** 装入过程简单。
*   **缺点：** 缺乏灵活性，依赖硬件结构，程序必须在指定位置运行。
*   **应用：** 单片机，MS-DOS `.com` 格式程序。

### 2.2 可重定位装入 (Relocatable Loading)

#### A. 静态重定位 (Static Relocation)
*   **时机：** 装入内存时（执行前）。
*   **原理：** 修改程序中的逻辑地址值，将其转换为物理地址。一旦定位，执行期间地址不再变化。
*   **优点：** 无需特殊硬件支持，易于实现。
*   **缺点：**
    *   必须分配**连续**存储空间。
    *   执行期间不能移动，不能动态扩充。
    *   内存利用率低，不便于共享。

#### B. 动态重定位 (Dynamic Relocation)
*   **时机：** 执行时（访问内存前实时转换）。
*   **原理：** 装入时不修改逻辑地址。使用**基址寄存器 (BR)** 存放起始地址，**变址寄存器/逻辑地址** 存放偏移量。
    *   $物理地址 = 基址 + 逻辑地址$
*   **优点：**
    *   程序可在内存中移动（解决碎片问题，如紧凑/Compact）。
    *   支持换入换出，缓解内存紧张。
    *   不必分配连续空间（配合分页/分段）。
    *   便于实现共享。
*   **缺点：** 需要附加硬件支持（MMU），软件算法复杂。

## 3. 分区存储管理方案 (连续存储管理)

**特点：** 要求连续分配存储空间，程序需一次性全部装入内存。
**分类：** 单一连续、固定分区、可变分区。

### 3.1 单一连续分区存储管理
*   **方式：** 内存分为系统区和用户区。一次只能装入一个用户程序，独占用户区。
*   **保护：** 使用**界地址寄存器**。每次访存检查地址是否越界（$A > a?$），越界则终止。
*   **优点：** 简单，适用于单用户单任务系统。
*   **缺点：** 内存浪费严重（内部碎片），不支持多道程序，不支持大作业。

### 3.2 固定分区管理 (Fixed Partitioning)
*   **方式：** 初始化时将内存划分为若干个大小相等或不等的固定分区。
*   **实现：**
    *   **上下界寄存器法：** 记录分区的起始和结束地址。
    *   **基址+长度寄存器法：** 记录基址和长度，动态地址转换。
*   **优点：** 支持多道程序，实现简单，开销小。
*   **缺点：**
    *   **内碎片 (Internal Fragmentation)：** 分区内未被利用的空间。
    *   作业需预先估计大小。
    *   分区数固定，限制并发度。
*   **调度策略：** 多队列法（每个分区一个队列）或单队列法。

### 3.3 可变分区管理 (Dynamic Partitioning)
*   **思想：** 预先不划分，作业时按需分配，大小等于作业所需。剩余部分作为新的空闲块。
*   **数据结构管理空闲区：**
    *   **线性表格法：** 数组记录空闲块的起始地址和大小。
    *   **链接法：** 双向链表连接空闲块。
*   **分配算法：**
    1.  **最先适应算法 (First-Fit)：**
        *   空闲区按**地址递增**排列。
        *   从头查找第一个满足要求的分区。
        *   *优：* 保留高地址大空闲区，利于大作业；*缺：* 低地址碎片多，查找效率随碎片增加而降低。
    2.  **下次适应算法 (Next-Fit / Circular First-Fit)：**
        *   从上次分配位置的下一个开始查找，循环进行。
        *   *优：* 空闲区分布均匀，查找快；*缺：* 难以保留大空闲区。
    3.  **最佳适应算法 (Best-Fit)：**
        *   空闲区按**容量递增**排列。
        *   查找最小但能满足要求的分区。
        *   *优：* 保留大空闲区；*缺：* 产生大量极小的外部碎片，无法利用。
    4.  **最坏/最大适应算法 (Worst-Fit)：**
        *   空闲区按**容量递减**排列。
        *   总是分配最大的空闲区。
        *   *优：* 分配快；*缺：* 破坏大空闲区，导致后来大作业无法装入。

*   **回收与合并规则：**
    *   释放区与前空闲区相邻 $\rightarrow$ 合并。
    *   释放区与后空闲区相邻 $\rightarrow$ 合并。
    *   前后都相邻 $\rightarrow$ 三者合并。
    *   都不相邻 $\rightarrow$ 插入新节点。

*   **碎片问题：**
    *   **外部碎片 (External Fragmentation)：** 内存中存在许多小的空闲块，总和足够但分散，无法容纳大作业。
    *   **解决方案 - 紧致 (Compaction)：** 移动所有已分配分区，使其连续，集中空闲空间。要求支持**动态重定位**。

## 4. 页式存储管理 (Paging)

**核心思想：** 解决作业不连续存放的问题。
*   **逻辑空间：** 称为“页” (Page)，大小相同，通常为 $2^k$。
*   **物理空间：** 称为“页帧”或“物理块” (Frame/Page Frame)，大小与页相同。
*   **映射：** 作业的页可以离散地分布在内存的任何页帧中。

### 4.1 基本原理
*   **地址结构：**
    *   逻辑地址 = 页号 ($P$) + 页内位移 ($d$)
    *   物理地址 = 页帧号 ($f$) + 页内位移 ($d$)
    *   若页面大小为 $2^k$，则地址低 $k$ 位为位移 $d$，高位为页号 $P$。通过位移操作即可分离，无需除法。
*   **页表 (Page Table)：**
    *   每个进程拥有一个页表，存放在系统空间。
    *   PCB 中有指针指向页表。
    *   作用：记录逻辑页号到物理页帧号的映射关系。
*   **地址转换过程：**
    1.  CPU 给出逻辑地址。
    2.  分离出页号 $P$ 和位移 $d$。
    3.  查页表，查页表的第 $P$ 项，找到对应的页帧号 $f$。
    4.  拼接 $f$ 和 $d$ 得到物理地址。
    5.  访问物理内存。
    *   *注意：* 若无快表，每次数据访问需**两次**访存（一次查页表，一次取数据）。

### 4.2 快表 (TLB - Translation Lookaside Buffer)
*   **组成：** 一组联想寄存器（Content-Addressable Memory），并行查找，速度极快。
*   **原理：** 缓存最近使用的页表项（页号 $\rightarrow$ 页帧号）。
*   **流程：**
    1.  先在 TLB 中查找页号。
    2.  **命中 (Hit)：** 直接获得页帧号，计算物理地址。
    3.  **未命中 (Miss)：** 访问内存中的页表，获取页帧号，并将该条目调入 TLB（可能替换旧条目）。
*   **性能：** 命中率通常可达 80%-90%。
*   **等效访问时间计算示例：**
    *   设访存时间 $t_m = 750ns$，TLB 查找时间 $t_t = 50ns$，命中率 $h = 80\%$。
    *   $EAT = h \times (t_t + t_m) + (1-h) \times (t_t + t_m + t_m)$

### 4.3 空闲内存页管理
*   **位示图 (Bitmap)：**
    *   用二进制位表示页帧状态（0空闲，1占用，或反之）。
    *   计算示例：内存 256MB，页大小 4KB。
        *   总页数 = $256 \times 1024 \times 1024 / (4 \times 1024) = 65536$ 页。
        *   位示图大小 = $65536 / 8 = 8192$ Bytes = 8 KB。
*   **空闲页帧链：** 将所有空闲页帧链接成链表。

### 4.4 共享与保护
*   **共享：** 不同进程的页表中，不同的逻辑页号可以指向同一个物理页帧（如共享代码段 EDIT1）。
*   **保护：**
    1.  **越界保护：** 页表长度寄存器。检查页号是否超过页表长度。
    2.  **访问权限保护：** 页表项中增加保护位（如 R/W/E）。
        *   R (Read), W (Write), E (Execute)。
        *   每次访问检查权限，不符则报异常。

### 4.5 页式管理的优缺点
*   **优点：**
    *   **无外碎片：** 只有页大小的内碎片。
    *   **离散分配：** 程序不必连续存放，提高内存利用率。
*   **缺点：**
    *   **内碎片：** 每个作业最后一页可能有浪费，平均浪费半个页面。
    *   **系统开销：** 页表占用内存空间；建立和管理页表花费时间。
    *   **硬件成本：** 需要 MMU 和 TLB 支持，增加成本和略微降低速度（尽管 TLB 缓解了这点）。
    *   **一次性装入：** 传统页式管理仍要求作业一次性全部装入内存（这是与虚拟存储的区别，虚拟存储允许部分装入）。

## 总结对比

| 特性 | 连续存储管理 (分区) | 页式存储管理 |
| :--- | :--- | :--- |
| **连续性** | 必须连续 | 离散 (不连续) |
| **碎片类型** | 内碎片 (固定分区) <br> 外碎片 (可变分区) | 仅内碎片 (最后一页) |
| **装入要求** | 一次性全部装入 | 一次性全部装入 (传统页式) |
| **地址转换** | 基址寄存器 (动态重定位) | 页表 + MMU (+ TLB) |
| **共享实现** | 困难 | 容易 (页表指向同一帧) |
| **复杂度** | 较低 | 较高 (需硬件支持) |
