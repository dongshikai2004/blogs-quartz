## 1. 为什么要引入进程的概念？

### 1.1 程序的顺序执行
*   **定义：** 在任何时刻，机器只执行一个操作，只有在前一个操作执行完后，才能执行后继操作。
*   **示例：**
    *   作业层面：$I_1 \rightarrow I_2 \rightarrow I_3 \rightarrow P_1 \rightarrow C_1 \rightarrow P_2 \dots$
    *   语句层面：`S1: a=x+y;` $\rightarrow$ `S2: b=a-5;` $\rightarrow$ `S3: c=b+1;`
*   **特点：**
    1.  **顺序性**
    2.  **资源独占性/封闭性：** 程序运行时独占全机资源，资源状态仅由该程序决定。
    3.  **可再现性：** 只要初始条件相同，无论连续运行还是断续运行，结果与执行速度无关。
*   **优点：** 封闭性和可再现性便于程序员调试。
*   **缺点：** 资源独占导致系统资源利用率非常低。

### 1.2 多道程序设计 (Multiprogramming)
*   **定义：** 同一时刻内存中存放多个作业，处理器交替运行不同的作业。
*   **目标：** 提高系统效率，尤其是资源利用率。
*   **特点：** 多道、宏观上并行、微观上串行。
*   **问题：** 系统管理复杂化。

### 1.3 程序的并发执行
*   **定义：** 计算机同时运行几个程序，CPU 不断地在几个程序之间切换。
*   **新特征（带来的挑战）：**
    1.  **资源共享性，失去了封闭性：** 多个程序共享资源，资源状态由多个程序改变。
    2.  **制约性：** 因共享资源或协调完成同一任务而引起。
    3.  **程序执行的间断性。**
    4.  **结果不可再现：** 执行结果与相对执行速度有关（时间有关的错误）。
        *   *示例：* 售票系统或变量 N 的读写竞争（Race Condition）。若程序 A 和 B 共享变量 N，不同执行顺序会导致不同结果（0, 1, 2 等）。
*   **结论：** 程序是一个静态概念，无法反映并发执行的动态特征（独立性、并发性、动态性）。因此引入**“进程”**概念。

## 2. 进程的概念

### 2.1 定义
*   **起源：** MIT (60年代初, MULTICS), IBM (CTSS/360, 称为 Task)。
*   **典型定义：**
    1.  进程是程序的一次执行。
    2.  进程是一个程序及其数据在处理机上顺序执行时所发生的活动。
    3.  进程是程序在一个数据集合上运行的过程，它是系统进行**资源分配和调度**的一个独立单位。
*   **传统 OS 定义：** 进程是程序的运行过程，是系统进行资源分配和调度的一个独立单位。

### 2.2 进程的特性
1.  **动态性：** 进程是程序的一次执行过程，是临时的，有生命期的。
2.  **独立性：** 进程是系统进行资源分配和调度的一个独立单位。
3.  **并发性：** 多个进程可在处理机上交替执行。
4.  **结构性：** 系统为每个进程建立一个进程控制块 (PCB)。

### 2.3 进程与程序的区别和联系
| 维度 | 程序 (Program) | 进程 (Process) |
| :--- | :--- | :--- |
| **性质** | 静态的（有序代码集合） | 动态的（程序的一次执行） |
| **存在性** | 永久的（可长久保存） | 暂时的（有生命周期） |
| **功能** | 无资源分配功能 | OS 资源分配和保护的基本单位 |
| **对应关系** | 一个程序可对应多个进程（多次执行）<br>一个进程可包括多个程序（调用关系） | 进程可创建其他进程 |
| **结构** | 仅代码 | 程序 + 数据 + **进程控制块 (PCB)** |

## 3. 进程的描述

### 3.1 进程的组成
进程由三部分组成：
1.  **程序 (Program)：** 描述进程要完成的功能，执行时不可修改。
2.  **数据 (Data)：** 执行时用到的数据（用户输入、常量、静态变量）。
3.  **进程控制块 (PCB, Process Control Block)：**
    *   **地位：** 进程的**灵魂**，进程存在的**唯一标志**。
    *   **作用：** 系统管理控制进程运行用的信息集合（含核心栈）。

**进程映像 (Process Image)：** 指进程执行的程序、数据、栈。其存放取决于存储管理机制。

### 3.2 进程控制块 (PCB) 详解
*   **定义：** OS 用来记录进程详细状态和相关信息的基本数据结构，与进程一一对应。
*   **包含的三大类信息：**

| 类型 | 具体内容 | 作用 |
| :--- | :--- | :--- |
| **标识信息** | 1. 进程标识 (PID)<br>2. 用户标识<br>3. 父进程标识 | 标识一个进程 |
| **现场信息**<br>(处理机状态) | 1. CPU 通用寄存器内容<br>2. CPU 控制和状态寄存器内容 (PC, PSW)<br>3. 堆栈指针等<br>*(实质是核心栈)* | 记录处理机现场信息，以备恢复之用 |
| **控制信息** | 1. **进程状态**<br>2. **调度信息** (优先级等)<br>3. 队列指针<br>4. 位置信息 (内存地址)<br>5. 资源占用信息 (打开的文件等)<br>6. 进程间通讯信息 (信号、信件)<br>7. 存储管理信息<br>8. 数据结构链接信息 |用于进程的调度管理、资源管理、通信等 |

*   **OS 实现实例：**
    *   Unix: `struct proc { ... }`
    *   Linux: `struct task_struct { ... }`
    *   Windows NT/2000/XP: `EPROCESS` (含地址空间、资源跟踪等) + `KPROCESS` / `ETHREAD` / `KTHREAD` (内核线程调度信息)。

### 3.3 PCB 的管理方式
操作系统对 PCB 进行集中统一管理：
1.  **线性表组织方式：** 所有 PCB 存放在一个数组中，通过下标访问。
2.  **链表组织方式：** 将具有相同状态的 PCB 组成队列。
    *   就绪队列（可按策略分多个，如高/低优先级）。
    *   阻塞队列（按阻塞原因分，如等待磁盘 I/O、等待磁带 I/O 等）。
3.  **索引方式：** 建立几张索引表（就绪索引表、阻塞索引表），记录首地址。
4.  **执行指针：** 系统设置专门指针指向当前运行进程的 PCB (如 UNIX 中的 `CURPRO`)。

## 4. 进程的状态

### 4.1 五种基本状态及转换
进程在其生命期内状态不断变化，状态信息记录在 PCB 中。

#### 状态定义：
1.  **新建 (New/Create)：** 进程正在被创建，尚未转到就绪状态。
2.  **就绪 (Ready)：** 已获得除处理机 (CPU) 外的一切所需资源，一旦得到 CPU 即可运行。
3.  **运行 (Running)：** 进程占有处理机，处理机正在执行该进程的程序。
4.  **阻塞 (Blocked/Waiting)：** 又称等待态、睡眠态。进程因等待某种条件（如 I/O、同步事件）而无法运行。
5.  **结束 (Exit/Terminated)：** 进程正在从系统中消失，因正常结束或异常终止。

#### 状态转换及原因：

| 转换方向 | 原因/条件 |
| :--- | :--- |
| **空 $\rightarrow$ 创建** | 一个新进程被产生来执行一个程序。 |
| **创建 $\rightarrow$ 就绪** | 进程创建完成，初始化后，一切就绪准备运行。 |
| **就绪 $\rightarrow$ 运行** | **进程被调度程序选中**，占用 CPU。 |
| **运行 $\rightarrow$ 就绪** | 1. **时间片用完**。<br>2. **中断机制**下，有更高优先级进程进入系统（可剥夺调度）。 |
| **运行 $\rightarrow$ 阻塞** | 进程请求某样东西且必须等待：<br>1. 请求 OS 服务但资源不可用。<br>2. 启动 I/O 但未完成。<br>3. 等待与其他进程通讯（如接收未发出的信件）。 |
| **阻塞 $\rightarrow$ 就绪** | 等待的事件已经发生，或等待的资源可用。 |
| **运行 $\rightarrow$ 结束** | 进程指示已完成或因错终止，OS 作结束处理。 |

*(注：图示通常为五态模型，其中“新建”和“结束”有时被视为过渡状态，核心循环为 就绪-运行-阻塞)*

### 4.2 进程挂起 (Suspend) - 七种状态模型
*   **定义：** 进程没有占用内存空间，进程映像在磁盘上。
*   **原因：** 通常由于内存不足或用户请求。
*   **状态扩展：** 在原有状态基础上增加“挂起”，形成更复杂的七态模型（如：就绪挂起、阻塞挂起等），涉及**解挂 (Activate)** 操作。

## 5. 进程的创建与结束

### 5.1 进程创建过程
操作系统为进程创建 PCB 和分配地址空间的过程。
**大致步骤：**
1.  接收参数：初始优先级、执行程序描述、输入参数、其他资源需求。
2.  请求分配 PCB 空间，获得内部数字进程标识 (PID)。
3.  用参数初始化 PCB 表。
4.  产生描述进程空间的数据结构，初始化进程空间（建立程序段、数据段、用户栈等）。
5.  设置处理机现场保护区，构造进程运行栈帧。
6.  置好父进程等关系域。
7.  将进程置为**就绪状态**。
8.  将 PCB 挂入**就绪队列**，等待调度。

### 5.2 进程结束过程
**大致步骤：**
1.  将进程状态改到**结束状态**。
2.  关闭所有打开的文件、设备。
3.  释放进程程序文件和其他资源。
4.  进行相关信息统计。
5.  清理相关进程链接关系：
    *   *Linux 特例：* 将结束进程的所有子进程链到 **1号进程 (init)**，作为其子进程，并通知父进程自己已结束。
6.  释放进程映像空间。
7.  释放进程控制块 (PCB)。
8.  调用进程调度与切换程序。

### 5.3 Linux 相关系统调用
*   `fork()`: 创建新进程。
*   `exit()`: 进程自我终止。
*   `wait()` / `waitpid(pid, &statloc, opts)`: 父进程阻塞等待子进程结束，回收资源。
