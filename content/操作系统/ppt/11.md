## 一、 段式存储管理 (Segmentation)

### 1. 基本思想与特点
*   **背景：** 页式管理对用户而言不够自然（用户看到的是逻辑模块，而非固定大小的页）。
*   **定义：** 按作业的“自然段”（如主程序、子程序、数据段等）将逻辑空间分成若干段，作业以**段**为单位分配内存。
*   **空间结构：**
    *   **逻辑空间：** 二维空间，由若干个自然段组成。
    *   **逻辑地址格式：** `段号(S), 段内偏移(d)`。编译及装配时地址记为 `(S, d)` 形式。
    *   **物理内存管理：** 采用**多道可变划分法**（动态分区），系统以段为单位分配物理内存。

### 2. 数据结构：段表 (Segment Table)
*   **位置：** 存放于系统空间。进程的 PCB 中存有段表始地址和段表长度。
*   **寄存器：** 段表起始地址寄存器、段表长度寄存器。
*   **段表：** 由段表项组成。
*   **段表项结构：**
    *   保护码
    *   段长
    *   本段内存始地址

### 3. 地址转换过程
1.  CPU 给出逻辑地址 `(S, d)`。
2.  硬件检查 `S` 是否越界（`S >= 段表长度` ?）。若越界则报错。
3.  根据 `S` 查段表，获取该段的**起始地址**和**段长**。
4.  检查 `d` 是否越界（`d >= 段长` ?）。若越界则报错。
5.  **物理地址计算：** `物理地址 = 段起始地址 + 段内偏移 d`。
6.  访问内存。
    *   *注：可使用联想存储器（TLB）加速查找。*

### 4. 优缺点
*   **优点：**
    *   **无内碎片：** 段大小可变，按需分配。
    *   **外碎片处理：** 可通过内存紧致（Compaction）消除。
    *   **便于共享：** 允许不同进程共享同一个或几个段（如共享代码段、库函数）。
    *   **便于保护：** 可以对不同的段设置不同的保护权限。
*   **缺点：**
    *   存在外碎片。
    *   内存分配算法较复杂。

## 二、 分页式与分段式的比较

| 比较项目 | 页式存储管理 | 段式存储管理 |
| :--- | :--- | :--- |
| **划分依据** | 系统管理需要（固定大小） | 用户应用需要（逻辑模块） |
| **页/段大小** | 各页面大小相同 | 段的大小不固定 |
| **逻辑地址** | 一维线性地址空间 | 二维地址空间（段号+段内偏移） |
| **页表/段表** | 页面较多，页表较长，查找费时 | 段较少，段表较短，查找速度快 |
| **碎片问题** | 存在内碎片 | 存在外碎片 |
| **内存共享** | 支持不好（需整页共享） | 支持好（可按段共享） |
| **存储扩充** | 纯页式不支持（需结合虚存） | 纯段式不支持（需结合虚存） |

## 三、 段页式存储管理 (Segmented Paging)

### 1. 基本思想
*   结合段式和页式的优点。
*   **对用户而言：** 与段式相同，逻辑地址涉及段号与段内位移。
*   **对物理内存而言：** 与页式相同，每段内部再划分为固定大小的页。
*   **实现方式：** 将作业分成若干段，每段用页式管理实现内存分配。

### 2. 地址结构
*   **逻辑地址：** `段号(S), 页号(P), 页内地址(W)`。
*   **转换过程：**
    1.  根据 `S` 查**段表**，得到该段的**页表始址**。
    2.  根据 `P` 查**页表**，得到**页帧号**。
    3.  **物理地址：** `页帧号 + 页内地址 W`。

### 3. 数据结构
*   **段表：** 每个段对应一个段表项，包含：保护码、长度、**页表始地址**。
*   **页表：** 每个段有一个独立的页表，包含：页帧号、保护码等。

### 4. 性能分析（等效访问时间）
*   **问题：** 访问一条指令或数据需要访问几次内存？
    *   理想情况（TLB命中）：1次访存（数据本身）。
    *   最坏情况（TLB未命中）：3次访存（查段表 + 查页表 + 访问数据）。
*   **计算公式示例：**
    *   设访存时间 $t = 750ns$，TLB搜索时间 $t_{TLB} = 50ns$，命中率 $H = 95\%$。
    *   $EAT = H \times (t_{TLB} + t) + (1-H) \times (t_{TLB} + t + t + t)$

### 5. 改进的页式管理
*   将段存放于线性逻辑空间，但不同段不放在同一页面内，解决了“页跨段”的问题，简化了管理。

## 四、 内存扩充技术

### 1. 提出原因
*   基本存储管理中，作业必须全部装入内存才能运行，限制了并发进程数和作业大小。
*   目标：借助大容量辅存，在逻辑上扩充内存容量。

### 2. 覆盖技术 (Overlay)
*   **原理：** 将用户空间分为**固定区**和**覆盖区**。
    *   固定区：常驻内存（如主程序、常用模块）。
    *   覆盖区：不同时运行的模块共享同一块内存区域。
*   **实现：** 由程序员显式调用覆盖系统函数，操作系统负责调入调出。
*   **优点：** 有效利用内存，提高并发性。
*   **缺点：**
    *   编程复杂度高（需精心设计覆盖结构）。
    *   以时间换空间（I/O开销大）。
    *   覆盖区内仍存在碎片。

### 3. 交换技术 (Swapping)
*   **原理：** 将处于等待状态或就绪状态的整个进程从主存换出到辅存，将待执行进程换入主存。
*   **应用：** 最早用于 CTSS 系统。
*   **优点：** 增加并发程序数目，可提供优先级服务。
*   **缺点：**
    *   换入换出控制增加 CPU 开销。
    *   存在重定位问题。
*   **与覆盖技术的区别：**
    *   覆盖技术作用于**作业内部**，影响程序结构。
    *   交换技术作用于**作业之间**，不影响程序结构。

## 五、 虚拟存储技术 (Virtual Memory)

### 1. 基础理论
*   **局部性原理：**
    *   **时间局部性：** 刚被访问的指令/数据很可能再次被访问（如循环变量）。
    *   **空间局部性：** 刚被访问的单元附近的单元很可能被访问（如数组遍历）。
*   **事实依据：**
    *   程序中不是所有指令都会执行（如错误处理、条件分支）。
    *   部分指令只执行一次（如初始化）。
    *   执行集中在某些区域。

### 2. 基本思想
*   **目的：** 提供比物理内存大得多的虚拟存储空间。
*   **手段：** 利用外存（磁盘）作为后备存储。
*   **容量：** 由计算机地址结构决定（如32位地址对应 $2^{32}$ 字节）。
*   **工作原理：**
    *   程序装入时，只需装入当前需要执行的部分。
    *   执行过程中，若访问的页/段不在内存，产生**缺页/缺段中断**。
    *   OS 将所需部分从外存调入内存，若内存满则淘汰旧页/段。
    *   重新执行指令。

### 3. 分类
*   虚拟页式存储管理（纯页式 + 请求调页）
*   虚拟段式存储管理（纯段式 + 请求调段）
*   虚拟段页式存储管理

## 六、 虚拟页式存储管理详解

### 1. 页表项扩展结构
为了支持虚存，页表项需增加以下字段：
*   **合法位 (Valid Bit)：** 1表示该页在内存，0表示不在。
*   **修改位 (Dirty Bit)：** 1表示该页在内存中被修改过，淘汰时需写回外存。
*   **页类型：**
    *   **零页：** 初始化为0的页，分配物理帧时直接清零，无需从磁盘读。
    *   **回写Swap页：** 需要写回交换区。
*   **保护码：** R/W/E 权限。
*   **外存块号：** 该页在外存（程序文件或Swap区）的位置。
*   **页帧号：** 若在内存，指示物理块号。

### 2. 交换区 (Swap Space)
*   **作用：** 存放运行时被修改的数据页和栈页。
*   **区别：**
    *   **程序文件：** 存放二进制代码和初始数据。只读，不需回写。
    *   **交换区：** 存放运行时产生的脏数据。需回写。

### 3. 页表建立过程 (以 fork() 为例)
1.  分配 PID 和 PCB。
2.  分配子进程页表空间。
3.  **复制父进程页表项：**
    *   **程序区：** 共享父进程页表项（Copy-on-Write 或只读共享），合法位置1。
    *   **数据/栈区：** 复制内容，标记为可读写，分配 Swap 空间或标记为零页/Swap页。
4.  初始化子进程现场，挂入就绪队列。

*(注：Windows 使用可执行文件初始化页表；UNIX fork 采用部分复制)*

### 4. 硬件动态地址转换流程
1.  CPU 发出虚拟地址。
2.  **查 TLB：** 若命中，直接获得页帧号，拼接物理地址访问内存。
3.  **查页表：** 若 TLB 未命中，查内存中的页表。
    *   检查**合法位**：
        *   若为 1：获得页帧号，更新 TLB，访问内存。
        *   若为 0：**产生缺页中断 (Page Fault)**。

### 5. 缺页中断处理流程
当发生缺页中断时，OS 执行以下步骤：
1.  **保存现场：** 保护 CPU 状态。
2.  **分析原因：** 确定缺失的页号。
3.  **检查合法性：** 确认访问是否非法（如访问非映射区域），若非法则终止进程。
4.  **寻找空闲页帧：**
    *   若有空闲帧，直接使用。
    *   若无空闲帧，执行**页面替换算法**选择牺牲页。
5.  **处理牺牲页（若被选中）：**
    *   检查**修改位**：
        *   若未修改：直接释放页帧，加入空闲链表。
        *   若已修改：检查页类型。若是零页或 Swap 页，分配 Swap 块号，调用 I/O 将数据写回外存。清除合法位，释放页帧。
6.  **调入新页：**
    *   若是**零页**：分配物理帧并清零，置合法位为1。
    *   若是**文件页/Swap页**：调用 I/O 子系统，从外存指定块号读取数据到物理帧。
7.  **更新页表：** 填入页帧号，置合法位为1，清除修改位。
8.  **更新 TLB。**
9.  **恢复现场：** 重新执行导致缺页的那条指令。

### 6. 页面淘汰策略
*   现代 OS 通常定时进行预淘汰或按需淘汰。
*   核心工作：判断是否写回外存（取决于修改位和页类型）。

## 七、 总结：“放”的策略

1.  **连续存放：**
    *   单道连续划分
    *   多道连续固定划分
    *   多道连续可变划分
2.  **不连续存放：**
    *   页式存储
    *   段式存储
    *   段页式存储
    *   改进的页式存储
