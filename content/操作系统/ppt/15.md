## 1. 引言

### 1.1 设备管理的目标与功能
*   **主要功能之一：** 负责管理所有外部设备。
*   **核心目标：** 提高设备的利用率，即提高 **CPU与I/O设备之间的并行操作程度**。
*   **用户界面要求：**
    *   **方便性：** 用户能独立于具体设备的复杂物理特性之外而方便地使用设备。
    *   **统一性：** 对不同的设备尽量使用统一的操作方式。

### 1.2 设备管理的任务
1.  动态地掌握并记录设备的状态。
2.  设备分配和释放。
3.  缓冲区管理。
4.  实现物理I/O设备的操作。
5.  提供设备使用的用户接口。
6.  设备的访问和控制。
7.  I/O缓冲和调度。

### 1.3 常见外部设备的分类
*   **人机交互设备：** 字符设备，通常以字符方式发送/接收数据。
*   **存储类型设备：** 块设备，读写以数据块为单位。
*   **网络通讯设备。**
*   **特点差异：** 设备的功能、控制复杂性、数据传输单位各异；设备驱动接口和使用接口不同。

## 2. I/O硬件组成

### 2.1 I/O体系结构
计算机系统的I/O体系结构主要由以下三部分组成：
1.  **I/O端口 (I/O Ports)：**
    *   连接到I/O总线上的设备的I/O地址集。
    *   每个设备寄存器有一个端口号。
    *   **设备寄存器包括：** 控制寄存器、状态寄存器、数据缓冲寄存器等。
2.  **I/O接口 (I/O Interface)：**
    *   处于一组I/O端口和对应的设备控制器之间的一种硬件电路。
3.  **I/O总线 (I/O Bus)：**
    *   CPU与I/O设备之间的通路。

> **PC的I/O体系结构模型：** CPU <-> I/O总线 <-> I/O接口/I/O控制器 <-> I/O设备

### 2.2 设备控制器 (I/O部件)
*   **组成：** I/O设备由机械部分（设备本身）和电子部分（设备控制器）组成。
*   **位置与作用：** 控制器处于CPU和I/O设备之间。操作系统与控制器打交道，控制器接收CPU命令，控制I设备工作。
*   **历史演变：** 早期CPU直接控制外设；引入I/O部件后，I/O指令功能加强，将CPU从与外设交互的细节中解放出来。
*   **控制器的任务：**
    *   在外设与内存之间完成 **比特流与字节块** 之间的转换。
    *   或在外设与CPU之间完成 **比特流（或外部信号）与字节流** 之间的转换。
    *   *示例：* 磁盘控制器将串行位流组装为字节，存入内部缓冲区形成块，验证后存入内存。
*   **控制器寄存器：**
    *   **控制寄存器：** 接收CPU发送的读写命令。
    *   **状态寄存器：** 包含设备的状态信息。
    *   **数据缓冲寄存器：** 通常为1B至4B。
    *   **数据缓冲区：** 许多控制器还有操作系统可读写的数据缓冲区（如视频RAM）。

> **IBM PC控制器寄存器编址示例：**
> *   键盘: 060-063
> *   硬盘: 320-32F
> *   打印机: 378-37F
> *   软盘: 3F0-3F7
> *   彩色显示器: 3D0-3DF

### 2.3 I/O数据传输的控制方式 (四种方式)

#### (1) 程序直接控制方式 (Programmed I/O)
*   **原理：** CPU直接控制I/O操作的全过程（发送命令、传输数据、测试状态）。需要处理机指令集包含控制类、测试类、读写类I/O指令。
*   **过程：**
    1.  CPU向I/O部件发读/写指令。
    2.  CPU反复读取I/O状态寄存器，检查状态（忙等）。
    3.  若就绪，通过CPU寄存器转发数据到内存。
    4.  重复直到一批数据传输完成。
*   **优点：** 简单。
*   **缺点：** CPU大部分时间用于测试硬件，效率低下；CPU与I/O串行工作。

#### (2) 中断控制方式 (Interrupt-driven I/O)
*   **原理：** CPU发出指令后转去执行其他进程。当I/O部件准备好数据后，利用**中断**通知CPU，再由CPU完成数据传输。
*   **过程：**
    1.  CPU启动设备工作。
    2.  CPU转去执行其他程序（并行工作）。
    3.  设备完成数据传输产生中断。
    4.  CPU响应中断，执行中断服务例程（检查错误、处理数据）。
    5.  中断返回。
*   **优点：** CPU不必等待，提高了利用率；实现了CPU和外设的并行工作。
*   **缺点：** 数据仍需通过CPU寄存器转发；每次传输一个字/若干字节都要经过CPU，消耗大量CPU时间；仅适于低速设备。

#### (3) 直接存储器访问方式 (DMA, Direct Memory Access)
*   **背景：** 前两种方式受限于CPU速度，且CPU管理I/O耗时多。
*   **原理：** DMA负责完成整个I/O操作，无需经CPU寄存器转发。仅在开始和结束时需CPU干预。
*   **DMA控制器组成：**
    *   数据缓冲寄存器
    *   主存地址寄存器 (MAR)
    *   传送字节个数计数器 (DC)
    *   控制/状态寄存器
    *   数据缓冲存储区
*   **工作过程：**
    1.  CPU向DMA控制器发布命令（操作类别、设备地址、内存首地址、数据长度）。
    2.  CPU做其他事情。
    3.  DMA控制器接管地址总线控制权。
    4.  DMA直接在控制器内部缓冲区与主存之间交换数据（挪用CPU周期）。
    5.  每传一个数据，MAR增1，DC减1。
    6.  DC=0时，DMA向CPU发出中断请求。
*   **优点：** CPU只需干预开始和结束；适于高速设备；数据传输不经过CPU寄存器。
*   **注意：** DMA工作时，CPU访问总线速度会变慢（因为DMA占用了总线）。

#### (4) 通道控制方式 (Channel I/O)
*   **定义：** 通道是一种专用的 **I/O处理机**。
*   **结构：** 一个CPU连接若干个通道，一个通道连接若干个控制器，一个控制器连接若干个设备。
*   **特点：**
    *   通道有自己的指令系统，若干条通道命令连接成 **通道程序**。
    *   CPU只需发出I/O指令（给出通道程序首地址和设备），通道独立执行通道程序完成任务。
    *   完成后，通道与设备一起发出中断请求。
*   **优点：**
    *   解决了I/O操作的独立性和各部件工作的并行性。
    *   CPU负担进一步减轻（比DMA干预更少）。
    *   实现了CPU与通道、通道与通道、通道上设备之间的多重并行。
*   **通道的三种类型：**
    1.  **字节多路通道 (Byte Multiplexor Channel)：**
        *   以字节为单位传输。
        *   分时执行多个通道程序（对应多台设备）。
        *   主要用于连接大量 **慢速设备**。
    2.  **选择通道 (Selector Channel)：**
        *   每次传送一批数据，速度快。
        *   一段时间内只能执行一个通道程序，只允许一台设备传输。
        *   用于 **固定头磁盘** 等高速设备。
    3.  **数组多路通道 (Block Multiplexor Channel)：**
        *   结合前两者优点。
        *   先为一台设备执行一条指令，再为另一台执行一条。
        *   可连接多台 **活动头磁盘机**。

> **I/O控制方式的发展过程总结：**
> CPU直接控制 -> 引入I/O部件(CPU直接控制部件) -> 引入中断驱动 -> 引入DMA -> I/O通道/I/O处理机

## 3. 设备I/O子系统 — 设备的使用方法

### 3.1 设备相关系统调用简介
主要用于对人机交互类慢速外设的使用。对于存储类和网络设备，通常通过文件系统或Socket接口间接使用。

*   **申请设备：** 参数说明设备名称。OS根据设备特性（独占/共享）及占用情况分配，返回成功标志。
*   **写入设备：** 将数据写入设备。
*   **读取设备：** 从设备读取数据
*   **释放设备：** 申请设备的逆操作。

> **Linux直接使用存储设备示例（绕过文件系统）：**
> ```c
> fd = open("/dev/sda", O_RDWR); // 申请U盘
> lseek(fd, 1024, 0);            // 定位到1024字节处
> write(fd, buffer, 36);         // 写入36字节
> ...
> close(fd);                     // 释放U盘
> ```
> *注意：必须清楚U盘的具体存储布局才能正确读写。*

### 3.2 三种设备使用方式

#### (1) 独占式使用 (Exclusive Use)
*   **定义：** 以一次设备使用过程（包含多次I/O操作）为单位使用设备。
*   **机制：** 申请时若空闲则独占，期间不允许其他进程申请，直到释放。
*   **适用：** 打印机等不可分割使用的设备。

#### (2) 分时式共享使用 (Time-sharing Shared Use)
*   **定义：** 以一次I/O操作为单位分时使用设备。
*   **机制：**
    *   如果一个逻辑上完整的数据可以用一次I/O操作完成，则不必独占。
    *   申请时不检查占用，只需累加使用者计数。
    *   不同进程的I/O请求排队，分时占用设备进行I/O。
*   **适用：** 磁盘等随机存取设备。

#### (3) SPOOLing虚拟设备方式 (Simultaneous Peripheral Operations On-Line)
*   **定义：** 设备本身是独占设备，但经过虚拟技术处理，改造成共享设备，供多个进程同时使用。又称“假脱机输入输出技术”。
*   **原理：**
    *   利用磁盘上的输入/输出井模拟慢速外设。
    *   利用文件模拟出多设备。
*   **示例（打印SPOOLing）：**
    1.  进程运行时，输出数据写到磁盘上的同一文件中。
    2.  文件排入打印输出队列。
    3.  打印守护进程 (Daemon) 占用打印机。
    4.  Daemon成批读出文件数据，送给打印机打印。
*   **优点：** 提高了独占设备的利用率，实现了虚拟共享。

**总结：**
本章重点讲解了I/O硬件结构（端口、接口、控制器）、四种I/O控制方式（程序直接控制、中断、DMA、通道）的演进与优缺点，以及三种设备使用策略（独占、分时共享、SPOOLing虚拟设备）。理解这些内容对于掌握操作系统如何高效管理异构外部设备至关重要。
