# 操作系统：进程的控制与调度 (Lecture 5)

## 1. 进程控制 (Process Control)

### 1.1 基本概念
*   **定义**：系统使用具有特定功能的程序段来创建、撤销进程，以及完成进程各状态之间的转换。
*   **实现者**：由操作系统内核实现。
*   **性质**：属于**原语 (Primitive)** 一级的操作，不能被中断。
    *   **原语**：由若干条机器指令构成的可完成特定功能的程序段，是一个“原子操作 (atomic operation)”。要么全做，要么全不做。通过屏蔽中断保证原子性。
    *   **分类**：进程控制原语、进程通信原语、进程管理原语等。

### 1.2 六大进程控制原语
1.  **进程创建原语 (Create)**
2.  **进程撤销原语 (Terminate)**
3.  **进程阻塞原语 (Block)**
4.  **进程唤醒原语 (Wakeup)**
5.  **进程挂起原语 (Suspend)**
6.  **进程激活/解挂原语 (Activate)**

### 1.3 详细原语过程

#### A. 进程创建 (Creation)
*   **引起事件**：系统初始化、用户登录、作业调度、提供服务、应用请求。
*   **步骤**：
    1.  申请空白 PCB (进程控制块)。
    2.  为新进程分配资源（程序、数据、用户栈内存等）。
    3.  初始化 PCB（填入进程名、优先级、起始地址、资源清单、家族关系等）。
    4.  将新进程插入**就绪队列**。

#### B. 进程撤销 (Termination)
*   **引起事件**：
    *   正常结束（任务完成、用户退出）。
    *   异常结束（越界、算术错误、I/O异常）。
    *   外界干预（死锁干预、父进程终止）。
*   **步骤**：
    1.  根据标识符检索 PCB，读出状态。
    2.  若正在执行，立即终止，置调度标志为真。
    3.  若有子孙进程，一并终止（防止成为孤儿/不可控进程）。
    4.  归还资源给父进程或系统。
    5.  从队列移出 PCB，等待信息搜集。

#### C. 进程阻塞 (Block)
*   **引起事件**：请求系统服务（如键盘输入）、启动 I/O 操作、等待新数据/消息。
*   **步骤**：
    1.  调用 `block` 原语，由**运行态 -> 阻塞态**。
    2.  中断 CPU，保存现场到 PCB。
    3.  插入相应事件的**阻塞队列**。
    4.  转进程调度。
*   **UNIX 相关调用**：`Sleep()` (定时阻塞), `Pause()` (等待信号), `Wait()` (等待子进程)。

#### D. 进程唤醒 (Wakeup)
*   **时机**：被阻塞进程期待的事件出现（如 I/O 完成中断、收到消息）。
*   **动作**：
    *   由中断处理程序或发送进程执行。
    *   将进程由**阻塞态 -> 就绪态**。
    *   插入**就绪队列**。

#### E. 进程挂起 (Suspend) & 激活 (Activate)
*   **挂起原因**：
    *   **实时系统**：根据需要将进程挂起一段时间（活动 -> 静止就绪/静止阻塞）。
    *   **分时系统**：内存交换（Swapping），将进程从内存换到外存（不被调度）。
*   **激活/解挂**：
    *   当挂起原因解除，调用解挂原语。
    *   由**静止状态 -> 活动状态**。
    *   若变为活动就绪，通常立即触发调度。

## 2. 进程执行与切换 (Process Execution & Switching)

### 2.1 执行模式
*   **用户态 (User Mode)**：运行用户程序。
*   **核心态 (Kernel Mode)**：运行操作系统核心程序。
*   **切换**：通过系统调用或中断从用户态转入核心态。

### 2.2 进程切换过程 (Context Switch)
指进程让出处理机，处理机转去运行其它进程的过程。
1.  **保存上下文**：保存当前进程的 PC, PSW, 寄存器等现场信息到其 PCB。
2.  **修改当前 PCB**：状态由“运行”改为其他（如就绪、阻塞）。
3.  **选择新进程**：按调度算法选择下一个进程。
4.  **修改新 PCB**：状态由“就绪”改为“运行”。
5.  **修改存储管理数据**：如内存起始地址。
6.  **恢复上下文**：恢复新进程的寄存器、PC 等，开始执行。

### 2.3 进程上下文组成
1.  **用户级上下文**：程序、数据、用户栈。
2.  **寄存器级上下文**：CPU 现场（PC, PSW, 栈指针, 通用寄存器）。
3.  **系统级上下文**：PCB、核心栈。

## 3. 进程调度 (Process Scheduling)

### 3.1 调度的层次
1.  **高级调度 (作业调度)**：选取输入井中的作业，创建根进程。控制并发度。
2.  **中级调度 (内存调度)**：决定哪些进程调入/调出内存（Swapping）。
3.  **低级调度 (进程调度)**：选取就绪进程占用 CPU。**最频繁发生**。
4.  **I/O 调度**：选取 I/O 请求执行。

### 3.2 调度方式
*   **非剥夺 (Non-preemptive)**：进程主动放弃 CPU 时才调度。简单，开销小。用于批处理。
*   **剥夺 (Preemptive)**：系统可根据原则强行剥夺 CPU。用于分时、实时系统。

### 3.3 引起调度的因素 (时机)
1.  **进程主动放弃**：
    *   执行完毕。
    *   发出 I/O 请求。
    *   等待事件/资源。
2.  **有新进程就绪 (支持剥夺)**：
    *   中断处理结束（如 I/O 完成），阻塞进程变就绪。
    *   资源释放，等待进程变就绪。
    *   其他原因导致进程就绪。
3.  **时间片到期 (支持剥夺)**：
    *   时钟中断，时间片用完。
    *   优先级发生变化。

**关键点**：在中断处理结束或自陷处理结束返回用户态前，若**请求调度标志**置位，则进行调度。

## 4. 进程调度算法 (Scheduling Algorithms)

### 4.1 评价指标
*   **吞吐量**：单位时间完成作业数。
*   **CPU 利用率**。
*   **周转时间** $T_i = \text{完成时刻} - \text{提交时刻}$。
*   **平均周转时间**。
*   **等待时间**。
*   **响应时间** (分时系统重要指标)。

### 4.2 常见算法

| 算法                   | 描述                                              | 优点                      | 缺点                             | 适用场景     |
| :------------------- | :---------------------------------------------- | :---------------------- | :----------------------------- | :------- |
| **先来先服务 (FCFS)**     | 按到达顺序排队                                         | 简单、公平、可靠                | 平均等待时间长，有利于长作业，不利于短作业 ("护航效应") | 批处理      |
| **最短作业优先 (SJF/SPF)** | 选择预计运行时间最短的                                     | **平均等待时间最短**，提高吞吐量      | 对长作业不利，可能"饥饿"；难以准确预估运行时间       | 批处理      |
| **最短剩余时间优先 (SRTF)**  | SJF 的剥夺版本                                       | 更优的平均等待时间               | 同上，且切换开销大                      | -        |
| **高响应比优先 (HRN)**     | $R = 1 + \frac{\text{等待时间}}{\text{运行时间}} R$ 大的先 | 兼顾长短作业，避免饥饿             | 计算复杂                           | 批处理      |
| **优先级调度**            | 按优先级高低执行                                        | 灵活，可强调紧迫性               | 静态优先级可能导致低优先级"无穷阻塞/饥饿"         | 通用       |
| **时间片轮转 (RR)**       | 每个进程运行一个固定时间片                                   | **公平性好，响应快**            | 时间片过大退化为 FCFS；过小增加切换开销         | **分时系统** |
| **多级反馈队列 (MLFQ)**    | 设置多个队列，不同优先级和时间片                                | **目前最通用、灵活**；平衡响应时间和吞吐量 | 算法复杂，参数调整困难                    | 通用/现代 OS |

#### 多级反馈队列 (MLFQ) 详解
*   设置多条就绪队列。
*   高优先级队列时间片短，低优先级队列时间片长。
*   新进程进入最高优先级队列。
*   若在一个时间片内未完成，降级到下一级队列。
*   若在低级别队列等待过久，可适当升级（防止饥饿）。

### 4.3 实时系统调度
*   **时钟驱动法**：硬实时，任务固定，周期性调度。
*   **加权轮转法**：根据权重分配 CPU 时间比例。

## 5. 作业与进程的关系

*   **作业 (Job)**：用户一次独立的使用过程。
*   **进程 (Process)**：资源分配单位，任务运行的实体。
*   **关系**：
    *   一个作业至少包含一个进程。
    *   **批处理系统**：作业调度程序选中作业 -> 创建根进程 -> 根进程解释作业控制语言 -> 根据需要创建子进程。
    *   **分时系统**：用户登录 -> 生成一个终端进程（运行命令解释器）-> 执行命令时可创建子进程。

## 6. 线程 (Threads)

### 6.1 引入原因
*   **进程间通信开销大**：进程有独立地址空间，数据交换需内核支持。
*   **并发性需求**：多核 CPU 发展需要加速单个作业内部的并行执行。
*   **轻量级**：线程共享进程资源，创建/切换/通信开销远小于进程。

### 6.2 线程定义与特点
*   **定义**：线程是**处理机调度的基本单位**；进程是**资源分配的基本单位**。
*   **结构**：
    *   **共享**：进程地址空间、代码段、数据段、打开文件、全局变量。
    *   **私有**：线程 ID、寄存器组、程序计数器 (PC)、栈 (Stack)、线程局部存储。
*   **优势**：
    1.  创建/撤销开销小。
    2.  切换迅速（只需保存少量寄存器上下文）。
    3.  通信效率高（直接读写共享内存）。
    4.  并发度高（充分利用多核）。

### 6.3 线程实现模型

| 模型 | 描述 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **用户级线程 (ULT)** | 内核 unaware，由用户库管理 | 切换无需陷入内核，速度快；可移植 | 一个线程阻塞，整个进程阻塞；无法利用多核并行 |
| **核心级线程 (KLT)** | 内核直接管理线程 | 一个线程阻塞不影响其他；可利用多核 | 切换需陷入内核，开销较大 |
| **两级组合 (Hybrid)** | 用户线程映射到核心线程 (如 Solaris LWP) | 结合两者优点 | 实现复杂 |

*   **Linux**: 线程被视为**轻量级进程 (LWP)**，使用 `clone()` 系统调用。
*   **Windows**: 使用 `CreateThread()`。

---

## 7. 总结重点
1.  **原语**是原子操作，用于进程控制。
2.  **进程切换**涉及保存/恢复上下文，开销较大。
3.  **调度算法**的选择取决于系统目标（吞吐量 vs 响应时间）。
4.  **多级反馈队列**是现代通用操作系统常用的调度算法。
5.  **线程**提高了并发性和效率，是调度的基本单位，而进程仍是资源分配单位。
