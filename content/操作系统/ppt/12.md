根据提供的《操作系统 Lecture 12: 存储管理（3）》课件内容，以下是关于**页面替换策略**及**多级页表**的详尽整理：

---

### 一、 页面置换策略概述

#### 1. 虚存的作用
*   解决主存空间不足的问题。
*   让更多的进程并发运行，提高系统的吞吐率。

#### 2. 页面置换算法的核心
*   **定义**：决定在需要调入页面时，选择内存中哪个物理页面被置换出去。
*   **出发点**：希望把未来不再使用的或者短时期内较少使用的页面调出。
*   **评价标准**：
    *   缺页发生频率少。
    *   必须防止系统发生**抖动（Thrashing）**。
    *   算法本身的复杂度小。

#### 3. 颠簸/抖动 (Thrashing)
*   **现象**：页面在内存与外存之间频繁调度，以至于调度页面所需时间比进程实际运行的时间还多，导致系统效率急剧下降甚至崩溃。
*   **主要原因**：
    *   页面淘汰算法不合理。
    *   分配给进程的物理页面数太少。

#### 4. 基本概念
*   **驻留集 (Resident Set)**：进程当前在内存中的合法页集合。
*   **访问串 (Reference String)**：进程访问虚空间的地址踪迹（通常简化为页号序列）。
    *   *例*：访问地址 0100, 0432... 若页面大小为100，则访问串简化为 1, 4...

#### 5. 策略分类
*   **驻留集大小固定的局部置换策略**：FIFO, OPT, LRU, CLOCK。
*   **驻留集大小可变的全局置换策略**：WS (Working Set), SWS (Sampled Working Set)。

---

### 二、 固定驻留集大小的局部置换策略

#### 1. FIFO (First-In First-Out, 先进先出)
*   **规则**：替换最早进入内存的页。
*   **特点**：
    *   实现方便，不需要额外硬件支持。
    *   效果不好。
    *   存在 **Belady 奇异 (Belady's Anomaly)**：即随着驻留集（分配帧数）增大，页故障数反而可能增加，不满足单调性。
*   **Belady 奇异示例**：访问串 `1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5`，在驻留集为 3 和 4 时可能出现异常。

#### 2. OPT (Optimal Replacement, 最佳置换算法)
*   **规则**：淘汰**下次访问距当前最远**的那些页中序号最小的页（即未来最长时间内不会被使用的页）。
*   **特点**：
    *   是理论上最优的固定驻留集置换策略。
    *   **不可实现**：因为需要预先得知整个未来的访问串序列。
    *   **用途**：仅作为一种衡量标准，用于评估其他可行策略的性能。
    *   对任意访问串均有最小的时空积。

#### 3. LRU (Least Recently Used, 最近最少使用)
*   **规则**：淘汰**上次使用距当前最远**的页（即过去最长时间内未被使用的页）。
*   **性质**：**栈算法 (Stack Algorithm)**。
    *   定义：若 $S(m, t) \subseteq S(m+1, t)$，则为栈算法。
    *   意义：LRU **没有 Belady 奇异**。因为驻留集增大时，原有的页依然保留，新加入的页只会减少缺页，不会增加。
*   **特点**：
    *   效果适中（接近 OPT）。
    *   实现费用高，需要硬件配合。
*   **实现方法之一**：
    *   给每个页帧设一个计数器。
    *   每访问一页，该页计数器清 0，其余页帧计数器加 1。
    *   淘汰计数值最大的页帧。

#### 4. CLOCK (时钟页面置换算法)
*   **思想**：基于 LRU 思想的近似算法。
*   **机制**：
    *   硬件在页面被访问时，设置页表项中的**访问位 (Use Bit)** 为 1。
    *   所有页面构成一个循环链表，指针像时钟一样移动。
    *   **淘汰过程**：
        1. 检查指针指向的页面。
        2. 若访问位为 1：清除访问位（置 0），指针移向下一页。
        3. 若访问位为 0：淘汰该页面，并将新页面调入此处。
*   **特点**：实用的页面置换算法，兼顾性能与实现成本。

#### 5. NRU (Not Recently Used, 最近未使用)
*   **思想**：兼顾 FIFO 和 LRU。
*   **机制**：
    *   页表项增加一位**使用位**。
    *   硬件每访存一次将对应页的使用位置 1。
    *   操作系统定时将所有使用位清 0。
    *   淘汰时任选一个使用位为 0 的页（表示在 OS 清 0 周期内没被使用过）。
*   **优化**：尽量避免淘汰被修改过的页。
*   **淘汰优先级**（使用位 U, 修改位 M）：
    1.  U=0, M=0 (最佳)
    2.  U=0, M=1
    3.  U=1, M=0
    4.  U=1, M=1 (最差)

### 三、 可变驻留集大小的全局置换策略

#### 1. 引入原因
*   **工作集 (Working Set)**：一段时间内程序访存有局部性，这些与局部性相关的页面集合。
*   **问题**：
    *   若驻留集 < 工作集 $\rightarrow$ 抖动。
    *   若驻留集 > 工作集 $\rightarrow$ 浪费内存。
    *   工作集是动态变化的，因此驻留集大小也应动态调整。

#### 2. WS (Working Set, 工作集算法)
*   **规则**：若驻留集中的某页有 $\Delta$ 个访问间隔没被访问，则将其淘汰。
*   **实现方式 1（开销大）**：
    *   每页设计数器。每访存一次，其他页计数器+1，当前页清 0。淘汰计数器等于 $\Delta$ 的页。
    *   *缺点*：每次访问都要更新所有计数器，开销太大，不实用。
*   **实现方式 2（改进）**：
    *   记录每页最后访问时间（硬时钟值）。
    *   定时检查：若 `当前时钟值 - 页表项时钟值 > Δ`，则淘汰。

#### 3. SWS (Sampled Working Set, 采样工作集算法)
*   **规则**：定时检查计时器，淘汰计时器值大于等于 $\Delta$ 的页面。
*   **实用操作系统实现 (Windows NT, Linux)**：
    *   采用 **SWS + 淘汰页数据延迟清除** 策略。
    *   **两个队列**：自由链表 (Free List) 和 修改链表 (Modified List)。
    *   **流程**：
        1.  定时做页淘汰（SWS）。
        2.  淘汰时不立即抹去数据，根据页面是否被修改，挂入自由链或修改链。
        3.  若修改链过长或自由链过短，将修改链中的页面回写磁盘后移至自由链。
        4.  **Paging In 时**：直接从自由链头部取页帧覆盖数据。
        5.  **重用机制**：若自由链/修改链中的页面再次被访问，将其从链中摘除，恢复通过页表项访问。

### 四、 多级页表与地址映射

#### 1. 多级页表提出的问题
*   **背景**：32位逻辑地址，页大小 4KB ($2^{12}$)。
*   **计算**：
    *   页表项数 = $2^{32} / 2^{12} = 2^{20}$ 个。
    *   若每项 4B，页表大小 = $4MB$。
    *   **缺点**：要求页表连续存放，占用大量连续物理空间。
*   **解决**：将页表再分页（多级页表）。

#### 2. 两级页表结构
*   **外部页表 (Page Directory)**：指向二级页表的指针。
*   **内部页表 (Page Table)**：指向物理页框。
*   **地址变换过程**：
    1.  逻辑地址分为：外部页号 ($P_1$)、内部页号 ($P_2$)、页内偏移 ($d$)。
    2.  查外部页表得到二级页表基址。
    3.  查二级页表得到物理块号。
    4.  拼接物理块号与页内偏移得到物理地址。

#### 3. 例题分析
*   **题目**：二级页表，按字节编址，页大小 $2^{10}$ B，页表项 2 B。逻辑地址空间大小为 $2^{16}$ 页。求页目录表至少包含多少个表项？
*   **解析**：
    *   逻辑地址空间总页数 = $2^{16}$。
    *   一个二级页表能管理的页数 = 页大小 / 页表项大小 = $2^{10} / 2^1 = 2^9$ 页。
    *   所需二级页表个数 = 总页数 / 单个二级页表容量 = $2^{16} / 2^9 = 2^7 = 128$。
    *   每个二级页表对应页目录表中的一个表项。
    *   **答案**：128 (选项 B)。

#### 4. 哈希页表 (Hashed Page Table)
*   **适用场景**超过 32 位地址空间（如 64 位系统）。
*   **结构**：
    *  以逻辑页号作为哈希值。
    *   哈希表每个表项是一个链表头。
    *   链表元素包含：逻辑页号、物理块号、下一元素指针。
*   **映射过程**:
    1.  逻辑页号哈希定位到表项。
    2.  遍历链表，比较逻辑页号。
    3.  匹配成功则获取物理块 a号；否则继续查找直到链表结束。

### 五、 页面大小的确定

*   **权衡因素**：
    *   **页面越大**：无用程序装入主存越多，**内部碎片**严重，但页表较小。
    *   **页面越小**：程序需要的页越多，**页表越大**，但内部碎片小。
*   **最佳页面大小推导**：
    *   设进程平均大小 $s$，页大小 $p$，页表项大小 $e$。
    *   进程所需页数 $\approx s/p$。
    *   页表占用空间 $= (s/p) \times e = se/p$。
    *   平均内部碎片 $= p/2$。
    *   **总开销函数** $C(p) = se/p + p/2$。
    *   求导令其为 0：$dC/dp = -se/p^2 + 1/2 = 0$。
    *   解得最佳页面大小：$p = \sqrt{2se}$。

---

### 六、 总结

*   **局部置换**：FIFO（简单但有Belady奇异）、OPT（理论最优不可实现）、LRU（性能好无奇异但成本高）、CLOCK（LRU的实用近似）。
*   **全局置换**：WS/SWS（动态调整驻留集大小，适应工作集变化，防止抖动）。
*   **页表优化**：多级页表解决连续空间需求，哈希页表适应大地址空间。
*   **页面大小**：需在页表开销和内部碎片之间取得平衡，最佳值为 $\sqrt{2se}$。
