根据提供的《操作系统 - Lecture 8 进程的通信》课件内容，以下是关于进程通信的详尽整理：

### 1. 进程通信概述
*   **定义**：简单来说就是在进程间传输数据（交换信息）。
*   **目的与要求**：掌握进程通信的基本实现方法。
*   **重点与难点**：通讯实现原理。

### 2. 进程通信的分类
根据交换信息量的多少和效率的高低，分为两类：

#### (1) 低级通信
*   **特点**：只能传递状态和整数值（控制信息），如 P、V 操作。
*   **缺点**：
    *   传送信息量小，效率低。
    *   每次通信传递的信息量固定，若传递较多信息需多次通信。
    *   编程复杂：用户需直接实现通信细节，容易出错。

#### (2) 高级通信
*   **特点**：提高信号通信效率，传递大量数据，减轻程序编制复杂度。
*   **三种主要方式**：
    1.  共享内存模式
    2.  消息传递模式
    3.  共享文件模式

### 3. 两种基本进程通讯方法

#### ① 共享存储 (Shared-memory)
*   **原理**：相互通讯的进程拥有共享存储区，通过直接读写共享存储区的变量来交互数据。
*   **同步与互斥**：需要在并发程序设计时安排进入程序，操作系统提供共享存储区及同步互斥工具。
*   **共享内存模式（间接通信）**：
    *   是 UNIX 系统中常被使用的最为快捷有效的方式之一。
    *   **注意**：内存共享区的互斥要通过其它机制实现。
    *   **安全隐患**：数据的发送方不关心谁接收，接收方也不关心谁发送。

#### ② 消息传递 (Message-passing)
*   **原理**：通过操作系统的相应系统调用进行消息传递通讯。

### 4. 消息传递的具体方式

#### (1) 直接通信方式 (Direct Communication)
*   **特点**：点到点的发送。
*   **原语操作**：
    *   `Send(DestProcessName, Message)`
    *   `Receive(SourceProcessName, Message)`
*   **基本思想**：进程在发送和接收消息时直接指明接收者或发送者的进程 ID。
*   **缺点**：必须指定接收进程 ID。
*   **举例 (UNIX 信号机制)**：
    *   Process A: `kill(1040, SIGUSR1);` // 向 1040 号进程发送 SIGUSR1 信号
    *   Process B: `Signal(SIGUSR1, func);` // 注册处理函数，收到信号时执行 func()

#### (2) 间接通信方式 (Indirect Communication)
*   **特点**：以**信箱 (MailBox)** 为媒介进行传递，支持广播。
*   **原语操作**：
    *   `Send(MailBox, Message)`
    *   `Receive(MailBox, Message)`
*   **信箱命名法基本思想**:
    *   系统为每个信箱设一个消息队列。
    *   消息发送和接收都指向该消息队列。
*   **优缺点**：
    *   **缺点**：必须有一个通讯双方共享的逻辑消息队列（如 UNIX 的 PIPE, FIFO 及 IPC 消息传递机制）。使用时需约定打开方式（写/读/读写）。
    *   **优点**：很容易建立双向通讯链（只要对信箱说明为读写打开）。

### 5. 消息传递通信的实现细节

#### 数据结构
*   **消息缓冲区结构**：
    *   Sender’s pid (发送者进程ID)
    *   下一消息链指针
    *   消息正文 (text)
*   **进程控制块 (PCB) 中的相关字段**：
    *   `Hptr`: 消息队列头指针
    *   `mutex`: 互斥信号量 (初值为 1)，用于保护消息队列
    *   `Sm`: 计数信号量 (初值为 0)，用于记录消息队列中的消息数

#### 工作原理与原语实现
系统维护一个空闲缓冲区池，每个进程有一个消息缓冲队列。

*   **Send(&A) 原语逻辑**：
    1.  `New(&p)`: 从空缓冲区队列获取一个 buffer `p`。
    2.  置 `sender’s pid`。
    3.  将消息送入 buffer `p`。
    4.  获得接收者 A 的 PID。
    5.  `P(mutex)`: 申请互斥锁。
    6.  将 buffer `p` 挂入接收者 A 的消息队列。
    7.  `V(Sm)`: 增加消息计数信号量。
    8.  `V(mutex)`: 释放互斥锁。
    *(注: new/dispose 函数访问空缓冲区池也需要互斥)*

*   **Receive(&A) 原语逻辑**：
    1.  `P(Sm)`: 等待消息（若队列为空则阻塞）。
    2.  `P(mutex)`: 申请互斥锁。
    3.  从本进程的消息队列取一个 buffer `f`。
    4.  `V(mutex)`: 释放互斥锁。
    5.  从 buffer `f` 中取得消息正文送入 A，并得到 sender’s pid 作为返回值。
    6.  `dispose(f)`: 释放 buffer `f` 回空缓冲队列。

#### 关键问题解答
*   **一个进程如何和一个消息队列关联？** 通过 PCB 中的 `Hptr` 指向该进程的消息队列。
*   **临界资源是什么？** 消息队列本身以及空闲缓冲区池。
*   **OS 如何实现同步互斥？** 使用信号量 `mutex` (互斥) 和 `Sm` (同步/计数)。

#### 提高并发性
*   将原来的生产者-消费者问题的共享资源（缓冲池）分解。
*   变为 $n$ 个满缓冲区队列（对应 $n$ 个进程）+ 1 个空缓冲区队列，从而减少竞争。

### 6. 管道通信 (Pipe)

*   **定义**：一种信息流缓冲机构。在 UNIX 系统中基于文件系统，在内核中通过文件描述符表示。
*   **组织方式**：先进先出 (FIFO)。
*   **实现方法**：
    *   `int pipe(int fd[2]);` 创建管道。
        *   `fd[0]`: 读取端
        *   `fd[1]`: 写入端
    *   `write(int handle, char* buf, unsigned len)`: 写入信息。
    *   `read(int handle, void* buf, unsigned len)`: 读取信息。

*   **管道的特点**：
    1.  **单向通信信道**：若需双向通信，通常需定义两个管道。
    2.  通过系统调用 `read()` 和 `write()` 进行读写操作。

*   **管道的分类**：
    1.  **匿名管道 (Anonymous Pipe)**：
        *   只适用于**父子进程**之间通信。
        *   原理：把信息从一个进程的地址空间拷贝到另一个进程的地址空间。
    2.  **命名管道 (Named Pipe / FIFO)**：
        *   有自己的名字和访问权限限制，像一个文件一样。
        *   可用于**不相关进程**间的通信。
        *   进程通过使用管道的名字来获得管道。
