根据提供的《操作系统 Lecture 16: 设备管理（2）》课件内容，以下是对该章节知识点的详尽整理：

### 一、 I/O 软件系统的组成与设计目标

#### 1. I/O 软件的组成部分
*   **I/O 交通管制程序**：负责各 I/O 设备之间的协调工作。
*   **I/O 调度程序**：负责设备的分配和调度。
*   **I/O 设备处理程序**：负责每类设备的具体操作。

#### 2. I/O 软件的设计思想与目标
*   **基本思想**：**分层构建**。较低层软件为较高层服务，使高层软件独立于硬件，为用户提供统一接口。
*   **设计目标**：
    1.  **设备独立性 (Device Independence)**：
        *   除底层外，其他软件不依赖具体硬件。
        *   **优点**：降低耦合性，增强适应性；方便用户使用。
    2.  **设备统一命名**：
        *   采用预先设计的统一逻辑名称（如序列字符串或整数）。
        *   名称不随物理设备变化而变化（例如 UNIX 中的 `/dev/fd0`）。
    3.  **出错处理**：
        *   尽可能在接近硬件层处理错误（可重试）。
        *   仅当低层无法解决时，才上报给高层。
    4.  **缓冲技术**：匹配数据到达率和离去率，提高吞吐量。
    5.  **设备分配**：处理共享设备（如磁盘）和独占设备（如打印机）的分配。

### 二、 I/O 软件的层次结构

I/O 软件通常分为四层，从上到下依次为：

#### 1. 用户层 I/O (User-level I/O)
*   **特点**：与设备控制细节无关，不直接打交道。
*   **功能**：将设备看作逻辑资源，提供简单的 I/O 函数（打开、关闭、读、写）。
*   **示例**：C 库函数 `fopen()`, `fread()`, `fwrite()`, `fclose()`, `printf()` 等。

#### 2. 设备无关的 I/O 软件 (Device-independent I/O Software)
*   **核心功能**：执行适用于所有设备的公共 I/O 功能，向上提供统一系统调用接口。
*   **主要任务**：
    *   **设备命名及映射**：将逻辑名转换为物理名/驱动程序（如 Linux 中通过 inode 找到主设备号，进而找到驱动）。
    *   **设备访问控制**：防止无权访问（如 UNIX 利用文件权限位 "rw" 保护）。
    *   **提供统一块尺寸**：向上层提供等长的逻辑块，屏蔽底层差异。
    *   **系统调用转调及数据缓冲**：
        *   缓和 CPU 与 I/O 速度矛盾。
        *   减少中断次数。
        *   以空间换时间。
    *   **存储设备块分配**：查找并分配空闲块。
    *   **设备的分配和释放**。
    *   **出错处理**：
        *   操作故障：由驱动程序处理。
        *   非操作故障（如磁盘损坏）：由本层处理并向上报错。

#### 3. 设备驱动程序 (Device Drivers)
*   **定义**：包含所有与特定设备相关的代码。
*   **功能**：接收来自上层的 I/O 请求，排入队列或直接执行。
    1.  设备初始化。
    2.  启动数据传输。
    3.  中断处理。
*   **具体职责**：
    *   控制和监督 I/O 控制器，进行错误处理。
    *   处理设备相关操作（排队、挂起、唤醒）。
    *   缓冲区管理。
    *   底层硬件细节处理（如磁盘的磁道、柱面、磁头运动等，只有驱动知道这些细节）。
*   **特点**：与硬件结构密切联系，不同硬件驱动结构不同；同类设备可提供通用驱动或厂家专用驱动。
*   **工作过程**:
    1.  将抽象要求转为具体要求（如第几块）。
    2.  检查请求合法性。
    3.  检查设备状态。
    4.  传送参数。
    5.  设置工作方式。
    6.  启动 I/O 设备。
*   **接口函数**：
    *   **初始化函数**：系统启动或加载时执行，登记接口。
    *   **卸载函数**：动态卸载时执行。
    *   **申请/释放设备函数**：独占或共享占用，初始化或清理。
    *   **I/O 操作函数**：启动 I/O 或将请求包排入队列。
    *   **中断处理函数**：I/O 完成后被调用，唤醒等待进程，启动下一个请求。

#### 4. 中断处理程序 (Interrupt Handlers)
*   **目的**：解决高速 CPU 与低速 I/O 之间的矛盾，提高效率。
*   **机制**：
    1.  进程发起 I/O 后阻塞。
    2.  I/O 完成，控制器发中断。
    3.  CPU 响应中断，执行中断处理程序。
    4.  检查状态：
        *   **正常**：唤醒等待进程；若有新请求则启动。
        *   **出错**：重试或向上报告错误。
    5.  中断返回，恢复进程或进行进程调度。

### 三、 设备管理的数据结构

1.  **设备表/部件控制块**：描述每个设备/部件的类型、标识符、状态、当前使用者进程ID等。
2.  **设备队列**：通过指针将相同物理属性的设备连成队列，方便分配管理。
3.  **I/O 请求包 (动态数据结构)**：
    *   每当进程发出块 I/O 请求时建立。
    *   包含请求参数、系统缓冲区地址等。
    *   I/O 完成后删除。
4.  **I/O 队列**：如请求包队列，用于排队等待处理的 I/O 请求。

---

### 四、 缓冲技术 (Buffering)

*   **目的**：缓和 CPU 与 I/O 设备间速度不匹配的矛盾，减少中断次数，实现预读和滞后写。
*   **类型**：
    1.  **单缓冲**：在内存系统空间分配一个缓冲区。
    2.  **双缓冲**：允许“用户区<->缓冲区”交换数据与“缓冲区<->外设”交换数据并行进行。
    3.  **循环缓冲**：引入缓冲池，采用生产者/消费者模型循环使用有限缓冲区。适合具有重复性及阵发性 I/O 的进程。

---

### 五、 读文件 I/O 操作步骤详解

1.  用户进程发出读文件系统调用。
2.  **设备独立 I/O 软件**检查参数；检查高速缓存，若有则直接读取；若无，转步骤 3。
3.  执行物理 I/O：逻辑名转物理名，检查权限，请求排队，**阻塞用户进程**。
4.  **核心执行设备驱动程序**：分配缓冲区，向设备控制寄存器发启动命令。
5.  **设备控制器**控制设备进行数据传输。
6.  传输完成（DMA方式），硬件产生**中断**。
7.  CPU 响应中断，执行**中断处理程序**：
    *   检查状态。
    *   若出错：重试或报错。
    *   若成功：数据传给用户空间，**唤醒等待进程**放入就绪队列。
8.  用户进程被调度，从断点恢复执行。

### 六、 常见存储外设

1.  **磁盘 (Disk)**：
    *   **数据组织**：盘面 -> 同心圆磁道 -> 扇区（基本传输单位）。
    *   **物理特性**：单磁头或多磁头活动头盘。
    *   **结构**：主轴、动臂、盘片、柱面、磁道、读写头。
2.  **光学存储**：CD-ROM, CD-R/RW, DVD 等。
3.  **Flash Memory**：无机械运动，但写入前需擦除。

### 七、 磁盘调度 (Disk Scheduling)

#### 1. 磁盘访问时间构成
*   **寻道时间 (Seek Time)**：磁头移动到指定磁道的时间（几十毫秒量级，优化重点）。
*   **延迟时间 (Rotational Latency)**：等待指定扇区旋转到磁头下的时间。
*   **传输时间 (Transfer Time)**：数据读写时间。
*   **目标**：着重减少寻道时间和延迟时间。

#### 2. 减少寻道时间的调度算法

| 算法名称 | 英文缩写 | 描述 | 优缺点 |
| :--- | :--- | :--- | :--- |
| **先来先服务** | FCFS |按请求到达顺序服务 | 简单、公平；但平均寻道时间长，性能差。 |
| **最短寻道时间优先** | SSTF | 总是选择距离当前磁头最近的磁道请求 | 性能较好；但可能导致远端请求**“饥饿”(Starvation)**。 |
| **扫描算法** | SCAN (电梯算法) | 磁头沿一个方向移动，处理沿途请求，直到端点后反向 | 解决了饥饿问题；但在两端请求少时效率略低。 |
| **循环扫描** | C-SCAN | 磁头单向移动处理请求，到达一端后立即快速返回另一端（不处理返回途中的请求） | 提供更均匀的等待时间。 |
| **LOOK / C-LOOK** | LOOK / C-LOOK | SCAN/C-SCAN 的改进版，磁头只需移动到最远请求处即可折返，无需到物理端点 | 减少了不必要的移动距离。 |

#### 3. 避免“磁臂粘着” (Arm Stickiness) 的算法
*   **现象**：某些进程高频访问同一磁道，垄断磁盘，导致其他请求长时间等待。
*   **N-Step-SCAN**：
    *   将请求队列分成若干长度为 N 的子队列。
    *   按 FCFS 处理子队列，每个子队列内部按 SCAN 处理。
    *   新请求放入其他队列，避免当前队列被新的高频请求插队。
    *   N=1 时退化为 FCFS；N 很大时接近 SCAN。
*   **FSCAN**：
    *   N-Step-SCAN 的简化版，只分两个队列。
    *   队列1：当前所有请求，按 SCAN 处理。
    *   队列2：扫描期间产生的新请求，推迟到下一次扫描处理。

#### 4. 减少延迟时间的方法
*   **扇区交替编号**：考虑到数据处理时间，扇区物理上不连续排列。
*   **盘面错开命名**：不同盘面的起始扇区位置错开。

### 八、 磁盘阵列 (RAID - Redundant Array of Independent Disks)

#### 1. RAID 的基本思想
*   **提高可靠性**：通过冗余（如镜像）。
*   **提高性能**：通过并行性（如数据分条分布到多盘）。

#### 2. 常见 RAID 级别

*   **RAID 0 (条带化)**：
    *   无冗余，数据分块分布在多个磁盘。
    *   **优点**：高性能（并行读写）。
    *   **缺点**：无容错能力，任一磁盘损坏数据全丢。
*   **RAID 1 (镜像)**：
    *   数据完全复制到另一个磁盘。
    *   **优点**：高可靠性，读取速度快。
    *   **缺点**：存储空间利用率低（50%）。
*   **RAID 0+1**：
    *   先做 RAID 0（条带化），再做 RAID 1（镜像）。
    *   若一个磁盘失效，其所在整个 RAID 0 组不可用，依赖另一组镜像。
*   **RAID 1+0 (RAID 10)**：
    *   先做 RAID 1（镜像对），再做 RAID 0（条带化镜像对）。
    *   **优势**：比 RAID 0+1 更健壮。单个磁盘失效只影响该镜像对，其他镜像对仍可用。
*   **RAID 2**：
    *   位级条带化，使用海明码（ECC）进行错误校验。需要多个校验盘。现已很少使用。
*   **RAID 3**：
    *   字节级条带化，使用**单独的一个奇偶校验盘**。
    *   控制器可检测扇区错误，用一个奇偶位即可校验。
*   **RAID 4**：
    *   块级条带化，使用**单独的一个奇偶校验盘**。
    *   **缺点**：校验盘成为瓶颈（过度使用）。
*   **RAID 5**：
    *   块级条带化，**分布式奇偶校验**。
    *   数据和奇偶校验信息均匀分布在所有 N+1 个磁盘上。
    *   **优点**：避免了 RAID 4 的校验盘瓶颈，兼顾性能与可靠性，允许一块磁盘失效。
*   **RAID 6**：
    *   类似 RAID 5，但使用 **P+Q 双重冗余**（如 Reed-Solomon 码）。
    *   **优点**：允许**两块**磁盘同时失效而不丢失数据。
