## 二、什么是操作系统 (核心概念)

### 1. 操作系统的目的
*   **方便用户**：提供友好的交互界面。
*   **提高资源利用率**：合理调度硬件资源。

### 2. 操作系统所在层次
*   **层次结构**：`硬件` － `操作系统` － `应用`
*   **定义**：操作系统是**软硬资源的控制中心**。它以尽量合理有效的方法组织多个任务共享使用计算机的各种资源，并提供编程和使用接口。

### 3. 操作系统作为接口 (三种方式)

操作系统向用户提供了三种主要的接口方式来操纵计算机系统：

| 接口方式 | 描述 | 适用场景 |
| :--- | :--- | :--- |
| **(1) 命令方式** | OS 提供一组联机命令 (语言)，用户通过键盘输入命令直接操纵系统。 | 早期系统、服务器管理、脚本编写 |
| **(2) 系统调用方式** | OS 提供一组系统调用，用户在应用程序中通过调用这些接口来操纵计算机。 | 程序员开发应用程序 |
| **(3) 图形、窗口方式** | 用户通过屏幕上的窗口和图标来操纵计算机系统和运行程序。 | 现代个人用户 (GUI) |

#### 系统层次结构示意图 (文本表示)

```text
用户层
  ├── 用户
  ├── 应用程序 (编译器、数据库管理器、编辑器等)
  ├── 接口层 (系统调用、命令、图标、窗口)
  ├── 操作系统 (内核 + 系统调用库 + 程序管理器)
  └── 硬件层 (计算机硬件)
```

> **易考点**：能够区分三种接口方式及其应用场景；理解操作系统在硬件与应用之间的中间层地位。

## 三、计算机系统软件构成

### 1. 狭义与广义操作系统

*   **狭义操作系统**：仅包含 **操作系统内核** (运行在内核态)。
*   **广义操作系统**：
    *   操作系统内核
    *   用户态运行的命令解释器（程序管理器）
    *   系统调用库
    *   许多提供系统常用功能的实用程序、库程序 (如 C 库、数学库、图形库、窗口库等)

### 2. 软件与硬件的存放关系
*   **存放位置**：操作系统及其他软件通常放在**磁盘**上。
*   **运行位置**：要运行的时候加载到**内存**里。

#### 系统软件层次结构
*   **顶层**：应用 (数据库、编辑器等)
*   **中间层**：命令解释器、系统调用库、程序管理器
*   **核心层**：操作系统内核
*   **底层**：硬件层 (处理机、主存 ROM、总线桥、外设、总线)

> **易考点**：狭义 OS 与广义 OS 的区别（是否包含库函数、命令解释器等用户态程序）。

## 四、操作系统面面观 (三大视角)

### 1. 操作系统作为特殊子程序
*   **核心功能**：操作系统内核为用户层程序提供**系统调用接口**。
*   **特性**：
    *   系统调用可看成是**特殊的公共子程序调用**。
    *   可被任意用户层程序调用，实现公共功能。
    *   用户无需重复编写底层功能代码，只需调用即可。

### 2. 操作系统作为资源管理者
为了充分发挥硬件资源作用，必须允许多用户任务**同时使用**计算机，减少资源闲置。
*   **示例**：当一个任务读写磁盘时，另一个任务可在处理机上运行（处理机、内存、磁盘同时忙）。

#### 资源共享的两种方法
| 共享方法 | 针对资源 | 描述 |
| :--- | :--- | :--- |
| **空分共享** | 存储资源 | 存储空间被多个用户（进程）共同以**分割**的方式占用。 |
| **时分共享** | 处理机、I/O 控制器、网卡等 | 多个用户任务**分时**使用该资源。 |

#### 时分共享的两种使用方法
| 使用方式 | 描述 | 典型示例 |
| :--- | :--- | :--- |
| **独占式共享** | 多任务分时共享，但某任务占用后执行**完整周期**的操作。使用前申请，使用后释放。 | **打印机**：打印一个完整文件期间独占，打印完释放。 |
| **分时式共享** | 任务占用资源无需使用逻辑上的完整周期，可随时被剥夺。 | **CPU**：程序可随时被剥夺 CPU，保存现场后下次继续运行。<br>**磁盘 I/O**：不同任务的 I/O 请求可交错执行。 |

> **易考点**：
> 1. 空分共享与时分共享的区别。
> 2. 独占式共享与分时式共享的区别（重点理解打印机与 CPU 的例子）。

### 3. 提供程序运行机制 (进程机制)
*   **背景需求**：
    1.  为程序及数据安排存放空间。
    2.  提高资源利用率，增加并发度（多程序分时占用处理机）。
    3.  支持程序未运行完就让另一个程序占用 CPU（需保存上一个程序的运行现场）。
*   **解决方案**：**进程机制**。
    *   **描述**：对实现各种用户意图的各个程序的执行进行描述和控制。
    *   **内容**：描述程序执行的状态、现场、标识等各种信息。
    *   **功能**：有选择地调度某个程序占用 CPU 运行。

## 五、课外知识：操作系统生命期

### 1. 操作系统开发与升级

#### (1) 交叉开发 (Cross Development)
*   **场景**：开发新操作系统，或将操作系统移植到新型处理机的机器。
*   **环境建立**：
    *   **目标机 (Target)**：操作系统将运行的机器。
    *   **工作机 (Host)**：开发操作系统的机器。
    *   **交叉编译**：在工作机上将源代码编译成目标机可运行代码。
    *   **模拟调试工具**：在工作机上模拟执行目标机代码。
    *   **内核生成环境**：工作机上编译链接工具。
*   **流程**：在交叉环境开发 -> 到目标机运行。

#### (2) 增量开发 (Incremental Development)
*   **场景**：在已有操作系统上作内核、驱动程序更新和升级。
*   **工具**：
    *   编译系统
    *   内核调试器
    *   内核生成环境
    *   打包工具 (生成 driver, boot 盘，系统安装盘)

### 2. 操作系统的安装
1.  由 ROM 程序从安装介质加载操作系统 loader。
2.  从安装介质加载并启动操作系统，检测系统配置，运行安装程序 (scripts)。
3.  运行分区程序对硬盘进行分区，将操作系统内核及所选软件包拷入硬盘。
4.  对系统部件进行设置。
5.  (重新生成操作系统内核放在盘上)。
6.  重新启动（这时启用硬盘上的内核）。
7.  (进一步配置系统，重新启动)。

#### 驱动程序安装过程示例
1.  驱动程序安装到硬盘。
2.  在系统配置文件中登记 (可能重链内核)。
3.  关机，插硬件，重启。
4.  操作系统初始化时加载驱动程序。
*   **注**：当前系统一般支持 **Plug & Play (即插即用)** 方式，动态带电插入硬件及动态加载驱动程序。

### 3. 系统启动与关机

#### 系统启动过程
1.  **加电**：激活 ROM 程序。
2.  **检测与加载**：
    *   ROM 程序检测系统配置。
    *   加载 MBR 程序。
    *   MBR 程序加载操作系统 Loader。
    *   操作系统 Loader 加载操作系统。
3.  **初始化**：
    *   进入操作系统初始化。
    *   根据系统配置等准备系统表格。
    *   加载 Driver，标识设备。
    *   *(注：对新增加设备，现代系统能在发现新硬件后动态安装并加载 driver)*。
4.  **正常控制**：进入操作系统正常控制阶段。

#### 系统关机过程
1.  **回写数据**：将数据回写入盘。
2.  **外设归初态**：将外设恢复到初始状态。
3.  **杀进程**：Kill 所有进程。
4.  **断电**。

> **易考点**：
> 1. 交叉开发中“工作机”与“目标机”的区别。
> 2. 系统启动的顺序 (ROM -> MBR -> Loader -> OS)。
> 3. 关机时的数据保护操作 (回写数据)。

## 六、总结与复习建议

### 核心知识点梳理
1.  **定义**：操作系统是资源管理者，是硬件与软件的接口。
2.  **接口**：命令、系统调用、图形窗口。
3.  **构成**：内核 (狭义) vs 内核 + 库 + 工具 (广义)。
4.  **管理**：空分共享 (存储) vs 时分共享 (CPU/IO)；独占式 vs 分时式。
5.  **机制**：进程是程序执行的基本单位，用于管理并发和现场保存。
6.  **生命周期**：开发 (交叉/增量)、安装、启动 (ROM->MBR->Loader->OS)、关机。

### 易考题型预测
*   **选择题**：操作系统的定义、接口的分类、狭义与广义的区别、启动流程顺序。
*   **简答题**：简述操作系统作为资源管理者的功能（空分/时分）；简述系统启动的主要步骤。
*   **理解题**：为什么需要进程机制？（为了并发执行和现场保存）。
