# PMBOK 第1-3章 核心知识体系整理

## 第1章 引论 (Introduction)

### 1.1 指南概述与目的
*   **定位**：收录项目管理中被普遍认可为**“良好实践”**的部分。
    *   **普遍认可**：大多数时候适用于大多数项目，获得一致认可。
    *   **良好实践**：能显著提高项目成功的可能性。
*   **核心价值观**：责任、尊重、公正、诚实。
*   **裁剪**：应以恰当组合管理项目的过程，即根据具体情况进行“裁剪”应用。

### 1.2 基本要素

#### 1.2.1 项目及项目管理
*   **项目的定义**：为创造**独特**的产品、服务或成果而进行的**临时性**工作。
*   **项目的三大特点**：
    1.  **独特性**：带来不确定性。
    2.  **临时性**：有明确的起点和终点（不代表持续时间短）。
    3.  **渐进明细**：随着信息越来越详细，估算越来越准确。
*   **可交付成果**：
    *   在某一过程、阶段或项目完成时，必须产出的任何独特并可核实的产品、成果或服务能力。
    *   可以是有形的（产品）或无形的（服务、成果）。
    *   **注意**：项目是临时的，但可交付成果一般不具有临时性（可能在项目结束后依然存在）。
*   **项目收尾条件**达成目标、无法达到目标、缺乏资金等（原则：让你收尾就收尾）。
*   **项目的作用**：驱动组织进行变更，创造**商业价值**。
    *   **商业价值**：特定项目成果为相关方带来的效益（有形、无形或兼有）。
*   **项目管理的定义**：将知识、技能、工具与技术应用于项目活动，以满足项目要求。
*   **制约因素管理**:
    *   包括：范围、质量、进度、成本、资源、风险。
    *   由管理层决定哪一个制约因素最重要。

#### 1.2.2 项目组合、项目集、项目、运营
| 概念 | 关注重点 | 核心逻辑 | 相互关系 |
| :--- | :--- | :--- | :--- |
| **项目集** | 以“正确”的方式开展 | 管理组成部分之间的**依赖关系**，确定最佳管理方法。 | 项目集内的项目彼此依赖或直接相关。 |
| **项目组合** | 开展“正确”的项目集和项目 | **选择要正确**。实现价值最大化，确定优先顺序，分配资源以符合组织战略。项目间不一定依赖或相关。 the |
| **项目** | 执行层面 | 创造独特的可交付成果。 | 服从于项目集或项目组合的战略。 |
| **运营** | 持续运作 | 关注产品的持续生产和/或服务的持续运作。 | 与项目不同，运营是持续的、重复的。 |

*   **共同点**项目组合、项目集和项目均需符合**组织战略**。

#### 1.2.3 生命周期和阶段
*   **项目生命周期类型**：
    1.  **预测型（瀑布型）**：早期确定范围、时间、成本。严格管理范围变更。
    2.  **迭代型**：早期确定范围，但时间和成本随理解深入定期修改。通过重复循环开发产品。
    3.  **增量型**：在预定时间区间内，通过一系列迭代渐进增加功能，产出可交付成果。
    4.  **适应型（敏捷/变更驱动）**：属于敏捷、迭代或增量型。小增量、快速迭代，每次交付最有价值成果。**相关方参与程度最高**。
    *   *注：由项目管理团队确定最适合的生命周期。*
*   **产品生命周期 vs 项目生命周期**：
    *   **产品生命周期**：概念 -> 交付 -> 成长 -> 成熟 -> 衰退。
    *   **项目生命周期**通常包含一个或多个与产品开发相关的阶段（开发生命周期）。
*   **阶段关口（Stage Gate）**：
    *   在阶段结束时进行。 a
    *   **目的**：比较项目绩效/进度与商业文件，做出决定（继续/终止/调整）。
    *   **别名**：阶段审查、阶段门、关键决策点、生死点、杀点、里程碑等。

#### 1.2.4 工作绩效数据、信息和报告
这是一个从原始数据到决策支持文件的转化过程：
1.  **工作绩效数据 (Data)**：执行过程中收集的**原始**观察结果和测量值（最底层）。
2.  **工作绩效信息 (Information)**：结合背景和跨领域关系，对各控制过程收集的数据进行**整合分析后得到的结果**。
3.  **工作绩效报告 (Reports)**：汇编工作绩效信息形成的实物或电子文件，用于制定决策、提出问题或引起关注。

*   **职责分工**：
    *   **项目发起人**：负责项目商业论证文件的制定和维护。
    *   **项目经理**：负责提供建议和见解。

#### 1.2.5 商业论证和效益管理计划
*   **商业论证**：
    *   文档化的经济可行性研究报告。
    *   **作用**：对收益进行有效性论证，是启动后续活动的依据，决定项目**是否值得投资**。
    *   **使用时机**：整个项目生命周期中使用（不仅在启动前）。
*   **效益管理计划**：
    *   描述项目实现效益的**方式**和**时间**。
    *   制定效益的**衡量机制。

#### 1.2.6 项目章程和项目管理计划
*   **项目章程**：
    *   由**项目发起人**发布。
    *   **作用**：正式批准项目成立，授权项目经理动用组织资源。
*   **项目管理计划**：
    *   描述**如何**执行、监督和控制项目的一份文件。

#### 1.2.7 项目成功的标准
*   不同相关方对项目成功的定义和重要因素看法可能不同。
*   **关键动作**：主要相关方和项目经理应**达成共识并予以记录**。

## 第2章 项目运行环境

### 2.1 概述
*   影响项目的两大主要来源：
    1.  **事业环境因素 (EEFs)**
    2.  **组织过程资产 (OPAs)**

### 2.2 事业环境因素 (EEFs)
*   **定义**：项目团队**不能控制**的，将对项目产生影响、限制或指令作用的各种条件。
*   **影响**可能提高或限制灵活性，产生积极或消极影响。
*   **分类**：
    *   **内部 EEFs**：组织文化、结构和治理；基础设施；信息技术软件；员工能力。
    *   **外部 EEFs**：商业数据库；政府或行业标准；物理环境要素（工作环境、天气、制约因素）。

### 2.3 组织过程资产 (OPAs)
*   **定义**：来自执行组织的，可用于执行或治理项目的任何工件、实践或知识。包括以往项目的经验教训和历史信息。
*   **分类及更新主体**：
    1.  **过程、政策和程序**：
        *   内容：指南、标准、模板、供应商清单、合同协议类型、变更控制程序、沟通要求等。
        *   **更新者**：通常**不是**项目工作的一部分，由 **PMO** 或项目以外的职能部门完成。
    2.  **组织知识库**：
        *   内容：配置管理库、财务数据库、测量指标库、**经验教训知识库**、以往项目档案。
        *   **更新者**：在**整个项目期间**结合项目信息而更新。
*   **关键点**：经验教训或组织过程资产应该**随时都要进行总结 in**。

### 2.4 组织系统

#### 2.4.1 组织结构类型对比
| 组织结构 | 项目经理权力 | 资源全职/兼职 status | 优点 | 缺点 |
| :--- | :--- | :--- | :--- | :--- |
| **职能型** | 很小 / 没有 | 兼职 | 职业路线明确 | 不适合跨部门沟通 |
| **弱矩阵** | 小 | 兼职 | 有利于跨部门沟通和资源充分利用 | 多头领导 |
| **平衡矩阵** | 小 ~ 中 | 兼职 | (折中方案) | (折中方案) |
| **强矩阵** | 中 ~ 大 | **全职** | - | - |
| **项目型** | 大 ~ 全部 | **全职** | 项目经理权力大 | 资源重复配置；项目结束人员“无家可归” |

*   **记忆技巧**：记住**平衡矩阵”，向左推是弱矩阵/职能型，向右推是强矩阵/项目型。
*   **注意**：**紧密矩阵（集中办公）**是一种团队建设技术或设施安排，**不是**一种组织结构。

#### 2.4.2 其他组织类型
*   **有机型/简单型组织**：非常灵活，适应变化。工作专业化少，管理层次少，决策分散，监督不多。
*   **多部门组织 (Multi-divisional)**：一个中心，多个半自治分区。中心下达财务指标。
*   **虚拟型组织**：临时召集人员利用特定机遇，目标完成后解散。

### 2.5 项目管理办公室 (PMO)
*   **定义**：对与项目相关的治理过程进行标准化，并促进资源、方法论、工具和技术共享的组织结构。
*   **PMO的三种类型**：
    1.  **支持型**：顾问角色。提供模板、最佳实践、培训、信息库。**控制程度很低**。
    2.  **控制型**：提供支持并要求服从。**控制程度中等**。
    3.  **指令型**：直接管理和控制项目。**控制程度很高**。
*   **PMO的主要作用**：
    *   **管理功能**：管理共享资源，识别/制定最佳实践和标准。
    *   **监督功能**：通过项目审计，监督对标准的遵守程度。
    *   **指导培训功能**：制定政策、程序、模板，提供指导和培训。
    *   **协调功能**：协调跨项目的沟通。

## 第3章 项目经理的角色

### 3.1 项目经理概述
*   **定义**由执行组织委派，领导团队实现项目目标的个人。
*   **职责**：无需承担每个角色，但应具备项目管理知识、技术知识、理解和经验。通过**沟通**领导团队进行规划和协调。
*   **影响力范围**：项目、组织、行业、专业学科、跨领域。

### 3.2 PMI 人才三角
项目经理应具备以下三类技能：
1.  **技术项目管理**：与项目、项目集、项目组合管理特定领域相关的知识、技能和行为（硬技能）。
2.  **领导力**：指导、激励和带领团队所需的知识、技能和行为，帮助组织达成业务目标（软技能）。
3.  **战略和商务管理**：关于行业和组织的知识和专业技能，有助于提高绩效并取得更好的业务成果（大局观）。

### 3.3 项目经理的权利来源
| 权力种类来源 | 描述 | 最易导致的结果 | 效果评价 | 有效对象 |
| :--- | :--- | :--- | :--- | :--- |
| **专家权力** | 个人具有的技能或技术专长 | 他人的忠诚 | **最好** | 与本专业相关者 |
| **参照权力**<br>(潜示权力) | 个人吸引他人并建立忠诚度的能力 | 他人的忠诚 | **较好** | 任何人 |
| **奖励权力** | 职位赋予的给予有价值物质奖励的能力 | 下属的忠诚或服从 | **较好** | 下属 |
| **正式权力**<br>(法定权力) | 职位和职务所拥有的权力 | 下属的服从 | 一般 | 下属 |
| **惩罚权力** | 职位赋予的使用或威胁使用惩罚手段的能力 | 下属的抵抗 | **不好**<br>(慎用) | 下属 |

### 3.4 领导风格
| 领导风格 | 特点/备注 |
| :--- | :--- |
| **放任型** | “无为而治”，允许团队自主决策和设定目标。适合创新型团队。 |
| **交易型** | 关注目标、反馈和成就以确定奖励。采用例外管理。 |
| **服务型** | 服务优先于领导，先为他人着想。关注他人成长、学习、人际关系、团体与合作。 |
| **变更型** | 通过理想化特质、鼓舞性激励、促进创新和个人关怀来提高追随者能力。 |
| **魅力型** | 精神饱满、热情、自信、说服力强，能够激励他人。 |
| **交互型** | 结合了交易型、变更型和魅力型的特点。 |

### 3.5 整合与复杂性
*   **整合是关键技能**：关注三个层面的整合：
    1.  **过程层面**
    2.  **认知层面**
    3.  **背景层面**
*   **项目复杂性的来源**：
    1.  组织的系统行为
    2.  人类行为
    3.  组织或环境中的不确定性

### 3.6 补充考点：项目过程中的趋势变化
在项目生命周期中，以下要素随时间变化的规律：

| 要素 | 变化趋势 | 备注 |
| :--- | :--- | :--- |
| **成本与人力投入** | **缓慢增加 -> 执行期最大 -> 迅速回落** | 开始时较低，执行期间逐渐增加，快要结束时迅速回落。 |
| **风险与不确定性** | **逐渐减小** | 随着项目进展，信息越明确，风险越低。 |
| **风险的影响/代价** | **逐渐增加** | 虽然风险概率降低，但若发生错误，后期修正代价极高。 |
| **变更的代价** | **逐渐增加** | 越往后变更，成本越高。 |
| **变更的数量** | **逐渐减少** | 项目初期变更多，后期趋于稳定。 |

### 3.7 重要区分
*   **过程组 vs 项目阶段**：
    *   过程组不同于项目阶段。
    *   如果将项目划分为若干阶段，则各过程组中的过程会在**每个阶段内**相互作用、相互交叉（即每个阶段都可能经历启动、规划、执行、监控、收尾过程组）。
