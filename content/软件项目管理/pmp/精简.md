## 第1章 引论

### 项目三特点
- **独特性** → 不确定性
- **临时性** → 起点+终点（不代表时间短）
- **渐进明细** → 信息越多→越准

### 可交付成果
- 有形成果（产品）+ 无形成果（服务）
- 项目临时，但可交付成果**一般不临时**

### 项目集 vs 项目组合 vs 运营

| 概念     | 关注重点 | 相互关系               |
| -------- | -------- | ---------------------- |
| 项目集   | 正确方式 | 项目间有依赖           |
| 项目组合 | 正确项目 | 不一定依赖、价值最大化 |
| 运营     | 持续运作 | 重复性                 |

### 生命周期类型
- **预测型**：早期定范围、难变更
- **迭代型**：范围早定、时间成本定期改
- **增量型**：渐进增加功能
- **适应型**：小增量快迭代、相关方参与最高

### 阶段关口
- 别名：阶段审查、生死点、杀点、里程碑
- 目的：继续/终止/调整

### 数据→信息→报告
- **数据**：原始观察（最底层）
- **信息**：整合分析
- **报告**：决策支持

### 关键文件
- **商业论证**：值不值得投资、全生命周期使用
- **效益管理计划**：实现效益的方式和时间、衡量机制
- **项目章程**：发起人发布、正式批准、授权PM
- **项目管理计划**：如何执行

### 成功标准
- 不同相关方看法不同 → **达成共识并记录**

## 第2章 运行环境

### EEFs vs OPAs
- **EEFs**：不能控制（内/外部制约）
- **OPAs**：可用资产（政策、程序、知识库）

### 内部EEFs
- 组织文化、基础设施、IT软件、员工能力

### 外部EEFs
- 商业数据库、政府标准、物理环境（天气/工作环境）

### OPAs更新权限
- **过程/政策/程序**：PMO或职能部门更新（非项目工作）
- **组织知识库**：整个项目期间更新

### 组织结构权力
| 类型     | PM权力  | 资源     | 特点                               |
| -------- | ------- | -------- | ---------------------------------- |
| 职能型   | 很小    | 兼职     | 职业路线明确、不适合跨部门         |
| 弱矩阵   | 小      | 兼职     | 多头领导                           |
| 平衡矩阵 | 小~中   | 兼职     | 折中                               |
| 强矩阵   | 中~大   | **全职** | -                                  |
| 项目型   | 大~全部 | **全职** | 权力大、资源重复、项目结束无家可归 |

- **紧密矩阵（集中办公）** ≠ 组织结构，是团队建设技术

### 其他组织类型
- **有机型**：灵活、决策分散
- **多部门**：中心+半自治分区
- **虚拟型**：临时召集、目标完成解散

### PMO三种类型
- **支持型**：顾问、控制很低
- **控制型**：要求服从、控制中等
- **指令型**：直接管理、控制很高

### PMO主要作用
- 管理资源、监督标准、指导培训、协调沟通

## 第3章 项目经理

### 人才三角
1. 技术项目管理（硬技能）
2. 领导力（软技能）
3. 战略商务（大局观）

### 权力来源（好→坏）
| 权力 | 来源       | 结果      | 评价           |
| ---- | ---------- | --------- | -------------- |
| 专家 | 技能专长   | 忠诚      | **最好**       |
| 参照 | 个人魅力   | 忠诚      | **较好**       |
| 奖励 | 职位给奖励 | 忠诚/服从 | 较好           |
| 正式 | 职位职务   | 服从      | 一般           |
| 惩罚 | 威胁惩罚   | 抵抗      | **不好、慎用** |

### 领导风格
- **放任型**：无为而治、适合创新
- **交易型**：目标反馈奖励、例外管理
- **服务型**：服务优先、关注成长
- **变更型**：理想化特质、鼓舞激励
- **魅力型**：热情自信、说服力强
- **交互型**：交易+变更+魅力

### 整合三层面
- 过程层面、认知层面、背景层面

### 复杂性来源
1. 组织系统行为
2. 人类行为
3. 不确定性

### 变化趋势
| 要素         | 趋势       |
| ------------ | ---------- |
| 成本人力     | 低→高→回落 |
| 风险不确定性 | 逐渐减小   |
| 风险代价     | 逐渐增加   |
| 变更代价     | 逐渐增加   |
| 变更数量     | 逐渐减少   |

### 过程组 vs 项目阶段
- 过程组 ≠ 项目阶段
- 每个阶段内都经历：启动→规划→执行→监控→收尾

## 第4章 整合管理

### 核心原则
- **PM负最终责任**、不能授权转移
- 商业文件PM**不能改**（只能建议）
- 项目人员流动大 → 更严格防止知识流失
- PM需介入启动和结束阶段

### 适应型环境关注点
- 合作决策氛围
- 团队应对变更能力
- 成员广泛技能

### 关键时机
- **项目章程** → 正式启动
- **PM任命**：最晚规划开始前（越早越好）
- **开工会议**：规划阶段最后事
- 目的：传达目标、获承诺、阐明角色职责

### 项目管理计划构成
- 各子管理计划（范围/需求/进度/成本/质量/资源/沟通/风险/采购/相关方/变更/配置）
- 三大基准：范围、进度、成本
- 绩效测量基准、生命周期描述、开发方法

### 变更控制
- 基准定后 → 必须走变更流程
- 变更请求**必须书面**
- 未批准就做 → **补变更**
- 不批准 → **取消变更**
- 已批准 → 必须执行（不同意的人可重新提变更）

### 变更请求四种类型
1. 预防措施（抗风险）
2. 纠正措施（纠偏差）
3. 缺陷补救（补质量）
4. 更新（改计划/基准）

### 问题日志
- 三要素：问题、责任人、解决期限

### 知识管理
- **显性知识**：可编撰的文字/图片
- **隐性知识**：难以表达（信念/洞察力/诀窍）
- **面对面互动**：最有利于建立信任
- **虚拟互动**：信任建立后维护用

### 经验教训登记册
- **项目早期创建**、全程更新
- 阶段结束归入经验教训知识库
- 题干问“如何有利于未来项目”→ 选“更新组织过程资产”或“总结经验教训”

### 收尾流程
- **正常**：验收→移交→总结→存档→释放
- **提前终止**：调查原因→移交成果→总结→存档→释放
- 收尾活动：满意度调查、庆功会

### 收尾后问题处理
- **已彻底收尾**：缺陷→运营部门；新功能→新项目
- **未彻底收尾**：缺陷→积极处理、走变更；新功能→新项目或走变更
- **收尾优先**：新旧项目冲突时，先收尾

## 第5章 范围管理

### 核心定义
- **产品范围**：特征功能（用需求衡量）
- **项目范围**：要完成的工作（用计划衡量）
- **范围=产品+项目**

### 关键区分
- **范围管理计划**：如何管理（无具体内容）
- **需求管理计划**：如何分析/记录/管理需求
- **范围基准**：具体内容（说明书+WBS+词典）

### 需求定义
- 产品/服务/成果必须具备的条件或能力

### 收集需求工具速记

**数据收集**
- **头脑风暴**：大量创意、畅所欲言
- **访谈**：一对一/多对多、机密信息
- **焦点小组**：同职能专家
- **问卷调查**：受众广、位置散、统计分析
- **标杆对照**：找最佳实践（内/外、同/不同行业）

**数据分析**
- **文件分析**：提取需求

**决策**
- **一致同意**：都同意
- **德尔菲**：专家、匿名、多轮、趋同、消除偏见
- **大多数同意**：>50%
- **相对多数**：得票最多（候选项>2）
- **独裁型**：一人决定
- **多标准决策**：决策矩阵

**数据表现**
- **亲和图**：分组大量创意
- **思维导图**：整合信息、激发新创意

**人际关系**
- **名义小组**：头脑风暴+投票排序
- **观察/交谈**：工作跟随、挖隐藏需求
- **引导**：跨职能协调差异
- **JAD**：软件开发、业务+开发集中
- **QFD**：制造、客户声音开始
- **用户故事**：角色+目标+动机

**其他**
- **系统交互图**：拓扑图、可视化交互
- **原型法**：渐进明细、模型→体验→反馈→修改（循环）

### 需求跟踪矩阵
- 连接需求→业务目标
- 确保每项需求都能交付
- 提供变更框架

### 范围说明书内容
- 产品描述、可交付成果、**验收标准**、**除外责任**

### 产品分析
- 产品分解、需求分析、系统分析、系统工程、价值分析、价值工程

### WBS原则
- **80小时**、4~6层、**100%原则**、责任唯一
- **滚动式规划**：远期先高层级
- **分解过细后果**：管理低效、资源浪费
- **控制账户**：范围+预算+进度整合、挣值测量

### 范围基准
- 范围说明书 + WBS + WBS词典

### 确认范围 vs 控制质量
|      | 确认范围         | 控制质量 |
| ---- | ---------------- | -------- |
| 关注 | 验收             | 正确性   |
| 签字 | 客户/发起人      | -        |
| 顺序 | 质量之后         | 确认之前 |
| 输入 | 核实的可交付成果 | -        |

### 未通过验收处理
1. 记录原因
2. 走变更、缺陷补救

### 范围蔓延
- 未经控制的范围扩大（未调时间/成本/资源）

## 第6章 进度管理

### 进度计划类型
- **具有未完项的迭代**：用户故事、优先级排序、时间盒
- **按需进度**：资源可用时立即提取

### 进度管理计划
- 无具体日期，是“如何管理”的指南
- 包含：模型制定、发布标准、计量单位、**控制临界值**（偏差百分数）

### 里程碑
- 不是活动、持续时间**零**、代表时间点

### 依赖关系
- **强制性**：法律/合同/内在性质（硬逻辑）
- **选择性**：最佳实践、快速跟进时审查（软逻辑）
- **外部**：非项目活动、不受控
- **内部**：项目活动间、受控

### 四种逻辑关系
- **FS**：最常用
- **FF**、**SS**、**SF**（很少用）

### 提前量 vs 滞后量
- **提前量**：紧后提前（-）
- **滞后量**：紧后推迟（+）

### 估算方法
| 方法     | 特点                             |
| -------- | -------------------------------- |
| 类比     | 快、成本低、不准、早期启动阶段用 |
| 参数     | 统计关系、依赖模型成熟度         |
| 三点     | 考虑不确定性                     |
| 自下而上 | 最准、最慢                       |
| 备选方案 | 比较不同方案                     |

### 储备
- **应急储备**：已知-未知、PM直接动用、可减少取消
- **管理储备**：未知-未知、走变更

### 关键路径法
- **关键路径**：最长路径=最短工期
- **总浮动**：不延误完工的可推迟时间
- **自由浮动**：不延误紧后的可推迟时间
- 总浮动可正/零/负

### 资源优化
- **资源平衡**：可能改关键路径、延长工期
- 触发：资源有限/过度分配/均衡使用
- **资源平滑**：不改关键路径、限浮动内
- 缺点：可能无法完全优化

### 假设情景分析
- “如果X出现会怎样？”

### 进度压缩
- **赶工**：加资源（加班/加急）→**增成本**
- **快速跟进**：并行→**增风险**、需审查选择性依赖

### 敏捷发布规划
- 发布进度时间轴（3-6个月）
- 确定迭代/冲刺次数

### 基准与计划
- **进度基准**：经批准的进度模型
- **项目日历**：可用工作日/班次

### 计划形式
- **横道图**：与管理层沟通、易读
- **里程碑图**：主要时点
- **网络图**：逻辑关系、纯逻辑图/逻辑横道图

### 控制进度工具
- **迭代燃尽图**：理想线vs实际、趋势线预测

## 第7章 成本管理

### 关注范围
- 活动资源成本
- 决策对**使用/维护/支持成本**的影响

### 适应型环境
- 轻量级估算、高层级预测、JIT短期规划

### 成本管理计划
- 无成本，是“如何管理”的指南

### 估算精度
- **ROM**：-25% ~ +75%
- **确定性**：-5% ~ +10%

### 估算方法
| 方法          | 特点                                 |
| ------------- | ------------------------------------ |
| 类比          | 专家判断、自上而下、快、成本低、不准 |
| 参数          | 统计关系                             |
| 三点          | 考虑不确定性                         |
| 自下而上      | 最准最慢                             |
| 备选方案      | 比较方案                             |
| 质量成本(COQ) | 短期降本vs后期问题                   |

### 成本基准
- 经批准的按时间段分配预算
- **不含管理储备**
- 只走变更控制程序修改

### 项目资金需求
- 增量投入、可能非均衡
- **总资金 = 成本基准 + 管理储备**

### 储备归属
- **应急储备**：在成本基准内、PM直接动用
- **管理储备**：不在成本基准、动用需变更、动用后加入基准

### 资金限制平衡
- 资金限制与计划支出有差异 → 调整进度计划

### 挣值核心指标
- **PV**：计划价值
- **EV**：挣值
- **AC**：实际成本
- **SV** = EV - PV（进度偏差）
- **CV** = EV - AC（成本偏差）
- **SPI** = EV/PV（进度绩效）
- **CPI** = EV/AC（成本绩效）

### 预测指标
- **EAC**：完工估算
- **ETC**：完工尚需估算

### TCPI
- 剩余工作/剩余预算
- 为实现管理目标需达到的绩效

### 储备分析
- 判断是否还需要或需要增加储备

## 第8章 质量管理

### 大师速记
| 大师     | 关键词                                   |
| -------- | ---------------------------------------- |
| 戴明     | PDCA、预防胜检查、85/15（管理层责任85%） |
| 朱兰     | 三步曲、质量vs等级、适合使用             |
| 克劳斯比 | 第一次做对、质量免费、非一致性成本       |
| 石川馨   | 鱼骨图、七工具                           |
| 田口     | 实验设计、稳健设计                       |
| 休哈特   | PDCA基础                                 |

### 关键区分
- **质量**：满足要求（低质量是问题）
- **等级**：功能多少（低等级不是问题）

- **预防**：不犯错（成本低、高效）
- **检查**：不流到客户（成本高、被动）

- **属性抽样**：合格/不合格
- **变量抽样**：程度多少

- **公差**：客户接受范围
- **控制界限**：过程稳定范围（±3σ、由数据计算）

### 质量管理概念
- **TQM**：全员参与
- **Kaizen**：持续小改进积累
- **客户满意**：符合要求+适合使用

### 质量水平（低→高）
1. 客户
2. QC（质量控制）
3. QA/管理质量
4. QP/规划
5. 文化

### 质量成本(COQ)
- **一致成本**：预防+评估（为防止失败）
- **非一致成本**：内部失败+外部失败（因失败产生）

### 管理质量 vs 控制质量
|      | 管理质量(QA) | 控制质量(QC)   |
| ---- | ------------ | -------------- |
| 关注 | 过程         | 结果           |
| 敏捷 | 所有成员     | 特定成员(传统) |

### 缺陷处理
- **个别缺陷**：变更请求→缺陷补救
- **大量缺陷**：上升到过程→QA

### 问题发现责任
- **客户发现问题**：QC没做好（漏检）
- **QC发现问题**：QA没做好（过程预防）

### 质量工具速记

**规划质量**
- 标杆对照、头脑风暴、访谈、成本效益、COQ、多标准决策、流程图、逻辑数据模型、矩阵图、思维导图、测试与检查规划

**管理质量(QA)**
- 核对单、过程分析、**根本原因分析(RCA)**、亲和图、**因果图(鱼骨图/石川)**、直方图、**帕累托图(二八原则)**、散点图、**质量审计**、DfX、问题解决(六步法)、PDCA/六西格玛

**控制质量(QC)**
- 核查表(计数表→帕累托)、统计抽样、检查、测试、**控制图**(规格界限vs控制界限)

### 质量审计五方面
1. 识别
2. 分享
3. 协助
4. 积累
5. 确认

### 问题解决六步法
1. 定义
2. 识别
3. 方案
4. 选择
5. 执行
6. 验证

## 第9章 资源管理

### 层级结构
- **WBS**：工作分解
- **OBS**：组织分解
- **RBS**：资源分解（类别+类型）

### RACI
- **R**：执行
- **A**：负责（每个任务只有一个）
- **C**：咨询
- **I**：知情

### 角色四要素
- 角色、职权、职责、能力

### 团队章程
- 价值观、共识、基本规则
- 作用：减少误解、指导新成员
- 可定期审查更新

### 培训
- 目的：减少差异
- 成本：项目预算（当前）/ 执行组织（未来）

### 获取资源
- **谈判**：职能经理（普通）、其他团队（稀缺/特殊）
- **预分派**：竞标承诺/专有技能/章程指定
- **虚拟团队**：注意沟通问题

### 多标准决策分析标准
- 可用性、能力、成本

### 塔克曼阶梯
1. **形成**：相互认识、不一定开诚布公
2. **震荡**：不同意见、有冲突
3. **规范**：开始协同、学习信任
4. **成熟**：组织有序、平稳高效
5. **解散**：释放人员

### 团队建设
- 形式：各种活动（包括开会）
- 目的：协同工作、增强信任

### 认可与奖励
- 计划来源：规划资源管理过程中编制
- **有效原则**：满足被奖励者重要需求
- **无形奖励**：同样有效、有时更有效
- **时机**：整个生命周期尽可能表彰

### 个人和团队评估
- 例子：360°绩效考核

### 团队绩效评价指标
- 个人技能改进
- 团队能力改进
- 离职率降低
- 凝聚力加强

### 冲突来源
- 资源稀缺、进度优先级、个人工作风格

### 冲突解决层级
1. 团队成员自行解决
2. PM协助
3. 正式程序（含惩戒）

### 冲突管理（好→坏）
| 方法      | 核心           | 特点                   |
| --------- | -------------- | ---------------------- |
| 合作/解决 | 共识达成       | **最理想**、开放式对话 |
| 妥协/调解 | 部分满意       | 各方一定程度满意       |
| 缓和/包容 | 强调一致       | 考虑他方需要           |
| 强迫/命令 | 用权力解决     | 紧急问题用             |
| 撤退/回避 | 推迟、推给别人 | **最差**               |

### 情商
- 识别/评估/管理自己、他人、团体的情绪
- 作用：减轻压力、加强合作

### 控制资源
- 确保资源适时适地可用
- 不再需要时释放
- 所有阶段持续开展

## 第10章 沟通管理

### 沟通活动分类维度
- **范围**：内部/外部
- **形式**：正式/非正式
- **方向**：向上/向下/横向
- **性质**：官方/非官方
- **媒介**：书面/口头

### PM时间占比
- **90%以上**时间在沟通

### 沟通构成
- 口头45%（用词7%+音调38%）
- 非口头55%

### 适用场景
- **书面**：解决复杂问题
- **官方**：体现真实意图

### 成功沟通两部分
1. 沟通策略
2. 沟通管理计划

### 5C原则
- **C**orrect（正确语法拼写）
- **C**oncise（简洁）
- **C**lear（清晰）
- **C**oherent（连贯）
- **C**ontrolling（受控）

### 沟通需求分析
- 确定信息需求（类型/格式/价值）

### 沟通渠道公式
- **N × (N-1) / 2**

### 沟通技术选择因素
- 紧迫性、可用性、易用性、敏感性、保密性

### 沟通模型
- 编码→传递→解码→确认收到→反馈
- **确认收到 ≠ 同意/理解**

### 双方责任
- **发送方**：清晰完整、确认被理解
- **接收方**：完整接收、正确理解、回应
- **共同**：注意噪音障碍

### 沟通方法
- **互动**：实时多向（如会议）
- **推式**：发给特定接收方（如邮件）
- **拉式**：受众主动获取（大量信息、网站）

### 沟通风格评估
- 用于不支持项目的相关方

### 沟通管理计划要素
- 相关方需求、信息细节、**问题升级程序**、时限频率、接收方、传递方法、资源分配、通用术语表

### 问题信号
- 信息多发/少发/迟发/看不懂/不方便阅读/虚拟团队问题 → 更新沟通管理计划

### 项目沟通记录
- 绩效报告、可交付成果状态、进度进展、成本、演示材料

### 监督标准（4正确）
- 正确时间、正确渠道、正确内容（理解一致）、正确受众

## 第11章 风险管理

### 目标
- 提高正面风险（机会）概率/影响
- 降低负面风险（威胁）概率/影响

### 风险三要素
- 风险事件、概率、影响

### 风险来源
- 项目的独特性

### 单个项目风险 vs 整体项目风险
- **单个**：对一个或多个目标的影响
- **整体**：对项目整体的影响（含所有不确定性）

### 未知风险应对
- 通过加强**项目韧性**

### 风险管理计划内容
- **RBS**（风险类别）
- 相关方风险偏好
- 风险概率和影响定义
- 概率和影响矩阵
- **不含具体风险列表**

### 识别风险参与原则
- 鼓励所有相关方参与
- 项目团队尤其重要（主人翁意识）
- 迭代过程

### 识别风险工具
- **头脑风暴**：自由/结构化
- **核对单**：过去常见（不能穷尽、需维护更新）
- **访谈**：信任保密环境
- **假设条件制约因素分析**
- **SWOT**：优势/劣势/机会/威胁：包含内部风险
- **提示清单**：RBS底层 / PESTLE/TECOP/VUCA

### 登记册 vs 报告
- **风险登记册**：单个风险（清单/责任人/应对措施）
- **风险报告**：整体风险+单个概述

### 定性分析作用
- 重点关注高优先级风险
- 为每个风险识别责任人

### 定性分析工具
- 风险数据质量评估
- 概率和影响评估
- **观察清单**：低概率低影响
- 其他参数（紧迫性/邻近性）
- 风险分类
- 概率和影响矩阵
- **气泡图**：显示三维数据

### 定量分析适用场景
- 大型/复杂/战略重要/合同要求/相关方要求

### 定量分析工具
- 访谈（信任保密）
- 不确定性表现方式
- **模拟（蒙特卡洛）**
- **敏感性分析（龙卷风图）**
- **决策树（EMV预期货币价值）**
- 影响图

### 威胁应对策略
| 策略  | 核心     | 适用   | 注意              |
| --- | ------ | ---- | --------------- |
| 上报  | 更高层管理  | -    | 团队不再监督          |
| 规避  | 消除威胁   | 高优先级 | 延长进度/缩小范围       |
| 转移  | 第三方    | -    | 付费、非消除风险        |
| 减轻  | 降概率/影响 | -    | 更简单流程/更多测试/冗余   |
| 接受  | 承认存在   | 低优先级 | 主动→应急储备；被动→定期复查 |

### 机会应对策略
| 策略 | 核心         | 举例          | 适用           |
| ---- | ------------ | ------------- | -------------- |
| 上报 | 更高层管理   | -             | -              |
| 开拓 | 100%发生     | 牛人/新技术   | 高优先级       |
| 分享 | 第三方享收益 | 合伙/合资     | 需新风险责任人 |
| 提高 | 增概率/影响  | 普通人加资源  | -              |
| 接受 | 承认存在     | 主动→应急储备 | 低优先级       |

### 应对后更新
- 变更请求（成本/进度基准）
- 项目管理计划更新
- 风险登记册更新（策略/行动/预警/应急/弹回/次生/残余）

### 实施风险应对工具
- **影响力**：鼓励责任人行动

### 监督风险工具
- **技术绩效分析**：计划vs实际
- **储备分析**：剩余储备 vs 剩余风险
- **风险审计**：评估过程有效性（PM负责频率）
- **风险审查会**：识别新风险、重新评估、关闭过时

## 第12章 采购管理

### 核心原则
- PM**无权**签署对组织有约束力的法律协议
- 由采购管理员/法务/高层执行

### 国际项目特殊性
- 文化差异和当地法律影响合同可执行性

### 协议多种形式
- 合同、SLA、谅解备忘录、MOA、订购单

### 规划采购工具
- **市场调研**：考察行业和卖方能力
- **自制或外购分析**：自己做还是买
- **独立成本估算**：评价卖方报价的基准
- 差异大→SOW缺陷/卖方误解

### 供方选择分析场景
| 方法          | 适用        |
| ------------- | ----------- |
| 最低成本      | 标准化/常规 |
| 仅凭资质      | 价值小      |
| 基于质量/技术 | -           |
| 基于质量+成本 | -           |
| 独有来源      | 单一来源    |
| 固定预算      | 预算不可调  |

### 招标文件形式
- 信息邀请书、报价邀请书、建议邀请书

### 采购工作说明书(SOW)
- 清晰/完整/简练
- 可修订、最终成协议一部分

### 实施采购流程
- 招→投→评→授

### 投标人会议
- **时机**：建议书提交前
- **目的**：确保所有卖方一致理解、公平

### 建议书评估方法
- 加权打分、筛选系统

### 采购谈判
- **时机**：合同签署前
- **主导**：有合同签署职权的成员
- **PM**：可协助

### 协议(合同)内容要素
- SOW、进度、验收标准、条款条件、变更流程、终止条款、不可抗力

### 索赔途径（优→劣）
1. **谈判**（首选）
2. **ADR**（调解/仲裁）
3. **起诉**（最后手段）

### 控制采购工具
- **绩效审查**：检查卖方工作
- **检查**：检查可交付成果
- **审计**：采购过程结构化审查、总结经验教训

### 变更控制原则
- 采购变更必须**书面**+**正式批准**

### 合同类型速记

**总价合同**（范围明确）

| 类型       | 特点                               |
| ---------- | ---------------------------------- |
| **FFP**    | 固定不变                           |
| **FPIF**   | 最高限价、分摊比例、与限价比取小值 |
| **FP-EPA** | 长期、调价（通胀）                 |

**成本补偿**（范围不明确）

| 类型     | 特点                     |
| -------- | ------------------------ |
| **CPIF** | 分摊比例、利润上下限     |
| **CPAF** | 奖励买方主观、不能申诉   |
| **CPFF** | 固定费用（基于初始估算） |

**工料合同**

| 类型    | 特点                            |
| ------- | ------------------------------- |
| **T&M** | 工期短、金额小、紧急、无详细SOW |

### 合同终止
- **正常关闭**：买方发出完成书面通知
- **提前终止**：协商一致/违约/买方便利
- **终止补偿**：已完成工作报酬 + 准备工作补偿

## 第13章 相关方管理

### 相关方定义
- 能影响项目
- 被项目影响
- **自认为**是相关方

### 管理目标
- 提高项目成功可能性
- **相关方满意度**作为项目目标

### 启动时机
- 项目章程批准、PM委任、团队开始组建后**尽早**

### 迭代开展时点
1. 进入不同阶段
2. 相关方变化（无关或新增）
3. 组织/社区重大变化

### 识别相关方分析维度
- 兴趣、权利、所有权、知识、贡献

### 分析模型
| 模型          | 适用              | 维度                   |
| ------------- | ----------------- | ---------------------- |
| 权力/利益方格 | 小型/简单         | 权力、利益、影响、作用 |
| 凸显模型      | 复杂/关系复杂     | 权力、紧迫性、合法性   |
| 优先级排序    | 数量巨大/频繁变化 | 分级管理               |

### 关键指标定义
- **权力**：职权
- **利益**：对结果关注程度
- **影响**：主动参与程度
- **作用**：改变计划/执行的能力

### 规划相关方参与工具
- **根本原因分析**：找支持/反对的根本原因
- **优先级排序**：最大利益+最高影响排最前
- **思维导图**：可视化关系
- **标杆对照**：与其他组织比较

### 相关方参与度评估矩阵
- **核心**：当前(C) vs 期望(D) → 差距即行动

### 五种参与水平
1. **不了解**：不知道项目
2. **抵制**：知道但抵制
3. **中立**：知道、不支持不反对
4. **支持**：知道且支持
5. **领导**：积极参与确保成功

### 管理相关方参与关键活动
- 引导参与（获取/确认/维持承诺）
- 管理期望（谈判/沟通）
- 风险管理（预测问题）
- 问题解决

### 监督相关方参与
- 动态调整、确保策略始终适应
