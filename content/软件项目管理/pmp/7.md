### 第7章 项目成本管理概述

**1. 定义与范围**
*   **定义**：项目成本管理包括为使项目在批准的预算内完成而对成本进行规划、估算、预算、融资、筹资、管理和控制的各个过程。
*   **关注点**：
    *   重点关注完成项目活动所需资源的成本。
    *   同时应考虑项目决策对项目产品、服务或成果的**使用成本、维护成本和支持成本**的影响。
*   **适应型环境 vs 预测型环境**：
    *   **适应型环境**：采用轻量级估算方法，快速生成对项目人力成本的高层级预测，便于在出现变更时调整。
    *   **详细估算**：适用于采用准时制（JIT）的短期规划。

### 7.2 估算成本

**核心概念**
*   **定义**：对完成项目工作所需资源成本进行近似估算的过程。
*   **计量单位**：
    *   通常用某种货币单位。
    *   有时也可采用其他计量单位（如人时数或人天数），以消除通货膨胀的影响，便于成本比较。
*   **估算精度区间**：
    *   **粗略量级估算 (ROM)**：区间为 **-25% 到 +75%**。
    *   **确定性估算**：随着信息越来越详细，区间可缩小至 **-5% 到 +10%**。

**工具和技术详解**

| 工具/技术名称 | 关键词与核心特征 |
| :--- | :--- |
| **类比估算** | 1. **本质**属于专家判断。<br>2. **整体估算**、**自上而下**估算。<br>3. **特点**：速度快、成本低、但**不准确**。<br>4. **可靠性条件**：当以往活动在**本质上**（而非表面上）类似，且团队成员具备必要专业知识时，最为可靠。 |
| **参数估算** | 利用历史数据之间的**统计关系**和其他变量。<br>准确性取决于参数模型的成熟度和基础数据的可量化性。 |
| **三点估算** | 考虑估算中的**不确定性和风险**，提高持续时间/成本估算的准确性。 |
| **自下而上的估算**从下到上逐层汇总，通常最准确但耗时最长。 |
| **数据分析** | **1. 备选方案分析**：比较不同的方案，确定最佳方式。<br>**2. 储备分析**：<br>- **应急储备**：用于应对**“已知—未知”**风险。<br>- **权限**：项目经理可以**直接动用**。<br>- **动态调整**：随着项目信息越来越明确，可以动用、减少或取消应急储备。<br>**3. 质量成本 (COQ)**：<br>- 评估是为达到要求而增加投入，还是承担不符合要求而造成的成本。<br>- 权衡是寻求短期成本降低，还是承担产品生命周期后期频繁出现问题的后果。 |

### 7.3 制定预算

**核心概念**
*   **定义**：汇总所有单个活动或工作包的估算成本，建立一个经批准的**成本基准**的过程。
*   **主要作用**：确定可据以监督和控制项目绩效的成本基准。

**工具和技术详解**

1.  **成本汇总**
    *   **流程**: 成本估算 -> 汇总到 WBS **工作包** -> 汇总至更高层次（如**控制账户**) -> 得出整个项目的**总成本**。

2.  **储备分析 (针对管理储备)**
    *   **目的**：应对项目范围中**不可预见**的工作，即**“未知—未知”**风险。
    *   **归属**：
        *   **不包括**在成本基准中。
        *   **属于**项目总预算和资金需求的一部分。
    *   **变更规则**：当动用管理储备资助不可预见的工作时，需将其增加到成本基准中，从而导致**成本基准变更**。

3.  **历史信息审核**
    *   有助于进行参数估算或类比估算。

4.  **资金限制平衡**
    *   **原则**：根据对项目资金的任何限制，来平衡资金支出。
    *   **调整措施**：如果发现资金限制与计划支出之间存在差异，可能需要**调整工作的进度计划**，以平衡资金支出水平。

*(注：此处已移除原7.3.1输入和7.3.3输出部分，但保留了关于成本基准和资金需求的核心定义以便理解上述工具)*

> **补充核心定义（源自被移除的输出部分的关键概念，对于理解本章至关重要）：**
> *   **成本基准**：经过批准的、按时间段分配的项目预算，**不包括**任何管理储备。只有通过正式的变更控制程序才能变更，用作与实际结果进行比较的依据。它是不同进度活动经批准的预算的总和。
> *   **项目资金需求**：项目资金以增量方式投入，可能是非均衡的。若有管理储备，则：**总资金需求 = 成本基准 + 管理储备**。

### 7.4 控制成本

**核心概念**
*   **定义**：监督项目状态，以更新项目成本和管理成本基准变更的过程。
*   **主要作用**：在整个项目期间保持对成本基准的维护。

**工具和技术详解**

1.  **挣值分析 (EVA)**
    *   核心指标：**BAC** (完工预算), **PV** (计划价值), **EV** (挣值), **AC** (实际成本)。

2.  **偏差分析**
    *   核心指标：
        *   **SV** (进度偏差)
        *   **CV** (成本偏差)
        *   **SPI** (进度绩效指数)
        *   **CPI** (成本绩效指数)

3.  **趋势分析**
    *   核心指标：
        *   **EAC** (完工估算)
        *   **ETC** (完工尚需估算)

4.  **储备分析**
    *   **用途**：监督项目中**应急储备**和**管理储备**的使用情况。
    *   **目的**：判断是否还需要这些储备，或者是否需要增加额外的储备。

5.  **完工尚需绩效指数 (TCPI)**
    *   **定义**：为了实现特定的管理目标，剩余资源的使用必须达到的成本绩效指标。
    *   **计算公式含义**：完成剩余工作所需的成本与剩余预算之比。

*(注：此处已移除原7.4.1输入和7.4.3输出部分)*

> **补充核心定义（源自被移除的输出部分的关键概念）：**
> *   **工作绩效信息**：包括有关项目工作实施情况的信息（对照成本基准），可以在工作包层级和控制账户层级上评估已执行的工作和工作成本方面的偏差。
> *   **成本预测**：无论是计算得出的 EAC 值，还是自下而上估算的 EAC 值，都需要记录下来，并传达给相关方。
