## 1. 课程回顾与正则表达式 (Regular Expressions)

### 1.1 正则式与语言集合的关系
- **核心概念：** 正则表达式 (Regular Expression, RE) 是描述**正则语言 (Regular Language)** 的形式化工具。
- **形式化定义：** 给定一个正则式 $R$，它代表一个语言集合 $L(R)$。
- **典型例题分析：**
    1.  **示例 1：** $a(a|b)^*b$
        -   **含义：** 以 $a$ 开头，以 $b$ 结尾，中间是任意数量的 $a$ 或 $b$ 的组合。
    2.  **示例 2：** $(b|ab)^*$ (课堂推导示例)
        -   **分析方法：** 从内向外拆解。
        -   **星号运算 ($*$)：** 表示闭包，即 $\epsilon \cup R \cup RR \cup RRR \dots$
        -   **语言特性总结：** 考试时若无法用自然语言精确描述，可列举典型字符串（如 $\epsilon, b, ab, bb, bab \dots$）并说明其符号特性。

### 1.2 语言的形式化 (Formalization)
- **为什么需要形式化？**
    -   自然语言描述（如"C 语言整型常量”）存在歧义。
    -   编译器设计需要先对**最小语义单元 (Token)** 进行形式化定义。
    -   **工具：** 正则表达式是词法分析中定义 Token 的核心工具。
- **实例：C 语言常量定义**
    -   参考 C99 标准或实验指导书。
    -   使用乔姆斯基生成式或正则式定义（如：`digit` -> `0|1|...|9`）。
    -   **重点：** 能够根据自然语言描述写出对应的正则式，或根据正则式描述语言集合。

> - **考试重点：** 根据正则式写出语言集合，或根据语言描述写出正则式。

## 2. 确定有限状态自动机 (DFA)

### 2.1 DFA 的定义 (五元组)
DFA 是识别正则语言的计算模型，形式化定义为五元组 $M = (Q, \Sigma, \delta, q_0, F)$：
1.  **$Q$ (States)：** 有穷状态集合。
2.  **$\Sigma$ (Alphabet)：** 输入字母表 (如 $\{0, 1\}$ 或 $\{a, b\}$)。
3.  **$\delta$ (Transition Function)：** 状态转移函数。
    -   定义：$\delta: Q \times \Sigma \to Q$
    -   特性：**确定性** (单射)，即对于当前状态和输入字符，下一个状态是**唯一**的。
4.  **$q_0$ (Start State)：** 初态，$q_0 \in Q$。
5.  **$F$ (Final States)：** 终态集合，$F \subseteq Q$。

### 2.2 DFA 的三种表示方法
1.  **数学定义 (五元组)：** 严谨，适合理论证明。
2.  **状态转移图 (State Transition Diagram)：**
    -   **圆圈：** 代表状态。
    -   **箭头：** 代表转移，弧上标记输入字符。
    -   **入口箭头：** 指向初态。
    -   **双圈：** 代表终态。
    -   *优点：直观，设计 DFA 时最常用。*
3.  **状态转移矩阵 (Transition Table)：**
    -   行：状态 $Q$。
    -   列：输入字符 $\Sigma$。
    -   内容：下一状态。
    -   *优点：适合编程实现和子集构造法计算。*

### 2.3 DFA 设计实例
- **实例 1：电梯控制模型**
    -   将实际物理过程抽象为状态（如：开门、运行、停止、错误）。
    -   输入为离散信号（如：有人、无人、按钮信号）。
    -   **核心思想：** 将现实问题映射为状态转移逻辑。
- **实例 2：识别特定字符串模式**
    -   **题目：** 设计 DFA 接受含有**奇数个 0** 或 **奇数个 1** 的字符串。
    -   **设计思路：**
        -   利用状态记录“奇/偶”性。
        -   例如：状态 $q_{even\_0}$ 和 $q_{odd\_0}$ 之间通过输入 `0` 互相跳转。
        -   终态设定为符合“奇数”条件的状态。
- **实例 3：C 语言注释识别**
    -   模式：`/* ... */`
    -   逻辑：看到 `/` 进入预备态，看到 `*` 进入注释内容态（自循环），看到 `*` 后检查是否为 `/` 以结束。

### 2.4 DFA 的识别机制
- **接受条件：** 输入字符串 $\alpha$ 处理后，若自动机停在**终态** ($q \in F$)，则称该字符串被接受。
- **语言定义：** DFA $M$ 接受的所有字符串集合称为 $L(M)$。
- **空串 $\epsilon$：** 若初态即为终态，则 DFA 接受空串。

## 3. 非确定有限状态自动机 (NFA)

### 3.1 NFA 与 DFA 的区别
NFA 放松了 DFA 的限制，定义更为灵活：
1.  **多态转移：** 对于同一状态和同一输入，可以转移到**多个**下一状态（$\delta: Q \times \Sigma \to 2^Q$）。
2.  **$\epsilon$-转移 (空转移)：** 可以在不消耗输入字符的情况下进行状态转移（输入 $\epsilon$）。

| 特性 | DFA | NFA |
| :--- | :--- | :--- |
| **下一状态** | 唯一 (1 个) | 不唯一 (状态集合) |
| **空转移** | 不允许 | 允许 ($\epsilon$) |
| **表达能力** | 等价 | 等价 (识别的语言类相同) |
| **实现难度** | 容易直接实现 | 需转换为 DFA 实现 |

### 3.2 NFA 的等价性
- **定理：** 对于任意 NFA，都存在一个等价的 DFA，使得 $L(NFA) = L(DFA)$。
- **意义：** NFA 设计更灵活（接近正则式思维），DFA 执行效率更高（无回溯）。编译器通常先构造 NFA，再确定化为 DFA。

## 4. NFA 到 DFA 的转换 (确定化)

**这是本章的核心算法，考试必考。**

### 4.1 核心思想：子集构造法 (Subset Construction)
- **原理：** 将 NFA 的**状态集合**看作 DFA 的**单个状态**。
- **目标：** 消除 $\epsilon$ 边，消除多义性转移。

### 4.2 关键概念：$\epsilon$-闭包 ($\epsilon$-closure)
- **定义：** 从状态 $s$ 出发，仅通过 $\epsilon$ 边能到达的所有状态集合（包含 $s$ 自身）。
- **记号：** $CLOSURE(s)$ 或 $\epsilon(s)$。
- **计算方法：**
    1.  将 $s$ 加入集合。
    2.  若集合中有状态 $t$ 存在 $\epsilon$ 边指向 $u$，将 $u$ 加入集合。
    3.  重复直到没有新状态加入。

### 4.3 转换步骤 (算法流程)
1.  **初始化：**
    -   DFA 初态 $D_0 = \epsilon\text{-}closure(NFA\_Start)$。
    -   将 $D_0$ 加入未标记状态队列。
2.  **迭代构造：**
    -   取出一个未标记的 DFA 状态 $T$ (它是 NFA 状态的集合)。
    -   对于每个输入字符 $a \in \Sigma$：
        -   计算 $U = \epsilon\text{-}closure(\bigcup_{s \in T} \delta_{NFA}(s, a))$。
        -   即：先走 $a$ 边，再走所有可能的 $\epsilon$ 边。
        -   若 $U$ 是新状态，加入队列；记录转移 $\delta_{DFA}(T, a) = U$。
3.  **确定终态：**
    -   若 DFA 状态 $T$ 中包含 NFA 的任意终态，则 $T$ 为 DFA 终态。
4.  **结束：** 当所有状态都被标记，转换完成。

### 4.4 实例演示 (课堂矩阵法)
- 利用**状态转移表**进行子集构造往往比画图更清晰。
- **步骤：**
    1.  第一行写初态的 $\epsilon$-闭包。
    2.  计算该集合在输入 `0` 和 `1` 下的新集合。
    3.  将新集合作为新状态，继续计算，直到不再产生新集合。
    4.  重命名集合（如 $\{1, 2, 5\}$ 命名为 $A$），画出 DFA 图。

## 5. 编译原理中的形式化思想 (重要观念)

- **计算机科学的本质：** 计算机科学是数学的应用科学。
- **形式化的重要性：**
    -   先有**形式化定义** (Formal Definition)，后有**编程语言**。
    -   先有**计算模型** (如自动机)，后有**自动生成算法** (如词法分析器生成工具 Lex/Flex)。
- **学习建议：**
    -   不要只关注写代码，要理解背后的数学模型（离散数学、形式语言）。
    -   理解 DFA/NFA 不仅是为考试，更是理解“机器如何识别符号”的基础。
    -   **大三关键期：** 若此时未建立形式化思维，后续学习（形式语义、高级编译优化）将非常困难。
