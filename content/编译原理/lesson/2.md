## 1. 编译器构造的理论基础

### 1.1 核心思想：可计算性与计算模型
*   **编译器的本质**：将源语言（Source Language）变换为目标语言（Target Language）的装置。
*   **机器处理能力**：
    *   早期计算机（20-40 年代）只能处理符号。
    *   核心问题：**这件事是否可计算？** (Computability)
    *   保障可计算性的基础：**计算模型** (Calculation Model)。
*   **经典计算模型**：
    *   自动机 (Automata)
    *   下推自动机 (Pushdown Automata)
    *   图灵机 (Turing Machine)
    *   *注：编译原理的设计建立在原始的可计算理论基础之上。*

### 1.2 形式语言理论 (Chomsky)
*   **背景**：自然语言（如中、英、法）过于复杂，无法直接被机器完全形式化处理。
*   **解决方案**：构造**程序设计语言**（人工语言），使其满足机器可读、可写、可计算。
*   **乔姆斯基 (Noam Chomsky) 的形式系统**：
    *   将语言分为：**语法 (Syntax)**、**语义 (Semantics)**、**语用 (Pragmatics)**。
    *   **文法 (Grammar)**：词与词之间的关系结构。
    *   **形式化**：用数学符号和规则描述语言（字母表 -> 词 -> 句子 -> 文章）。
*   **编译器的底层逻辑**：
    *   **计算模型**是底层逻辑。
    *   **程序设计语言**是计算模型的产物。
    *   **编译器**是实现从源语言到目标语言变换的计算过程。

## 2. 经典编译器的体系结构

### 2.1 前端与后端 (Front-end & Back-end)
编译器通常分为两个主要部分，以中间代码 (IR) 为界：

| 阶段 | 主要任务 | 处理对象 | 依赖 |
| :--- | :--- | :--- | :--- |
| **前端 (Front-end)** | 语言分析 | 源语言 (文法、语义) | 与源语言相关，与机器无关 |
| **后端 (Back-end)** | 代码生成与优化 | 目标代码 (机器指令) | 与目标机器相关 (指令集、寄存器) |

### 2.2 编译过程的五个主要阶段
1.  **词法分析 (Lexical Analysis)**：
    *   输入：源程序字符流。
    *   任务：扫描、识别**最小语义单元**（词/Tokens）。
    *   输出：词流 (Token Stream) 及属性。
2.  **语法分析 (Syntax Analysis)**：
    *   输入：词流。
    *   任务：检查词的组合是否符合**文法规则**。
    *   输出：语法树 (Parse Tree) 或抽象语法树 (AST)。
    *   *方法*：自顶向下 (Top-down) 或 自底向上 (Bottom-up)。
3.  **语义分析与中间代码生成 (Semantic Analysis & IR Gen)**：
    *   任务：检查语义合理性（如类型检查），生成**中间表示 (IR)**。
    *   *注*：语义分析与中间代码生成通常是一趟活。
    *   **中间代码 (IR)**：如四元式、三元式。不可直接执行，但机器可读，便于优化和移植。
4.  **代码优化 (Code Optimization)**：
    *   任务：提高目标代码的效率（时间、空间）。
    *   *例子*：循环外提 (Loop Invariant Code Motion)、删除冗余计算。
    *   *注意*：编译器原理中的优化主要针对**中间代码**或**目标代码**，而非源码。
5.  **目标代码生成 (Target Code Generation)**：
    *   任务：将 IR 映射为特定机器的**汇编或机器指令** (0/1 二进制)。
    *   依赖：目标机指令集 (如 x86, ARM)。

### 2.3 辅助模块
*   **符号表 (Symbol Table)**：贯穿整个编译过程，记录变量、函数等符号的属性（类型、作用域等）。
*   **出错处理 (Error Handling)**：定位、报告错误，并尝试恢复以继续编译。

### 2.4 编译器的“遍” (Passes)
*   **定义**：从头到尾扫描源程序的次数。
*   **单遍编译**：效率高，但优化能力弱（如早期编译器）。
*   **多遍编译**：
    *   第一遍：生成中间代码。
    *   第二遍：优化中间代码。
    *   第三遍：生成目标代码。
    *   *权衡*：遍数越多，优化越好，但编译时间越长，存储消耗越大。

## 3. 词法分析详解 (Lexical Analysis)

### 3.1 词法分析器的功能
1.  **扫描 (Scan)**：从左到右读取源程序字符。
2.  **识别 (Recognize)**：识别出具有独立意义的最小单元（词/Tokens）。
3.  **输出 (Output)**：生成**词流**及**属性**（如：标识符、关键字、常数、运算符）。
    *   *属性示例*：词的类型、词长、在符号表中的位置。

### 3.2 词的形式化描述：正规式 (Regular Expression)
词法分析的核心数学基础是**正规式**（正则表达式）。

*   **字母表 ($\Sigma$)**：语言中允许使用的基本符号集合（如 C 语言的键盘符号）。
*   **正规式定义规则**：
    1.  $\epsilon$ (空串) 和 $\emptyset$ (空集) 是正规式。
    2.  字母表上的任意字符 $a$ 是正规式。
    3.  若 $R$ 和 $S$ 是正规式，则：
        *   $R | S$ (或/选择)
        *   $RS$ (连接)
        *   $R^*$ (克林闭包，任意次重复)
        *   $(R)$ (括号，优先级最高)
*   **优先级**：$*$ (闭包) > 连接 > $|$ (或)。同级运算从左到右。
*   **正规集**：正规式所描述的符号串的集合。

### 3.3 词法分析的计算模型：有限自动机 (DFA)
*   **理论基础**：计算理论证明，**每一个正规式都对应一个确定的有限自动机 (DFA)**。
*   **实现逻辑**：
    1.  用正规式描述词法规则。
    2.  将正规式转换为 DFA。
    3.  编写 DFA 的代码（状态转换逻辑）。
    4.  该代码即为词法分析器。
*   **意义**：实现了编译器的**自动生成**（无需手动编写识别代码，只需提供规则）。

### 3.4 C 语言词法示例
*   **标识符 (Identifier)**：字母或下划线开头，后跟字母、数字或下划线。
    *   正规式示意：`(letter | '_') (letter | digit | '_')*`
*   **关键字**：`if`, `while`, `int` 等（需查表确认）。
*   **运算符**：`+`, `-`, `*`, `/`, `==`, `>=` 等。
*   **界符**：`(`, `)`, `{`, `}`, `;`, `,` 等。
*   **注释**：`// ...` 或 `/* ... */` (通常被词法分析器过滤，不传给语法分析器)。

## 4. 编译器生成工具与环境

### 4.1 自动化工具
*   **Lex / Flex**：词法分析器生成工具。
    *   输入：词法规则（正规式）。
    *   输出：C 语言编写的词法分析器代码 (`lex.yy.c`)。
*   **Yacc / Bison**：语法分析器生成工具。
*   **编译器自动生成原理**：
    *   形式化描述 (正规式/文法) + 计算模型 (自动机) = 自动生成代码。
    *   这是计算机科学的魅力：**数学描述直接转化为可运行代码**。

### 4.2 主流编译器架构
*   **GCC (GNU Compiler Collection)**：
    *   支持多语言 (C, C++, Fortran 等)。
    *   支持多后端 (x86, ARM 等)。
    *   优化选项：`-O1`, `-O2`, `-O3` (优化级别越高，编译时间越长，代码效率越高)。
*   **LLVM**：
    *   模块化设计，前端与后端分离清晰。
    *   **IR (Intermediate Representation)**：LLVM IR 是其核心，强大的中间码优化器。
    *   支持挂载不同前端 (Clang for C/C++) 和不同后端。
    *   广泛应用于工业界、学术界及竞赛。

### 4.3 实验环境要求
*   **操作系统**：Linux (推荐 Ubuntu)。
*   **工具链**：GCC, LLVM, Make, GDB。
*   **标准参考**：C99 标准 (定义字母表、词法、文法规则)。

## 5. 重点总结与考点提示

1.  **编译器的底层逻辑**：计算模型（自动机/图灵机）+ 形式化语言理论。
2.  **词法分析核心**：
    *   任务：扫描、识别、输出 Token 及属性。
    *   数学工具：**正规式 (Regular Expression)**。
    *   计算模型：**有限自动机 (DFA)**。
3.  **正规式运算**：
    *   掌握 $*$ (闭包)、连接、$|$ (或) 的优先级与含义。
    *   理解 $\epsilon$ (空串) 与 $\emptyset$ (空集) 的区别。
    *   **考点**：根据自然语言描述写出正规式；正规式的等价变换。
4.  **自动生成**：理解 Lex 等工具的工作原理（规则 -> 自动机 -> 代码）。
5.  **大语言模型 (LLM) 与编译器的区别**：
    *   编译器：确定性、语义等价、基于规则形式化。
    *   LLM：概率性、基于统计、容错率高但确定性低（不适合系统级关键代码生成）。
