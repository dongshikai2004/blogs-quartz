## 1. 正则表达式 (Regular Expressions, RE) 回顾与练习

### 1.1 核心能力要求
*   **直观理解**：看到正则表达式能迅速描述其代表的语言集合（字符串特征）。
*   **计算准确性**：考试中涉及正则运算（如闭包、连接、并）的题目，需快速且准确计算，避免耗时过长导致做不完试卷。
*   **易错点**：
    *   幂运算分配律不成立（例如 $(A+B)^*$ 不等于 $A^* + B^*$）。
    *   复杂的嵌套结构容易算错，建议草稿纸演算展开。
    *   **例题分析**：$(A^*B^*)^*$ 等价于 $(A+B)^*$，即由A和B组成的任意字符串集合。

### 1.2 正则运算性质总结
| 运算 | 交换律 | 结合律 | 分配律 | 备注 |
| :--- | :---: | :---: | :---: | :--- |
| **并 (Union, \|)** | ✅ | ✅ | ❌ (对连接不可逆) | 类似于集合的并集 |
| **连接 (Concatenation)** | ❌ | ✅ | ✅ (左/右分配于并) | $A(B\|C) = AB \| AC$，但 $(A\|B)C \neq AC \| BC$ 的反例需注意上下文，通常连接对并满足分配律，但反之不成立 |
| **闭包 (Kleene Star, *)** | - | - | - | $\epsilon$ 与空集的闭包关系需理清 |

> **注意**：连接运算不可交换，因为字符串顺序至关重要（$AB \neq BA$）。

## 2. 正则表达式与有限自动机 (FA) 的等价转换

这是词法分析的核心理论基础：**正则表达式 $\iff$ NFA $\iff$ DFA**。

### 2.1 从正则表达式构造 DFA 的思路
1.  **理解语言特性**：分析正则式描述的字符串集合特征（如：以A开头，以A结尾，中间任意）。
2.  **构建 NFA/DFA 状态图**：
    *   **关键要素**：输入符号、状态转移函数 ($\delta$)、初态、终态。
    *   **循环的处理**：遇到闭包 ($*$) 必然存在环（Loop）。线性思维被打破，需考虑回溯或自环。
    *   **状态合并技巧**：
        *   若多个状态在接收相同输入后到达相同的后续状态集合，可能为等价状态。
        *   通过“排列分组”思想理解字母组合。

### 2.2 从 FA (NFA/DFA) 构造正则表达式 (状态消除法/阿登定理思路)
当给定一个复杂的自动机，求其对应的正则表达式时，使用**广义转换图 (GNFA)** 方法：

1.  **拓广自动机**：
    *   添加一个新的唯一**初态** $X$，通过 $\epsilon$ 边指向原初态。
    *   添加一个新的唯一**终态** $Y$，原所有终态通过 $\epsilon$ 边指向 $Y$。
2.  **状态消除步骤**：
    *   逐个移除中间状态。
    *   **基本替换规则**：
        *   **串联 (Connection)**：$R_1$ 接 $R_2$ $\rightarrow$ $R_1 R_2$
        *   **并联 (Union)**：$R_1$ 或 $R_2$ $\rightarrow$ $R_1 | R_2$
        *   **自环 (Star)**：状态 $i$ 上有自环 $R$，进出边分别为 $R_{in}, R_{out}$ $\rightarrow$ $R_{in} R^* R_{out}$
3.  **最终结果**：当只剩下初态 $X$ 和终态 $Y$ 时，它们之间的边上标记的正则表达式即为该自动机识别的语言。

> **考试重点**：掌握上述三个基本图形替换规则，能够手动推导简单自动机对应的正则式。

## 3. DFA 的最小化 (Minimization)

通过子集构造法得到的 DFA 可能包含冗余状态，需进行最小化处理以提高效率。

### 3.1 两类可优化状态
1.  **无关状态 (Unreachable States)**：
    *   定义：从初态出发，没有任何输入序列能到达的状态（无入度或不可达）。
    *   处理：**直接删除**。
2.  **等价状态 (Equivalent States)**：
    *   定义：两个状态 $p$ 和 $q$，对于**任意**输入字符串 $w$，从 $p$ 和 $q$ 出发要么同时到达终态，要么同时到达非终态。
    *   处理：**合并**为一个状态。

### 3.2 划分法 (Partitioning Algorithm / Hopcroft's Algorithm 简化版)
用于寻找等价状态的标准算法步骤：

1.  **初始划分**：将状态集划分为两个集合：**终态集** ($F$) 和 **非终态集** ($S-F$)。
    *   *原理*：终态和非终态显然不等价（一个接受空串，一个不接受）。
2.  **迭代细化 (Refinement)**：
    *   检查当前划分中的每个子集。
    *   对于子集 $G$ 中的状态，检查它们在输入字符 $a$ 下的转移目标是否属于同一个子集。
    *   如果 $p, q \in G$，但 $\delta(p, a)$ 和 $\delta(q, a)$ 落在不同的子集中，则 $p$ 和 $q$ **不等价**，需将 $G$ **分裂**。
3.  **终止条件**：当没有任何子集可以再分裂时，算法结束。每个子集内的状态互为等价状态，可合并。

> **解题技巧**：
> *   先找**不等价**比找等价更容易。
> *   观察状态图，若两个状态对某输入转移到明显不同的区域（如一个去终态，一个去死状态），则立即分割。
> *   非终态内部的等价划分是难点，需仔细追踪转移路径。

## 4. 词法分析器 (Lexer) 的设计与实现

### 4.1 词法分析器的功能
*   **输入**：源程序字符流。
*   **输出**：**Token 流** (属性字流)。
*   **Token 结构**：通常包含 `<种别码, 属性值>`。
    *   *传统结构*：`<Type, InnerCode>` (如 `<Keyword, "if">`)。
    *   *现代/LLM结构*：指向**符号表 (Symbol Table)** 的指针。这样可以在词法分析阶段完成标识符收集，供后续语义分析使用。

### 4.2 设计流程
1.  **定义 Token 类型**：关键字、标识符、常量、运算符、界符。
2.  **为每类 Token 构建 DFA**：
    *   关键字 DFA：通常是确定的字符串匹配。
    *   标识符 DFA：字母开头，后跟字母或数字。
    *   数字 DFA：数字序列等。
3.  **合并 DFA**：
    *   将所有类的 DFA 通过**并运算**合并为一个大的 DFA。
    *   解决冲突：优先匹配最长串 (Longest Match)，若长度相同，按优先级（如关键字优先于标识符）。
4.  **实现方式**：
    *   **手工编码**：适用于简单语言。
    *   **自动生成工具**：Flex/Lex。

### 4.3 代码实现策略
#### A. 数据中心法 (Table-Driven / 查表法)
*   **核心思想**：将 DFA 的状态转移表作为数据结构。
*   **结构**：
    *   **主表**：状态索引 -> 分表地址。
    *   **分表**：输入字符 -> 下一状态。
*   **优点**：逻辑与数据分离，修改正则只需改表，无需改代码；易于自动生成。
*   **应用**：Flex 等工具底层多采用此法。

#### B. 程序中心法 (Direct-Coded / 过程法)
 each state corresponds to a code block/function.
*   **核心思想 using `switch-case` or `if-else` branches to implement state transitions.
*   **优点**：执行效率略高（无查表开销），直观。
*   **缺点**：代码冗长，难以维护，难以自动生成。

## 5. 前沿视角：编译原理与大模型 (LLM)

*   **Tokenization 的重要性**：
    *   在大模型中，计费和处理的基本单位是 **Token**。
    *   Token 本质上是词法分析中的“词”，但针对自然语言进行了优化（如子词分割 Subword Tokenization）。
    *   高效的 Tokenizer 能降低计算成本，提升模型性能。
*   **缓冲技术**：
    *   编译器使用双缓冲区解决 lookahead 问题。
    *   LLM 处理长文本时也面临类似的缓冲和上下文窗口管理问题（KV Cache 等技术）。
*   **职业建议**：
    *   纯 CRUD 工程师易被 AI 替代。
    *   核心竞争力在于**底层逻辑**和**计算思维**：形式化描述问题 $\rightarrow$ 选择计算模型 $\rightarrow$ 设计数据结构与算法。
    *   推荐学习资源：CMU CSAPP (Computer Systems: A Programmer's Perspective)。

## 6. 下一步预告：语法分析 (Syntax Analysis)

### 6.1 为什么需要新的模型？
*   DFA/NFA 只能识别**正则语言**（线性结构，无记忆或有限记忆）。
*   编程语言的句子具有**嵌套结构**（如括号匹配 `((...))`），需要**无限记忆**来记录历史状态。
*   **结论**：DFA 无法识别上下文无关语言，需要更强大的计算模型。

### 6.2 新计算模型：下推自动机 (PDA, Pushdown Automaton)
*   **结构**：有限控制器 + **栈 (Stack)** + 输入带。
*   **能力**：栈提供了“历史记录”功能，可以处理递归和嵌套结构。
*   **对应文法**：**上下文无关文法 (CFG, Context-Free Grammar)**。

### 6.3 文法基础概念
*   **形式化定义**：$G = (V_N, V_T, P, S)$
    *   $V_N$：非终结符 (Non-terminals)，表示语法范畴（如 `<句子>`, `<主语>`）。
    *   $V_T$：终结符 (Terminals)，即 Token。
    *   $P$：产生式 (Productions)，规则。
    *   $S$：开始符号。
*   **表示法**：
    *   巴克斯范式 (BNF)：`<A> ::= <B> | <C>`
    *   乔姆斯基生成式：$A \rightarrow \beta$
*   **分析方法**：
    *   **自顶向下 (Top-Down)**：从开始符号推导到句子（预测分析，LL）。
    *   **自底向上 (Bottom-Up)**：从句子归约到开始符号（移进-归约，LR）。
