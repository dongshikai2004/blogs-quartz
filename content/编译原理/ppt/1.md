# 《编译原理与技术》第一章：编译引论 - 详尽整理
## 2. 编译程序的基本概念

### 2.1 定义
*   **编译程序 (Compiler)**：一种语言处理系统（系统软件），将某一种程序设计语言（源语言）写的程序翻译成逻辑等价的另一种语言（目标语言，通常是汇编或机器语言）的程序。
*   **核心功能**：
    1.  **翻译**：高级语言 $\rightarrow$ 低级语言。
    2.  **检查**：在翻译过程中检查源程序的语法和语义，报告错误和警告。
*   **地位**：居于计算机系统中最靠近硬件的一层，是人机交互的基本界面，也是软件学科的重要分支。

### 2.2 关键术语
| 术语 | 英文 | 定义 |
| :--- | :--- | :--- |
| **源语言** | Source Language | 用来编写源程序的语言（如 C, Pascal）。 |
| **源程序** | Source Program | 用源语言写的程序。 |
| **目标语言** | Target Language | 目标程序的描述语言（如汇编, 机器码）。 |
| **目标程序** | Target Program | 源程序经过编译后生成的程序。 |
| **宿主语言** | Host Language | 编译程序本身的实现语言。 |
| **宿主机/目标机** | Host/Target Machine | 编译程序运行或目标程序运行的环境。 |

### 2.3 编译 vs. 解释
*   **编译执行 (Compilation)**：
    *   过程：源程序 $\xrightarrow{编译程序}$ 目标程序 $\xrightarrow{计算机执行}$ 结果。
    *   特点：生成独立的目标程序，可重复执行，效率高。类似自然语言的**笔译**。
*   **解释执行 (Interpretation)**：
    *   过程：源程序 $\xrightarrow{解释程序}$ 直接逐句翻译并执行。
    *   特点：不生成目标程序，每次运行需重新解释，效率较低，但交互性好。类似自然语言的**口译**。

### 2.4 编译程序的表示
1.  **函数表示**：$T = C(S)$
    *   $T$: 目标程序, $C$: 编译程序, $S$: 源程序
2.  **T型图表示**：体现编译程序的三个要素（源语言、目标语言、宿主语言）。
3.  **符号表示**。

## 3. 编译程序的构造与实现途径

1.  **手工构造**: 使用机器语言、汇编语言或高级语言直接编写。
2.  **自动构造工具**: 使用 Lex (词法分析生成器), Yacc (语法分析生成器)。
3.  **自展方式 (Bootstrapping)**:
    *   利用一系列自展步骤形成编译程序。
    *   **例子**：构造 PC 机上 C 语言的编译器。
        *   Step 1: 用汇编写 C1 (子集) 编译器 A；用 C1 写 C2 (超集) 编译器 B；用 C2 写完整 C 编译器 C。
        *   Step 2: 用 A 编译 B，得到 C2 编译器 D。
        *   Step 3: 用 D 编译 C，得到完整 C 编译器 E。
4.  **移植方式 (Porting)**:
    *   利用具有自编译性的高级语言（如 C）编写新语言的编译器。
    *   **例子**：在已有 C 编译器的机器 A 上，用 C 语言编写 Pascal 编译器，编译后得到机器 A 上的 Pascal 编译器。

## 4. 编译程序的结构与组织 (核心部分)

编译过程通常分为**前端 (Front End)** 和 **后端 (Back End)**，中间通过**中间代码**连接。此外还有公共辅助模块。

### 4.1 主要阶段

#### 1. 词法分析 (Lexical Analysis / Scanner)
*   **任务**：对源程序字符流进行线性扫描，识别出具有独立意义的**单词 (Token)**。 for example: 标识符、关键字、常数、运算符、界符。
*   **输出**：属性字流 (Token Stream)。每个 Token 包含类别属性和值属性。
*   **示例**：
    *   源码：`a[index] = 12 * 3;`
    *   Token流：`(id, a)`, `(lbracket, [)`, `(id, index)`, `(rbracket, ])`, `(assign, =)`, `(int, 12)`, `(mult, *)`, `(int, 3)`, `(semicolon, ;)`

#### 2. 语法分析 (Syntax Analysis / Parser)
*   **任务**：依据语法规则，检查 Token 序列是否符合语法结构，识别语法范畴。
*   **输出**：**语法树 (Syntax Tree)** 或分析树 (Parse Tree)，一种层次结构。
*   **示例**：确认 `a[index] = 12 * 3` 是一个合法的赋值表达式语句。

#### 3. 语义分析 (Semantic Analysis)
*   **任务**：依据语义规则对语法树进行检查和处理（如类型检查、作用域检查）。这是最实质性的翻译准备阶段。
*   **输出**： annotated syntax tree (带标注的语法树) 或初步中间代码。

#### 4. 中间代码生成 (Intermediate Code Generation)
*   **定义**：一种结构简单、含义明确的记号系统，介于源语言和机器语言之间。
*   **特点**：易于产生，易于优化，易于翻译为目标代码。
*   **常见形式**：四元式 (Quadruples)、三元式、逆波兰表示。
*   **示例**：
    *   源码：`x = (a + b) * c`
    *   四元式序列：
        1.  `(+, a, b, T1)`
        2.  `(*, T1, c, T2)`
        3.  `(=, T2, _, x)`

#### 5. 代码优化 (Code Optimization)
*   **任务**：在不改变程序语义的前提下，对中间代码或目标代码进行变换，以提高运行效率（时间/空间）。
*   **示例**：
    *   循环不变代码外提：`x = x1 / cos(x2)` 若在循环中不变，移至循环外。
    *   删除无用代码。

#### 6. 目标代码生成 (Code Generation)
*   **任务**：根据中间代码和符号表信息，生成特定机器的汇编代码或机器代码。
*   **考虑因素**：寄存器分配、存储管理、指令选择。
*   **示例**：
    *   `MOV R0, index`
    *   `MUL R0, 2` (假设整型占2字节)
    *   `MOV R1, &a`
    *   `ADD R1, R0`
    *   `MOV *R1, 6` (即 `a[index] = 6`)

### 4.2 公共辅助模块
1.  **表格管理 (Table Management)**：
    *   管理符号表 (Symbol Table)、常量表等。
    *   各阶段通过查表获取变量类型、地址等信息。
2.  **出错处理 (Error Handling)**：
    *   发现、定位、定性并报告错误。
    *   尽可能从错误中恢复，以便继续编译后续代码。

### 4.3 遍 (Pass)
*   **定义**：对源程序或其内部表示从头到尾扫描一遍，并进行相关处理。
*   **影响因素**：宿主机内存大小、编译功能强弱、源语言复杂度、优化需求、实现环境。
*   **模型**：
    *   **单遍扫描**：词法、语法、语义、代码生成串联执行（节省空间，但功能受限）。
    *   **多遍扫描**：如 AHPL模拟器（两遍），第一遍分析，第二遍生成。现代编译器通常为多遍以支持复杂优化。

## 5. 编译阶段与自然语言翻译的类比

| 编译阶段 | 自然语言翻译过程 |
| :--- | :--- |
| 词法分析 | 查字典、识别单词 |
| 语法分析 | 分析识别句子结构 |
| 语义分析/中间代码生成 | 翻译初稿 |
| 代码优化 | 对初稿润色修饰 |
| 目标代码生成 | 译文定稿 |

## 6. 典型编译器结构实例

### 6.1 GCC (GNU Compiler Collection) 结构
*   **前端 (Front End)**：针对不同语言（C, C++, Fortran等）有各自的 Parser，生成统一的语法树 (Syntax Tree)。
*   **中间表示 (IR)**：转换为 **RTL (Register Transfer Language)** 表示。
*   **优化 (Optimization)**：
    *   机器无关优化：Jump Threading, Common Subexpression Elimination 等。
    *   机器相关优化：基于 `machine.md` (机器描述文件) 和 `tm.h`。
*   **后端 (Back End)**：
    *   指令选择 (Instruction Selection)
    *   寄存器分配 (Register Allocation) - 使用干扰图 (Interference Graph)
    *   代码发射 (Code Emission) -> 汇编代码
*   **后续步骤**：Assembler (汇编) -> Linker (链接) -> 可执行文件。

### 6.2 StarCore / SCC 结构
*   **输入**：C Source Files
*   **预处理**：Preprocessor (.i)
*   **前端 (CFE)**：生成 IR file (.eln/.eld)
*   **优化器 (Optimizer)**：
    *   High-Level Optimizations
    *   Low-Level Optimizations
    *   Global/Non-Global Optimization
*   **后端**：生成 Optimized Assembly Code (.asm)
*   **汇编与链接**：ASSEMBLER -> LINKER -> Executable Program

### 6.3 Tiger Compiler (基于 Appel 的书)
*   结构清晰，常用于教学：
    *   Lexer -> Parser -> Semantic Analysis -> IR Trees -> Canonicalization -> Instruction Selection -> Register Allocation (Graph Coloring) -> Code Emission.

## 7. 第一章总结 (Key Takeaways)
1.  **编译程序是什么**：将源语言翻译为目标语言的系统软件。
2.  **基本流程**：词法 $\rightarrow$ 语法 $\rightarrow$ 语义 $\rightarrow$ 中间代码 $\rightarrow$ 优化 $\rightarrow$ 目标代码。
3.  **核心数据结构**：符号表、语法树、中间代码（四元式/RTL）。
4.  **构造方法**：自展、移植、工具生成。
5.  **重要概念**：源/目标/宿主语言，遍 (Pass)，前端/后端划分。
