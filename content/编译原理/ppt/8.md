# 第8章 代码优化 (Code Optimization)

## 8.1 代码优化综述

### 8.1.1 代码优化概述
*   **定义**：在不改变程序运行效果（语义等价）的前提下，对编译的程序进行变换，生成更高效的目标代码。
    *   **等价**：不改变程序执行结果。
    *   **变换**：引起程序形式上的变动。
*   **优化目的**：
    *   **时间效率**：源程序运行时间尽可能短。
    *   **空间效率**：程序及数据所占空间尽可能少。
*   **为什么要实施优化**：
    1.  优化程度是衡量编译器质量的重要指标。
    2.  无法苛求程序员掌握所有编程技巧和优化手段。
    3.  编译器统一处理各种源程序，存在固有缺陷，需要通过优化弥补。
*   **改进途径**：改进算法、源程序级等价变换、利用系统库、编译时优化等。

### 8.1.2 优化技术分类

#### 一、按涉及源程序的范围分类
1.  **局部优化**：在基本块（Basic Block）内进行的优化。
2.  **循环优化**：在隐式或显式循环体内进行的优化。
3.  **全局优化**：在整个源程序范围内进行的优化。

#### 二、按编译阶段分类
1.  **中间代码级优化**：目标代码生成前的优化。
2.  **目标代码级优化**：目标代码生成后的优化。

#### 三、按具体实现技术分类（常见优化方法）
1.  **常量合并与传播 (Constant Folding & Propagation)**
    *   将常量运算在编译期计算出来。
    *   *例*：`X=2; Y=X+10;` $\rightarrow$ `X=2; Y=12;`
2.  **公共子表达式删除 (Common Subexpression Elimination)**
    *   如果表达式 `E` 之前已计算过，且变量值未变，则重用之前的结果。
    *   *例*：`d=e+f+g; y=e+f+z;` $\rightarrow$ 引入临时变量 `x=e+f`，则 `d=x+g; y=x+z;`
3.  **循环不变量外提 (Code Motion / Loop Invariant Code Motion)**
    *   将循环中值不变的运算移到循环体外。
    *   *例*：`for(i...){ d[i]=2*b+1; }` ($b$不变) $\rightarrow$ `z=2*b+1; for(i...){ d[i]=z; }`
4.  **无用赋值删除 (Dead Store Elimination)**
    *   如果一个变量被赋值后，在下次赋值前未被使用，则删除前面的赋值。
    *   *例*：`a=5; a=7;` $\rightarrow$ `a=7;`
5.  **无用转移语句删除**
    *   删除不必要的跳转。
    *   *例*：`if(x) ... else goto J1; J1: goto J2;` $\rightarrow$ `if(!x) goto J2;`
6.  **死代码删除 (Dead Code Elimination)**
    *   删除永远执行不到或对结果无影响的代码。
    *   *例*：`char c; if(c>300)...` (char类型最大值通常小于300，条件永假) $\rightarrow$ 直接执行 `else` 分支。
7.  **函数内嵌 (Function Inlining)**
    *   将函数调用替换为函数体代码，减少调用开销。
8.  **运算强度削弱 (Strength Reduction)**
    *   用低强度的运算替代高强度运算（如用加法替代乘法）。
    *   *静态步长*：`table[i*4]` 在循环中可转化为 `t1=t1+4`。
    *   *动态步DP*：`(i+step)*4` 转化为 `t1 = t1 + step*4`。
9.  **复合变量循环转换**
    *   针对多维数组或复杂下标的循环，简化地址计算。

### 8.1.3 具优化功能编译器的组织
*   **结构**：前端 $\rightarrow$ **代码优化器** $\rightarrow$ 代码生成器
*   **优化器内部模块**：
    *   控制流分析
    *   数据流分析
    *   代码变换

## 8.2 局部优化 (Local Optimization)

### 8.2.1 基本块定义与划分
*   **基本块 (Basic Block)**：程序中顺序执行的最大语句序列。
    *   **特点**：惟一入口、惟一出口；内部无跳转进入或跳出。
*   **划分算法**：
    1.  **确定入口语句**：
        *   程序的第一个语句。
        *   由条件/无条件转移语句指向的语句。
        *   紧跟在条件/无条件转移语句后面的语句。
    2.  **构造基本块**:从每个入口语句开始，直到下一个入口语句之前（不含）或程序结束。
    3.  **删除不可达代码**：不属于任何基本块的语句。

### 8.2.2 程序的控制流图 (Control Flow Graph, CFG)
*   **定义**：$G=(N, E, n_0)$
    *   $N$：结点集合，每个结点是一个基本块。
    *   $n_0$：首结点（程序第一个基本 block）。
    *   $E$：有向边集合。
*   **边的形成条件**：
    1.  基本块 $B_i$ 之后紧接着 $B_j$，且 $B_i$ 末尾不是无条件转移或停止语句。
    2.  $B_i$ 末尾是 `goto S` 或 `if...goto S`，且 $S$ 是 $B_j$ 的入口。

### 8.2.3 基本块的 DAG 表示与应用
*   **DAG (Directed Acyclic Graph)**：无环路有向图，用于表示基本块内四元式的依赖关系。
*   **DAG 结点标记**：
    *   **叶结点**：标识符（初值加下标0，如 $A_0$）或常量。
    *   **内部结点**：运算符（如 $+$, $*$, $-$）。
    *   **附加标识符**：结点上可附加多个变量名，表示这些变量在该点具有相同的值。
*   **四元式与 DAG 对应**：
    *   0型 `A=B`：复制指针。
    *   1型 `A=op B`：单目运算。
    *   2型 `A=B op C`：双目运算。
*   **DAG 构造算法 (Algorithm 8.1)**：
    1.  **构造叶结点**：若操作数结点不存在则创建。
    2.  **常量合并**：若操作数均为常量，直接计算结果，创建常量结点。
    3.  **捕捉公共子表达式**：检查是否已有相同运算符且后继结点相同的内部结点。若有，复用该结点；若无，创建新结点。
    4.  **处理赋值**：将左部变量名附加到结果结点上。若该变量名之前附加在其他结点上，先删除旧关联。
*   **应用**：
    *   根据 DAG 重新生成四元式序列，即可得到优化后的代码（删除了冗余计算和无效赋值）。
    *   若知道某些变量在块后不再活跃，可省略最后的复制操作。

## 8.3 控制流分析与循环查找

### 8.3.1 循环的定义
*   **循环 (Loop)**：程序流图中具有**惟一入口结点**的**强连通**子图。
    *   **强连通性**：子图中任意两点间都有通路。
    *   **惟一入口**：子图外的结点只能通向入口结点，不能通向子图内其他结点。

### 8.3.2 必经结点 (Dominators)
*   **定义**：若从首结点 $n_0$ 到结点 $n_j$ 的所有通路都经过 $n_i$，则称 $n_i$ 支配 $n_j$，记作 $n_i \ DOM \ n_j$。
*   **性质**：自反性、传递性、反对称性。
*   **必经结点集 $D(n)$**：结点 $n$ 的所有必经结点集合。

### 8.3.3 回边 (Back Edge)
*   **定义**：若有向边 $a \rightarrow b$，且满足 $b \ DOM \ a$（即 $b$ 支配 $a$），则该边为回边。
*   **作用**：回边标志着循环的存在。

### 8.3.4 利用回边查找循环
*   **算法步骤**：
    1.  计算流图中所有结点的必经结点集 $D(n)$。
    2.  找出所有回边 $\langle n, d \rangle$。
    3.  对于每条回边 $\langle n, d \rangle$，构造循环 $L$：
        *   $L$ 初始包含 $d$ 和 $n$。
        *   寻找所有能到达 $n$ 且不经过 $d$ 的结点，加入 $L$。
        *   $d$ 是该循环的惟一入口。

## 8.4 数据流分析基础 (Data Flow Analysis)

数据流分析旨在收集程序范围内的信息，用于全局优化。

### 8.4.1 基本概念
*   **点 (Point)**：语句在基本块中的位置（入口前、出口后、语句间）。
*   **定值点 (Definition, def)**：变量获得值的位置（赋值、输入、函数参数结合）。
*   **引用点 (Use, use)**：变量被使用的位置。
*   **到达-定值 (Reaching Definitions)**：变量 $A$ 在点 $d$ 的定值到达点 $p$，指存在从 $d$ 到 $p$ 的通路，且通路上没有对 $A$ 的其他定值。
*   **UD链 (Use-Definition Chain)**：在引用点 $P$，所有能到达 $P$ 的变量 $A$ 的定值点集合。
*   **DU链 (Definition-Use Chain)**：在定值点 $P，该定值能到达的所有变量 $A$ 的引用点集合。
*   **活跃变量 (Live Variable)**：在点 $P$，若存在从 $P$ 开始的通路引用了 $A$ 在 $P$ 的值，则 $A$ 在 $P$ 活跃；否则死亡。

### 8.4.2 数据流方程通用形式
$$ out[B] = gen[B] \cup (in[B] - kill[B]) $$
*   $gen[B]$：块 $B$ 产生/生成的信息。
*   $kill[B]$：块 $B$ 注销/杀死的信息。
*   **方向**：
    *   **前向分析 (Forward)**：由 $in$ 推 $out$（如：到达-定值，可用表达式）。$in[B] = \bigcup_{P \in Pred(B)} out[P]$
    *   **后向分析 (Backward)**：由 $out$ 推 $in$（如：活跃变量）。$out[B] = \bigcup_{S \in Succ(B)} in[S]$

### 8.4.3 典型数据流方程

#### 1. 到达-定值 (Reaching Definitions) - 前向
*   **用途**：常量传播、删除无用赋值、构建UD链。
*   **方程**：
    *   $out[B] = gen[B] \cup (in[B] - kill[B])$
    *   $in[B] = \bigcup_{P \in Pred(B)} out[P]$
*   **Gen/Kill 含义**：
    *   $gen[B]$：B中定值且能到达出口的定值点。
    *   $kill[B]$：B中定值的变量在其他地方的所有定值点（被B中的新定值覆盖）。
*   **求解**：迭代法，初始 $in[B]=\emptyset$，直到收敛。

#### 2. 可用表达式 (Available Expressions) - 前向
*   **定义**：表达式 $x+y$ 在点 $P$ 可用，指从初始结点到 $P$ 的每条路径上都计算了 $x+y$，且最后计算后 $x,y$ 未被重新定值。
*   **用途**：公共子表达式删除。
*   **方程**：
    *   $out[B] = gen[B] \cup (in[B] - kill[B])$
    *   $in[B] = \bigcap_{P \in Pred(B)} out[P]$  (**注意：这里是交集 $\cap$**)
    *   初始：$in[Start] = \emptyset$, 其他 $in[B] = U$ (全集)。

#### 3. 活跃变量 (Live Variables) - 后向
*   **用途**：寄存器分配、删除死代码。
*   **方程**：
    *   $in_L[B] = use_L[B] \cup (out_L[B] - def_L[B])$
    *   $out_L[B] = \bigcup_{S \in Succ(B)} in_L[S]$
*   **Gen/Kill (此处为 Use/Def)**：
    *   $use_L[B]$：B中引用且在此之前未定值的变量。
    *   $def_L[B]$：B中定值且在此之前未引用的变量。

## 8.5 循环优化的实施

循环是程序执行最频繁的部分，优化收益最大。

### 8.5.1 准备工作
1.  **查找循环**：利用回边确定循环结构。
2.  **建立前置结点 (Preheader)**：
    *   在循环入口结点前建立一个新基本块。
    *   将原指向入口的外部和内部回边都指向该前置结点。
    *   前置结点唯一后继是循环入口。
3.  **数据流分析**：获取UD链、DU链、活跃变量信息等。

### 8.5.2 代码外提 (Code Motion)
将循环不变运算移到循环前置结点中。

*   **不变运算判定 (Algorithm X1)**：
    1.  运算对象是常数。
    2.  运算对象的定值点在循环外（通过UD链判断）。
    3.  运算对象只有一个到达定值点，且该定值点也是不变运算。
    *   *迭代标记*，直到没有新的不变运算被发现。

*   **外提条件 (Algorithm X2)**：
    对于不变运算 $S: A = B \ op \ C$，需满足以下任一组合：
    1.  **安全性条件**：
        *   (i) $S$ 所在结点是循环所有出口结点的**必经结点**。
        *   (ii) $A$ 在循环中其他地方**未再定值**。
        *   (iii) 循环中所有 $A$ 的引用点只有 $S$ 中的定值能到达。
    2.  **或者**：
        *   $A$ 在离开循环后**不再是活跃的**（通过活跃变量分析得知）。
        *   且满足上述 (ii) 和 (iii)。

*   **执行顺序**：若 $S$ 的操作数是在循环中定值的不变运算，必须先外提操作数的定值语句。

### 8.5.3 强度削弱与删除归纳变量

#### 1. 基本概念
*   **基本归纳变量 (Basic Induction Variable)**：循环中仅有 $I = I \pm C$ 形式赋值的变量（$C$为循环不变量）。
*   **归纳变量 (Induction Variable)**：变量 $J$ 的值总是基本归纳变量 $I$ 的线性函数：$J = C_1 * I \pm C_2$。$J$ 与 $I$ 同族。

#### 2. 优化步骤
1.  **查找归纳变量 (W1)**：
    *   找出所有基本归纳变量。
    *   寻找形如 $K = J * C, K = J \pm C$ 等的变量，若 $J$ 是归纳变量且满足定值约束，则 $K$ 也是同族归纳变量。
2.  **强度削弱 (W2)**：
    *   对于 $J = C_1 * I + C_2$，引入新变量 $S$。
    *   初始化：在循环入口前设置 $S = C_1 * I_{initial} + C_2$。
    *   迭代：在 $I = I + n$ 之后，添加 $S = S + C_1 * n$。
    *   替换：用 $S$ 替换原代码中对 $J$ 的计算。
    *   *效果*：将乘法转换为加法。
3.  **删除归纳变量 (W3)**：
    *   如果基本归纳变量 $I$ 仅用于计算同族其他变量或作为循环终止条件，且存在同族变量 $J$ 可以替代其测试功能。
    *   将循环条件中的 $I$ 替换为 $J$ 的线性变换。
    *   删除对 $I$ 的更新语句（如果 $I$ 不再被引用）。

### 8.5.4 综合考量
*   **优化次序**：通常先局部优化，再全局/循环优化。
*   **代价平衡**：优化本身也消耗编译时间和空间。需权衡目标代码效率提升与编译开销。
*   **重点**：最内层循环优化收益最高。
