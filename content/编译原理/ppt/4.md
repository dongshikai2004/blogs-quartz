# 第4章 语法分析 (Syntax Analysis) —— 自上而下分析法

## 1. 语法分析综述 (4.1)

### 1.1 语法分析程序的功能
*   **定义**：完成语法分析任务的程序称为**语法分析器** (Parser)。
*   **输入**：词法分析器输出的单词符号流（源程序 $L_1$ 形式）。
*   **依据**：源语言的语法规则（文法 $G$）。
*   **任务**：
    1.  从词法分析结果中识别出相应的语法范畴。
    2.  进行语法检查：判定输入串 $\alpha$ 是否属于文法 $G$ 产生的语言 $L(G)$，即 $\alpha \in L(G)$ ?
    3.  报告和处理语法错误。
    4.  生成分析树或推导序列作为输出。

### 1.2 语法分析方法分类
主要分为两大类：
1.  **自上而下分析法 (Top-Down Parsing)**：
    *   **方向**：从开始符号 $S$ 出发，推导出输入串 $r$ ($S \Rightarrow^* r$)。
    *   **本质**：是一种**产生**的方法，面向目标。
    *   **核心**：选择产生式的合适候选式对句型中的非终结符进行替换（最左推导）。
    *   **典型方法**：递归下降分析、LL(1)分析。
2.  **自下而上分析法 (Bottom-Up Parsing)**：
    *   **方向**：从输入串 $r$ 开始，归约到开始符号 $S$ ($r \Rightarrow^* S$)。
    *   **本质**：是一种**辨认**的方法，基于目标。
    *   **核心**：寻找可归约串（句柄），用产生式左部代替右部。
    *   **典型方法**：算符优先分析 (OPG)、LR分析。

> **示例对比**：
> 文法 $G$: $S \to aA$, $A \to BaA \mid \varepsilon$, $B \to + \mid - \mid * \mid ,$
> 输入串 $r$: `a*a+a`
> *   **自上而下**：$S \Rightarrow aA \Rightarrow aBaA \Rightarrow a*aA \Rightarrow a*aBaA \Rightarrow a*a+aA \Rightarrow a*a+a$
> *   **自下而上**：`a*a+a` $\Leftarrow$ `a*Ba` $\Leftarrow$ `a*A` ... $\Leftarrow$ $S$ (具体步骤见课件例4.2)

## 2. 不确定的自上而下分析法 (4.2)

一般的自上而下分析存在**不确定性**，主要原因及解决方法如下：

### 2.1 不确定性的原因
1.  **左递归 (Left Recursion)**：
    *   **现象**：若文法含有 $A \to A\alpha$，在推导时会出现 $A \Rightarrow A\dots$，导致无限循环，无法终止。
    *   **类型**：直接左递归、间接左递归。
2.  **回溯 (Backtracking)**：
    *   **现象**：当非终结符有多个候选式时，需逐个试探。若选错，需退回重新选择，效率极低。
    *   **原因**：候选式的 **FIRST集** 相交，或存在公共左因子。

### 2.2 解决方法

#### 一、消除文法的左递归

**1. 消除直接左递归**
若产生式为 $P \to P\alpha_1 \mid P\alpha_2 \mid \dots \mid P\alpha_n \mid \beta_1 \mid \dots \mid \beta_m$ （其中 $\beta_i$ 不以 $P$ 开头）：
*   **改写规则**：
    $$
    \begin{aligned}
    P &\to \beta_1 P' \mid \beta_2 P' \mid \dots \mid \beta_m P' \\
    P' &\to \alpha_1 P' \mid \alpha_2 P' \mid \dots \mid \alpha_n P' \mid \varepsilon
    \end{aligned}
    $$
*   **示例**：
    $E \to E+T \mid T$ 改写为：
    $E \to TE'$
    $E' \to +TE' \mid \varepsilon$

**2. 消除间接左递归**
*   **算法思路**：
    1.  将非终结符排序 $A_1, A_2, \dots, A_n$。
    2.  对于每个 $A_i$，将其产生式中所有形如 $A_i \to A_j \gamma$ ($j < i$) 的项，用 $A_j$ 的所有候选式代入展开。
    3.  消除 $A_i$ 中可能产生的直接左递归。
    4.  简化文法，去除多余规则。
*   **前提条件**: 文法不含回路（$P \Rightarrow^+ P$）且不含以 $\varepsilon$ 为右部的产生式（或者先处理 $\varepsilon$ 产生式）。

#### 二、消除回溯 (提取左公因子)

**1. FIRST 集合定义**
*   $FIRST(\alpha) = \{ a \mid \alpha \Rightarrow^* a\dots, a \in V_T \}$
*   若 $\alpha \Rightarrow^* \varepsilon$，则 $\varepsilon \in FIRST(\alpha)$。

**2. 不带回溯的条件**
对于非终结符 $A$ 的产生式 $A \to \alpha_1 \mid \alpha_2 \mid \dots \mid \alpha_n$：
*   若所有 $\alpha_i$ 均不能推出 $\varepsilon$，且 $FIRST(\alpha_i)$ 两两互不相交，则无需回溯。
*   若有 $\alpha_i \Rightarrow \varepsilon$，则需结合 FOLLOW 集判断（见LL(1部分）。

**3. 提取左公因子**
若候选式有公共前缀（左公因子），如 $A \to \delta\beta_1 \mid \delta\beta_2 \mid \dots$：
*   **改写规则**：
    $$
    \begin{aligned}
    A &\to \delta A' \\
    A' &\to \beta_1 \mid \beta_2 \mid \dots
    \end{aligned}
    $$
*   **目的**：使各候选式的 FIRST 集不相交，从而确定性地选择产生式。

## 3. 递归下降分析法与递归下降分析器 (4.3)

### 3.1 基本概念
*   **定义**：由一组递归过程（函数）组成的分析程序，每个过程对应文法的一个非终结符。
*   **要求**：文法必须是 **LL(1) 文法**（无左递归、无回溯/冲突）。
*   **工作原理**：
    *   主程序调用开始符号对应的过程。
    *   过程中根据当前输入符号，选择对应的候选式进行匹配。
    *   遇到终结符则与输入串匹配并前进指针；遇到非终结符则递归调用对应过程。

### 3.2 构造方法
1.  为每个非终结符编写一个子程序。
2.  子程序内部逻辑：
    *   查看当前输入符号 `lookahead`。
    *   根据 `lookahead` 所属的 FIRST 集，决定使用哪个候选式。
    *   依次处理候选式中的符号序列。

### 3.3 状态图辅助设计
*   为每个非终结符构建有限自动机（FA）状态图。
*   初态为进入点，终态为返回点。
*   弧标记为终结符或非终结符。
*   可通过状态图的化简来优化递归下降分析器的结构。

### 3.4 示例
文法：
$E \to TE'$
$E' \to +TE' \mid \varepsilon$
$T \to FT'$
$T' \to *FT' \mid \varepsilon$
$F \to (E) \mid i$

伪代码示例：
```c
void E() { T(); E_prime(); }
void E_prime() {
    if (lookahead == '+') { match('+'); T(); E_prime(); }
    // else do nothing (epsilon)
}
void T() { F(); T_prime(); }
void T_prime() {
    if (lookahead == '*') { match('*'); F(); T_prime(); }
}
void F() {
    if (lookahead == 'i') { match('i'); }
    else if (lookahead == '(') {
        match('('); E(); match(')');
    }
    else error();
}
```

## 4. LL(1) 分析法与 LL(1) 分析器 (4.4)

### 4.1 LL(1) 的含义
*   **L** (Left-to-right)：从左向右扫描输入串。
*   **L** (Leftmost derivation)：产生最左推导。
*   **1** (Lookahead 1)：只需向前看一个输入符号即可确定使用哪个产生式。

### 4.2 LL(1) 分析器的逻辑结构
LL(1) 分析器通常采用**预测分析表**驱动的方式，包含三个主要部分：
1.  **分析栈 (Stack)**：
    *   存放文法符号。
    *   初始状态：栈底为 `#`，栈顶为开始符号 $S$。
2.  **输入缓冲区 (Input Buffer)**：
    *   存放待分析的输入串，以 `#` 结尾。
3.  **总控程序 (Driver) + LL(1) 分析表 (M)**：
    *   **分析表 M[A, a]**：二维数组，行是非终结符，列是终结符（含 `#`）。
    *   表项内容：一条产生式 $A \to \alpha$ 或 `error`。

### 4.3 LL(1) 分析算法
1.  初始化：栈置为 `#S`，输入指针指向第一个字符。
 each step:
    *   设栈顶符号为 $X$，当前输入符号为 $a$。
    *   **若 $X$ 是终结符**：
        *   若 $X == a == \#$：分析成功，停止。
        *   若 $X == a \neq \#$：弹出 $X$，输入指针后移。
        *   若 $X \neq a$：报错。
    *   **若 $X$ 是非终结符**：
        *   查表 $M[X, a]$。
        *   若 $M[X, a] = X \to Y_1 Y_2 \dots Y_k$：
            *   弹出 $X$。
            *   将 $Y_k, \dots, Y_2, Y_1$ **逆序**压入栈（保证 $Y_1$ 在栈顶）。
            *   *(注：若产生式为 $X \to \varepsilon$，则只弹出 $X$，不压入任何符号)*。
        *   若 $M[X, a] = \text{error}$：报错。

### 4.4 LL(1) 分析表的构造

构造分析表的关键是计算 **FIRST** 集和 **FOLLOW** 集。

#### 1. FIRST 集构造算法
*   若 $X$ 是终结符，$FIRST(X) = \{X\}$。
*   若 $X \to \varepsilon$，$\varepsilon \in FIRST(X)$。
*   若 $X \to Y_1 Y_2 \dots Y_k$：
    *   将 $FIRST(Y_1)$ 中非 $\varepsilon$ 元素加入 $FIRST(X)$。
    *   若 $\varepsilon \in FIRST(Y_1)$，则将 $FIRST(Y_2)$ 中非 $\varepsilon$ 元素加入，依此类推。
    *   若所有 $Y_i$ 都能推出 $\varepsilon$，则 $\varepsilon \in FIRST(X)$。

#### 2. FOLLOW 集构造算法
$FOLLOW(A)$ 表示在句型中紧跟在 $A$ 后面的终结符集合。
*   **规则**：
    1.  对于开始符号 $S$，$\# \in FOLLOW(S)$。
    2.  若有产生式 $A \to \alpha B \beta$：
        *   将 $FIRST(\beta)$ 中非 $\varepsilon$ 元素加入 $FOLLOW(B)$。
    3.  若有产生式 $A \to \alpha B$ 或 $A \to \alpha B \beta$ 且 $\beta \Rightarrow^* \varepsilon$：
        *   将 $FOLLOW(A)$ 中的所有元素加入 $FOLLOW(B)$。

#### 3. 构造分析表 M 的算法
对于文法 $G$ 的每个产生式 $A \to \alpha$：
1.  对于每个 $a \in FIRST(\alpha)$，置 $M[A, a] = A \to \alpha$。
2.  若 $\varepsilon \in FIRST(\alpha)$，则对于每个 $b \in FOLLOW(A)$，置 $M[A, b] = A \to \alpha$。
3.  若 $\varepsilon \in FIRST(\alpha)$ 且 $\# \in FOLLOW(A)$，置 $M[A, \#] = A \to \alpha$。
4.  表中其余未定义项填 `error`。

### 4.5 LL(1) 文法的判定
**定义**：若文法 $G$ 的 LL(1) 分析表 $M$ 中**不含多重定义入口**（即每个格子最多只有一个产生式），则 $G$ 是 LL(1) 文法。

**充要条件**：
对于文法中任意非终结符 $A$ 的两个不同产生式 $A \to \alpha \mid \beta$：
1.  $FIRST(\alpha) \cap FIRST(\beta) = \emptyset$。
2.  若 $\beta \Rightarrow^* \varepsilon$，则 $FIRST(\alpha) \cap FOLLOW(A) = \emptyset$。
3.  若 $\alpha \Rightarrow^* \varepsilon$，则 $FIRST(\beta) \cap FOLLOW(A) = \emptyset$。
    *(简言之：任何两个候选式的可选首符集不相交；若能推空，其首符集与 Follow 集也不相交)*

**性质**：
1.  LL(1) 文法一定是无二义性的。
2.  左递归文法一定不是 LL(1) 文法。
3.  存在非 LL(1) 的语言。
4.  可以判定任意文法是否为 LL(1)，但无法判定任意 CFL 是否能由 LL(1) 文法产生。

## 5. 重点习题与实例解析

### 实例：表达式文法
原文法（含左递归和二义性）：
$E \to E+E \mid E*E \mid (E) \mid i$

**步骤1：消除二义性（引入优先级）**
$E \to E+T \mid T$
$T \to T*F \mid F$
$F \to (E) \mid i$

**步骤2：消除左递归**
$E \to TE'$
$E' \to +TE' \mid \varepsilon$
$T \to FT'$
$T' \to *FT' \mid \varepsilon$
$F \to (E) \mid i$

**步骤3：计算 FIRST 和 FOLLOW**
*   $FIRST(E) = FIRST(T) = FIRST(F) = \{ (, i \}$
*   $FIRST(E') = \{ +, \varepsilon \}$
*   $FIRST(T') = \{ *, \varepsilon \}$
*   $FOLLOW(E) = FOLLOW(E') = \{ ), \# \}$
*   $FOLLOW(T) = FOLLOW(T') = \{ +, ), \# \}$
*   $FOLLOW(F) = \{ *, +, ), \# \}$

**步骤4：构造分析表**
| | i | + | * | ( | ) | # |
|---|---|---|---|---|---|---|
| **E** | $E \to TE'$ | | | $E \to TE'$ | | |
| **E'** | | $E' \to +TE'$ | | | $E' \to \varepsilon$ | $E' \to \varepsilon$ |
| **T** | $T \to FT'$ | | | $T \to FT'$ | | |
| **T'** | | $T' \to \varepsilon$ | $T' \to *FT'$ | | $T' \to \varepsilon$ | $T' \to \varepsilon$ |
| **F** | $F \to i$ | | | $F \to (E)$ | | |

*(注：表中空白处为 error)*

### 判断题解析
1.  **语法分析的任务是分析语句是如何构成程序的。** -> **错**。任务是识别语法结构，检查合法性，而非分析程序构成逻辑。
2.  **S->qQ|q, Q->cQd|ε 是 LL(1) 文法。** -> **错**。$S$ 的两个候选式 $qQ$ 和 $q$ 有公共左因子 $q$，且 $FIRST(qQ)=\{q\}, FIRST(q)=\{q\}$ 相交，需提取左公因子。
3.  **LL(1)中的"1"指当前指针P++所指的字符。** -> **错**。指向前看 **1** 个符号。
4.  **“下”指的是被分析的源程序串。** -> **对**。（自上而下：从树根到树叶/源串；自下而上：从源串到树根）。
5.  **递归下降属于自上而下。** -> **对**。

### 填空题解析
*   **FOLLOW(A) 计算**：
    文法：$S \to AB \mid bC$, $A \to \varepsilon \mid b$, $B \to \varepsilon \mid aD$, $C \to AD \mid b$, $D \to aS \mid c$
    *   $FOLLOW(S) = \{ \# \}$
    *   $S \to AB \implies FOLLOW(S) \subseteq FOLLOW(B)$, $FIRST(B)-\{\varepsilon\} \subseteq FOLLOW(A)$
    *   $S \to bC \implies FOLLOW(S) \subseteq FOLLOW(C)$
    *   $C \to AD \implies FOLLOW(C) \subseteq FOLLOW(D)$, $FIRST(D)-\{\varepsilon\} \subseteq FOLLOW(A)$
    *   $D \to aS \implies FOLLOW(D) \subseteq FOLLOW(S)$
    *   推导得：$FOLLOW(S)=FOLLOW(B)=FOLLOW(D)=FOLLOW(C)=\{\#\}$
    *   $FOLLOW(A) = FIRST(B) \cup FIRST(D) \cup FOLLOW(S) \dots$ 具体需仔细推导，课件中给出 $FOLLOW(A)=\{ \#, a, c \}$ (注：此处需根据具体依赖关系验证，课件结论为 $\{ \#, a, c \}$)。
    *   $FIRST(S) = FIRST(A) \cup FIRST(b) \cup \dots = \{ b, a, c, \varepsilon \}$? 课件中 $FIRST(S)$ 填空答案为 `b, a, ε` (可能未考虑C推导出的c，需依具体题目上下文，通常 $S \to bC$ 贡献 $b$, $S \to AB$, $A \to b/\varepsilon, B \to aD/\varepsilon$ 贡献 $b, a, \varepsilon$)。
