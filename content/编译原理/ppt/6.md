# 第6章 语义分析与中间代码生成

## 6.1 语义分析综述

### 1. 地位与作用
*   **位置**：位于编译器前端（分析）与后端（综合）之间，是编译程序最实质性的工作。
*   **意义**：第一次对源程序的语义作出解释，引起源程序质的变化。
*   **任务**：按照语法分析器识别的语法范畴进行语义检查和处理，产生相应的中间代码或目标代码。

### 2. 中间代码 (Intermediate Code)
*   **定义**：介于源语言和目标代码之间的一种代码。
*   **引入目的**：
    1.  方便生成目标代码；
    2.  便于优化；
    3.  便于移植（使编译器前端与后端分离）。

---

## 6.2 语法制导翻译与属性文法

### 1. 基本概念
*   **语法制导翻译 (Syntax-Directed Translation)**：为文法的每一个产生式配一个相应的语义子程序（或语义规则描述的语义动作），并在语法分析的同时调用它。
*   **属性文法 (Attribute Grammar)**：把语义引入文法，给产生式中的文法符号附加“属性”，构成涉及语义的翻译文法。
    *   **形式定义**：$A = (G, V, F)$
        *   $G$：一般为二型文法（上下文无关文法）。
        *   $V$：属性的有穷集。
        *   $F$：关于属性的断言或谓词的有穷集。

### 2. 属性类型
*   **综合属性 (Synthesized Attribute)**：
    *   分析树中，如果一个结点的属性值是通过其**子结点**的属性值计算得到的，则称为综合属性。
    *   *示例*：表达式求值。
        ```text
        E -> E1 + T   { E.val = E1.val + T.val }
        T -> F        { T.val = F.val }
        F -> digit    { F.val = digit.lexval }
        ```
*   **继承属性 (Inherited Attribute)**：
    *   分析树中，如果一个结点的属性值是由该结点的**父结点**和(或)**兄弟结点**的属性定义的，称为继承属性。
    *   *示例*：变量声明中的类型传递。
        ```text
        D -> T L      { L.in = T.type }
        T -> int      { T.type = integer }
        L -> id       { addtype(id.entry, L.in) }
        ```

## 6.3 中间语言

中间语言的设计需考虑从源语言到目标语言的翻译跨度及目标机指令集特点。主要形式包括：N元式、逆波兰式、图（树）。

### 1. 逆波兰式 (后缀式 / Reverse Polish Notation)
*   **形式定义**：$e_1 e_2 ... e_n \theta$ ($n \ge 1$)，即运算对象在前，运算符在后。
*   **特点**：适于栈式存储的计算机（堆栈机）。
*   **示例**：
    *   $a * b \rightarrow ab*$
    *   $-a \rightarrow a@$ (@表示单目负)
    *   $a+b*(c+d)*(e+f) \rightarrow abcd+*ef+*+$
*   **控制流处理**：引入操作符 `JF` (Jump if False) 和 `J` (Jump)。
    *   `IF <expr> then <s1> else <s2>` 的逆波兰结构：
        1. `<expr> <label_false> JF`
        2. `<s1>`
        3. `<label_next> J`
        4. `<label_false>: <s2>`
        5. `<label_next>:`

### 2. N元式 (N-tuples)
N个域的记录结构 $(D_1, D_2, ..., D_n)$，包含操作符域和操作对象域。

#### A. 三元式 (Triples)
*   **形式**：`NO. (OP, ARG1, ARG2)`
*   **特点**：
    *   `ARG1` 和 `ARG2` 可以是常数、变量名，也可以是前面某一个三元式的**编号**（代表该三元式的计算结果）。
    *   **缺点**：优化时若移动三元式顺序，引用该结果的编号需要全部修改，维护困难。
*   **示例**：$X = A + B * C$
    1. `(*, B, C)`
    2. `(+, A, (1))`
    3. `(=, X, (2))`

#### B. 间接三元式 (Indirect Triples)
*   **结构**：由**间接码表**（指向三元式表的指针列表）和**三元式表组成**。
*   **优点**：优化时只需调整间接码表的顺序，无需修改三元式表内部的引用，时空效率类似于四元 each。
*   **示例**：$(a+b)^2 + (a+b)^3$
    *   **三元式表**：
        1. `(+, a, b)`
        2. `(**, (1), 2)`
        3. `(+, a, b)`  *(注：此处可复用(1)，但通常为了清晰或不同上下文可能重新列出，间接三元式通过指针共享)*
        4. `(**, (1), 3)` *(假设复用(1))*
        5. `(+, (2), (4))`
    *   **间接码表**：按执行顺序排列的指针序列。

#### C. 四元式 (Quadruples)
*   **形式**：`NO. (OP, ARG1, ARG2, Result)`
*   **特点**：
    *   显式指定结果存放位置 `Result`（通常是临时变量 $T_i$）。
    *   **优点**：便于优化，因为操作数直接指向变量或临时变量，不依赖指令序号。
    *   **缺点**：比三元式多占用空间。
*   **示例**：$X = A + B * C$
    1. `(*, B, C, T1)`
    2. `(+, A, T1, T2)`
    3. `(=, T2, _, X)`

### 3. 图/树 (Graphs/Trees)
*   **结构**：一个三元式对应一棵子树。
    *   `OP` 为根节点。
    *   `ARG1` 为左子节点。
    *   `ARG2` 为右子节点。
*   **关系**：
    *   语法分析的结果通常为树。
    *   树的后序遍历 $\rightarrow$ 逆波兰式。
    *   树的线性化表示 $\rightarrow$ 三元式/四元式。

### 4. 各种表示法的比较综述
*   **逆波兰式**：适于栈式机器。
*   **四元式**：便于优化，常用。
*   **间接三元式**：优化时的时空效率类似四元式，且节省空间。
*   **三元式**：优化困难。

## 6.4 符号表 (Symbol Table)

### 1. 作用
*   存放源程序中有关标识符的属性信息的数据结构。
*   **语义检查依据**：如类型检查、作用域检查。
*   **代码生成依据**：如地址分配、存储空间大小。
*   **动态性整个编译过程中动态地采集、记录、变更、引用。

### 2. 标识符的主要属性
1.  **名字 (Name)**：用于查重（考虑作用域）。
2.  **类型 (Type)**：决定存储空间分配及可施加的运算。
3.  **存储类别 (Storage Class)**：如 static, auto, register 等。
4.  **作用域/可视性 (Scope/Visibility)**：所在层次Level。
5.  **其他**：
    *   变量：地址 (Addr)。
    *   数组：维数 (Dim)、每维大小、体积。
    *   过程/函数：入口地址、形参表。
    *   标号：定义标志、地址。

### 3. 结构
通常包含 **名字域** 和 **属性信息域**。对于复杂类型（如结构体、数组），可能需要建立额外的信息向量表（内情向量表）。

## 6.6 语句翻译与中间代码生成

### 设计要点
1.  根据语义确定语句的目标结构（控制流图布局）。
2.  确定中间代码形式（如四元式）。
3.  构造语义子程序。
4.  关键技术：**拉链-返填技术** (Backpatching)。

### 6.6.1 说明类语句的翻译
*   **特点**：一般不产生目标代码，主要作用是填充符号表。
*   **常量说明**：将常量值填入符号表或常量表。
*   **简单变量说明**：
    *   文法：$D \rightarrow \text{int } id \mid \text{float } id \mid D, id$
    *   动作：调用 `fill(ENTRY(id), type)` 将类型填入符号表，并分配相对地址。
*   **复合类型说明 (Struct)**：
    *   结构成员连续存放。
    *   需记录每个成员的偏移量 (Offset) 和类型。
    *   结构整体不能直接引用，只能引用成员。
    *   **内情向量表**：记录结构的总长度、成员信息等。

### 6.6.2 赋值语句与表达式翻译

#### 1. 赋值语句 $A \rightarrow V = E$
*   **目标结构**：先计算表达式 $E$，再将结果存入 $V$。
*   **语义子程序**：
    ```text
    A -> i = E { GEN('=', E.PLACE, '_', ENTRY(i)) }
    ```
    *   `E.PLACE`：存放表达式值的变量或临时变量名。
    *   `ENTRY(i)`：左部变量在符号表中的入口。

#### 2. 表达式翻译
*   **文法**：$E \rightarrow E \ op \ E \mid op \ E \mid id$
*   **语义子程序**：
    1.  $E \rightarrow E_1 \ op \ E_2$:
        ```text
        { E.PLACE = NEWTEMP; 
          GEN(op, E1.PLACE, E2.PLACE, E.PLACE) }
        ```
    2.  $E \rightarrow op \ E_1$ (单目):
        ```text
        { E.PLACE = NEWTEMP; 
          GEN(op, E1.PLACE, '_', E.PLACE) }
        ```
    3.  $E \rightarrow id$:
        ```text
        { E.PLACE = ENTRY(id) }
        ```
*   **示例**：`a = b + c + d`
    1.  `(+, b, c, t1)`
    2.  `(+, t1, d, t2)`
    3.  `(=, t2, _, a)`

### 6.6.3 控制流类语句翻译

#### 1. 关键技术：拉链-返填 (Backpatching)
*   **问题**：在一遍扫描编译中，跳转目标标号可能在跳转语句之后才定义（先引用后定义此时目标地址未知）。
*   **解决**：
    *   **拉链 (Chaining)**：当遇到未定义标号的跳转指令时，将其地址字段暂填为0，并将该指令链接到一个以标号为头部的链表中。
    *   **返填 (Backpatching)**：当扫描到标号定义处时，知道其真实地址，然后遍历链表，将链表中所有跳转指令的地址字段修改为真实地址。
*   **数据结构**：
    *   **标号表 (LT)**：记录标号名、定义标志 (1:已定义, 0:未定义)、地址 (若已定义) 或 链头指针 (若未定义)。
    *   **代码区**：跳转指令中保存下一个同标号跳转指令的位置（形成链表）。

#### 2. 条件语句翻译
*   **布尔表达式属性**：
    *   `E.true`：条件为真时的出口标号。
    *   `E.false`：条件为假时的出口标号。

*   **If-Then**: `if (E) then S1`
    ```text
    S -> if E then S1
    { E.true = newlabel; E.false = newlabel; // 或者 E.false 指向 S 之后的代码
      S.code = E.code | GEN(jT, _, _, E.true) | GEN(jF, _, _, E.false) 
             | LABEL(E.true) | S1.code | LABEL(E.false) }
    ```
    *(注：具体实现中，若没有else，E.false通常直接指向S1之后的下一条指令，无需生成显式的jF，或者生成jF指向出口)*

*   **If-Then-Else**: `if (E) then S1 else S2`
    ```text
    S -> if E then S1 else S2
    { E.true = newlabel; E.false = newlabel; S.next = newlabel;
      S.code = E.code 
             | GEN(jT, _, _, E.true) 
             | GEN(jF, _, _, E.false)
             | LABEL(E.true) | S1.code | GEN(j, _, _, S.next)
             | LABEL(E.false) | S2.code
             | LABEL(S.next) }
    ```

#### 3. 循环语句翻译
*   **While**: `while (E) do S1`
    ```text
    S -> while E do S1
    { S.begin = newlabel; E.true = newlabel; E.false = newlabel;
      S.code = LABEL(S.begin) | E.code 
             | GEN(jT, _, _, E.true) 
             | GEN(jF, _, _, E.false)
             | LABEL(E.true) | S1.code | GEN(j, _, _, S.begin)
             | LABEL(E.false) }
    ```

*   **For**: `for (e1; e2; e3) S`
    *   逻辑等价于：
        ```c
        e1;
        L1: if (!e2) goto L2;
        S;
        e3;
        goto L1;
        L2:
        ```
    *   **四元式结构**：
        1.  `e1.code`
        2.  `LABEL(L1)`
        3.  `e2.code`
        4.  `GEN(jF, _, _, L2)`  // e2为假跳出
        5.  `LABEL(L3)` // 可选，或直接接S
        6.  `S.code`
        7.  `GEN(j, _, _, L4)` // 跳过e3后的回跳？不，通常是S执行完直接去e3
           *(修正：标准for循环结构如下)*
           
           **更准确的四元式序列**：
           8. `e1.code`
           9. `LABEL(L1)`
           10. `e2.code`
           11. `GEN(jF, _, _, L2)`  // 条件失败去L2(出口)
           12. `LABEL(L3)` // 循环体入口? 不，通常S紧接着
           13. `S.code`
           14. `LABEL(L4)` // e3入口
           15. `e3.code`
           16. `GEN(j, _, _, L1)` // 回到条件判断
           17. `LABEL(L2)` // 出口

#### 4. 多分支语句 (Switch)
*   **结构**：
    1.  计算表达式 $e$。
    2.  生成测试表或一系列比较跳转。
    3.  若 $e == c_1$ goto $L_1$; ...
    4.  若都不匹配，goto $L_{default}$。
    5.  各 case 代码块末尾加 `goto next` (break)。

### 6.6.4 数组说明与数组元素引用的翻译

#### 1. 数组说明
*   **信息登记**：在符号表中登记数组名、类型、维数、每维上下界、总体积。
*   **内情向量表 (Descriptor)**：对于多维数组或动态数组，单独存储计算地址所需的参数（如基地址 $a_0$、常数部分 $C$、维数信息等）。

#### 2. 数组元素地址计算
*   **存储方式**：通常按行优先（Row-major）或列优先。C语言为按行优先。
*   **地址公式**（以二维数组 $A[d_1][d_2]$，下标从0开始为例）：
    $$ Addr(A[i][j]) = a_0 + (i \times d_2 + j) \times w $$
    其中 $w$ 是每个元素的大小。
*   **一般化公式**（$n$维，下标下限可能不为0）：
    $$ Addr = a_0 - C + V $$
    *   **不变部分 (Constant Part, C)**：编译时可计算。
        $$ C = (low_1 \times d_2 \dots d_n + \dots + low_n) \times w $$
        *(具体公式依下标下限而定，文档中给出的是下限为1时的推导，最终形式为 $a_0 - C$)*
    *   **可变部分 (Variable Part, V)**：运行时计算。
        $$ V = (i_1 \times d_2 \dots d_n + i_2 \times d_3 \dots d_n + \dots + i_n) \times w $$
*   **目标代码结构**：
    1.  计算 $V$ 的代码。
    2.  计算 $CUN = a_0 - C$ (编译时算出常数，运行时加载)。
    3.  $T = CUN + V$ (得到最终地址)。
    4.  引用 $T$ 处的值。

*   **动态数组**：
    *   编译时仅分配描述符空间。
    *   运行时确定维数大小，计算 $C$ 和体积，分配数据空间，填写描述符。
    *   生成额外的初始化代码。

### 6.6.5 过程/函数说明和调用的翻译

#### 1. 函数说明
*   登录函数名、入口地址到符号表。
*   记录局部量的作用域 (Level)。
*   为形参和局部变量分配活动记录 (Activation Record) 中的偏移量。

#### 2. 函数调用 `call id(Elist)`
*   **语义处理步骤**：
    1.  **检查**：函数是否定义？实参/形参数量、类型、顺序是否匹配？
    2.  **计算实参**：生成计算实参表达式的代码。
    3.  **传递参数**：生成将实参值送入约定位置（寄存器或栈）的代码。
    4.  **保存现场**：保存返回地址、主调函数的状态。
    5.  **转移**：生成跳转到函数入口的代码 (`CALL` 或 `JUMP`)。
*   **返回处理**：
    1.  将返回值存入约定位置。
    2.  恢复主调函数的活动记录。
    3.  生成返回指令 (`RETURN`)。

## 课后习题解析 (基于文档末尾内容)

### 选择题
1.  **逆波兰式**：句子 `i+i+(i*i)*i`
    *   语法树推导：
        *   `i*i` -> `ii*`
        *   `(i*i)*i` -> `ii*i*`
        *   `i+(i*i)*i` -> `iii*i*+`
        *   `i+i+...` -> `ii+iii*i*+`
    *   选项分析：
        *   A) `iii*++` (错)
        *   B) `ii+iii**+` (错，运算顺序不对)
        *   C) `ii+ii*i*+` (对应 `i+i+(i*i)*i` ? 让我们仔细检查)
            *   `i+i` -> `ii+`
            *   `i*i` -> `ii*`
            *   `(i*i)*i` -> `ii*i*`
            *   `... + ...` -> `ii+ ii*i* +`
            *   结果：`ii+ii*i*+`
    *   **答案：C** (注意：原题选项C写作 `ii+ii*i*+`，符合推导)

2.  **中间代码生成依据**：
    *   中间代码是语义分析的输出，依据的是**语义规则**。
    *   **答案：C**

3.  **与生成中间代码目的无关的是**：
    *   目的：便于优化、便于移植、便于生成目标代码。
    *   “便于存储空间的组织”主要是符号表和运行环境管理的任务，虽然中间代码有助于此，但不是引入中间代码的*核心*初衷（核心是解耦前端后端以利于优化和移植）。但在某些语境下，B也可能被视为相关。然而，对比其他三个选项（优化、移植、生成目标代码是教科书标准的三大理由），B是最弱的关联。
    *   **答案：B** (通常标准答案为B，因为存储组织更多依赖于符号表和激活记录管理，而非中间代码本身的存在理由)

4.  **不采用拉链-返填技术的语句**：
    *   拉链-返填主要用于解决**控制流**中向前跳转目标未知的问题。
    *   转向语句 (Goto)、条件语句 (If)、循环语句 (While/For) 都涉及跳转。
    *   **赋值语句**是顺序执行的，不涉及跳转标号的先引用后定义问题。
    *   **答案：B**

### 判断题
1.  **使用语法制导翻译方法的编译程序能同时进行语法分析和语义分析。**
    *   **正确**。这是语法制导翻译的核心特征。
2.  **返填就是稍后填写转移指令的地址。**
    *   **正确**。
3.  **对任何一个编译程序,产生中间代码是不可缺少的。**
    *   **错误**。简单的编译器可以直接生成目标代码，不一定需要显式的中间代码表示（尽管现代编译器几乎都有）。
4.  **三元式同间接三元式是等价的。**
    *   **错误**。它们在表示能力上等价，但在优化性能和实现机制上不同（间接三元式更利于优化）。如果是指“表达的语义等价”，那是对的；但如果指“数据结构等价”，则是错的。通常这类题目考察的是它们的区别，故判错的可能性大，或者题目意指“可以相互转换且表达相同逻辑”，但在编译原理语境下，强调其**不等价**的实现特性。*注：根据常见题库，此题通常判**错**，因为间接三元式引入了间接层，结构不同。*
5.  **语法制导翻译方法可用来产生各种中间代码，又可用来产生目标代码。**
    *   **正确**。

**2. 语句标号表内容 (一遍扫描，拉链-返填)**

假设编译器按顺序扫描，遇到标号引用时标号未定义，遇到标号定义时进行返填。

| 语句标号 (Li) | 定义与否标志 (1:已定义, 0:未定义) | Addr (地址或链头) | 说明 |
| :--- | :---: | :--- | :--- |
| **L1** | 0 | *Chain_Ptr_1* | 在 `for` 开头被引用（跳转回来），此时未定义。链头指向第一条引用它的跳转指令。 |
| **L2** | 0 | *Chain_Ptr_2* | 在 `S1` 或 `S2` 后可能被引用（跳去增量），此时未定义。 |
| **L3** | 1 | *Addr_L3* | `if` 条件处，定义即使用，通常直接生成代码，或标记为已定义。 |
| **L4** | 1 | *Addr_L4* | `S1` 入口，定义即使用。 |
| **L5** | 1 | *Addr_L5* | `S2` 入口，定义即使用。 |
| **(返填发生时)** | | | |
| **L1** | 1 | *Real_Addr_L1* | 当扫描到 `L1:` 标记处（循环开始），更新标号表为1，并遍历 *Chain_Ptr_1* 修改所有指向L1的跳转指令地址。 |
| **L2** | 1 | *Real_Addr_L2* | 当扫描到 `L2:` 标记处（增量开始），更新标号表为1，并返填。 |

**实际填写顺序（模拟扫描过程）：**

1.  遇到 `goto L1` (循环尾部的回溯): L1未定义。
    *   表项: `L1 | 0 | ptr_to_this_jump`
2.  遇到 `goto L2` (if break? 或循环步): L2未定义。
    *   表项: `L2 | 0 | ptr_to_this_jump`
3.  遇到 `L3:` (定义):
    *   表项: `L3 | 1 | current_addr`
4.  遇到 `L4:` (定义):
    *   表项: `L4 | 1 | current_addr`
5.  遇到 `L5:` (定义):
    *   表项: `L5 | 1 | current_addr`
6.  遇到 `L1:` (定义，循环头):
    *   更新表项: `L1 | 1 | current_addr`
    *   **执行返填**：找到 `ptr_to_this_jump`，填入 `current_addr`。
7.  遇到 `L2:` (定义，增量头):
    *   更新表项: `L2 | 1 | current_addr`
    *   **执行返填**：找到 `ptr_to_this_jump`，填入 `current_addr`。
